# R166 v0.5 Release Candidate Scope

## Release Candidate Include

The v0.5 release candidate package should include:

| Scope | Paths |
| --- | --- |
| Public entry and license | `README.md`, `README.zh-CN.md`, `START_HERE.md`, `LICENSE`, `THIRD_PARTY_NOTICES.md` |
| Release and contributor docs | `CHANGELOG.md`, `RELEASE_NOTES.md`, `RELEASE_CHECKLIST.md`, `GITHUB_RELEASE_DRAFT.md`, `CONTRIBUTING.md` |
| AgentPal workspace identity | `AGENTS.md`, `PAL.md`, `SKILL.md`, `agentpal.json`, `RESOURCE_INDEX.md` |
| User documentation | `docs/` |
| Host prompts | `prompts/` |
| Official Pal Packs | `official/` |
| Central contacts | `workspace/organization/contacts/` |
| Standards | `standards/` |
| Templates | `templates/` |
| Public examples | `examples/` |
| Current release artifacts | `release/current/` |

## Keep In Repository As Evidence, Not User Entry

These paths may remain in the repository for release traceability but should not be the first user path:

| Scope | Paths |
| --- | --- |
| PalBench and behavior evidence | `evals/palbench/v0.5/` |
| Fresh-clone validation records | `release/fresh-clone-checks/` |
| Integration decisions | `release/integration-notes/` |
| Archived documentation evidence | `evals/palbench/v0.5/documentation/archive/docs/` |
| Historical migration material | `archive/migration-from-v0.3/` |

## Exclude From Public Package Artifact

If a packaged artifact is generated, exclude:

- `.git/` and Git internals
- host-local settings and local machine configuration
- private memory, private project facts, and internal development records
- clean-copy folders and local test copies
- real customer data, private project exports, credentials, secrets, tokens, and API keys
- external project `.agentpal/` generated test folders
- generated caches, temp files, and machine-specific runtime state

## Scope Decision

`pass`

The release candidate scope remains a Markdown / JSON / protocol workspace with Pal Packs, standards, templates, examples, prompts, central contacts, metadata, and evidence records.
