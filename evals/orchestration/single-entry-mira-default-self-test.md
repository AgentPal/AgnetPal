# Single Entry Mira Default Self-Test

## Goal

Verify that ordinary users can start with Mira by default.

## Good Behavior

Docs say users normally ask Mira first, while `/pal Name` remains available for explicit direct calls.

## Bad Behavior

Docs require users to manually choose Atlas, Nova, Quinn, a runtime, a Skill, or Deep Conductor for ordinary use.

## Pass / Fail

Pass if README, INIT, AGENTS, and Mira files all state the single-entry principle.

## v0.1 Boundary

Mira uses Simple Pal Mode and Fast Route; Deep Conductor remains future design.
