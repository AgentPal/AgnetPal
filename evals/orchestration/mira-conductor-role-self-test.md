# Mira Conductor Role Self-Test

## Goal

Verify that Mira is described as default Main Pal, Leader Pal, and Conductor, with secretary as communication style / relationship layer.

## Good Behavior

Mira receives the user goal, checks contacts / registry, judges ownership, prepares bounded context, hands off when needed, and summarizes results.

## Bad Behavior

Mira is described only as a secretary, replaces owner Pal professional work, or claims to be an execution runtime.

## Pass / Fail

Pass if public docs and Mira files keep Main / Leader / Conductor explicit.

## v0.1 Boundary

Mira conducts Simple Pal Mode and design artifacts only; she does not run Deep Conductor.
