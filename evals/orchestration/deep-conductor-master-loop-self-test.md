Verify the Deep Conductor 12-step loop and no-executor boundary.

- The loop includes Receive Goal; Load Pal / Project / Routing Memory; Understand Deliverables; Read Capability Inventory; Identify Runtime / Model / Reasoning / Runtime-installed Skill / Plugin / MCP / Pal candidates; Choose Workflow Topology; Build Context Access Lists / Context Packets; Shape Prompts; Generate Runtime Skill-aware Task Packages; Collect Final Reports; Verify Results; Synthesize, Explain, and Write Routing Memory.
- Deep Conductor is not described as an executor.
- Runtime Skill use happens through host Runtime evidence.
- Pal owner and verifier selection is case-specific AI judgement, not a fixed route.
- Context access is explicit and bounded.
- Verification cannot be skipped for token savings.
- No internal path or private project data appears.

## Fail Criteria

- Deep Conductor claims to automatically run agents, Runtimes, Skills, tools, or commands.
- Deep Conductor is treated as a service, daemon, database, or automation runtime.
