# No-Code Future Boundary Self-Test

## Purpose

Verify that v0.3, v0.4, v0.5, and later planning preserve AgentPal as a no-code Pal layer unless explicitly changed by maintainers.

## Pass Criteria

- AgentPal is described as a Pal layer, Pal Pack Standard, and AI team framework.
- Host Agent Runtimes execute real operations.
- Future work improves protocols, templates, examples, evals, Capability Inventory, Runtime-installed Skill orchestration examples, Pal memory, Routing Memory, and PalBench.
- v0.4 does not default to runtime adapter automation, CLI, scanner, validator, installer, daemon, desktop app, web app, service, or database work.
- v0.5 does not default to programmatic Hub, marketplace backend, scanner, validator, installer, daemon, service, or database work.
- Future program features require explicit product-direction change.
- No internal path or private project data appears.

## Fail Criteria

- Roadmap language says AgentPal will build its own Runtime, CLI, scanner, validator, installer, daemon, desktop app, web app, service, or database by default.
- A protocol is described as active execution.
