# PBC-009 Direct Owner Suggests Candidate

## User Input

```text
/pal Vega Research this API change and prepare the implementation plan too.
```

## Expected Context Packet

- direct Pal candidate: Vega;
- Vega can own research framing;
- implementation stage needs another Pal-layer owner candidate;
- runtime candidate may execute later from task package.

## Expected Workflow

Direct owner Pal staged judgement.

## Expected Output

- Vega accepts research-stage ownership;
- Vega names implementation-stage candidate, such as a development Pal candidate, based on current contacts / registry;
- Vega may recommend Mira resume conductor responsibility for the multi-stage task;
- candidates are not fixed routes.

## Failure Modes

- Vega claims all implementation ownership;
- "API/code means Atlas" is stated as a rule;
- Runtime owns implementation without Pal-layer owner;
- clarification is asked before stage judgement.

## Scoring Rubric

Score 0 / 1 / 2 for direct Pal respect, staged owner judgement, implementation owner candidate, candidate language, and runtime boundary.
