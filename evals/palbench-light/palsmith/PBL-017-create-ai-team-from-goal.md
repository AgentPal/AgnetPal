# PBL-017 - Create AI Team From Goal

## Category

PalSmith / AI team.

## User Input

```text
/pal PalSmith Build an AI team for a local services business that plans offers, writes copy, manages docs, and verifies launches.
```

## Baseline Behavior

A direct assistant may list many vague agents or create a fixed dispatch table.

## Expected AgentPal Behavior

PalSmith proposes a small no-code Pal team design with entry/leader expectation, member boundaries, collaboration rules, Context Packet needs, and a creation package. Team members are candidate capability holders, not permanent route targets.

## Required Assets / Context

- PalSmith AI team creation guidance.
- User goal, recurring work, desired team size, privacy boundary.
- Existing Pal list only as current contacts / registry context.

## Expected Task Judgement

- domain_focus: AI team / Pal asset design.
- selected owner: PalSmith for team blueprint and governance package.
- stage candidates: product, research, writing, document, quality, or system candidates only as the case supports.
- verification_needs: no overlap, no fixed route, no automatic execution claim, health check.

## Expected Output Shape

- focused clarification;
- small team blueprint;
- role cards and boundaries;
- collaboration and Context Packet rules;
- Runtime Task Package and team health-check plan.

## Scoring Rubric

| Metric | 0 | 1 | 2 |
| --- | --- | --- | --- |
| task_understanding | Produces a big vague agent list. | Team is scoped but lacks governance. | Small team with clear roles and evidence needs. |
| no_hardcoded_routing | Creates permanent task assignment table. | Some candidate language. | Member roles are candidate capability holders. |
| owner_judgement | No PalSmith governance. | PalSmith speaks but weak boundaries. | PalSmith owns blueprint and package. |
| no_future_as_current | Claims team runs automatically. | Ambiguous. | Simple Pal Mode and no-code boundary are clear. |
| verification_quality | No health checks. | Basic review. | Team health and repair checks are defined. |

## Failure Modes

- The team is described as an active multi-agent runtime.
- Member Pals are assigned permanent natural-language triggers.
- Private business material is assumed public.

## Notes

- This is an agent-usage workflow eval, not a model benchmark.
- Candidates are not fixed routes.
- Deep Conductor is not active unless explicitly marked as future design.
