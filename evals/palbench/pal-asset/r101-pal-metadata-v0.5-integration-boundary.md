# R101 Pal Metadata v0.5 Integration Boundary

Date: 2026-06-28

## Purpose

This eval checks that R101 integrates R100-A/B/C/D Pal Metadata v0.5 outputs
into shared navigation without expanding AgentPal into a runtime, scanner,
connector, marketplace, keyword router, or official Pal bulk-upgrade task.

## Pass Criteria

- `README.md` and `README.zh-CN.md` link the Pal Metadata v0.5 upgrade path.
- `RESOURCE_INDEX.md` contains sections for Pal Metadata v0.5, Official Pal
  Metadata Upgrade Preparation, Pal Asset Index Templates, and R101 Integration.
- `agentpal.json` exposes a `pal_metadata_v0_5` metadata index with false safety
  flags.
- `docs/09-roadmap/v0.5-local-development-scope.md` names Pal Metadata v0.5 as
  a local no-code development scope.
- `docs/00-overview/pal-metadata-v0.5-upgrade-path.md` exists.
- R101 integration notes and validation records exist.
- R100-A/B/C/D public assets remain present.
- No official Pal files are modified by R101.
- Central contacts are not modified by R101.
- No real official `asset-manifest.json` files are generated by R101.
- No active keyword/domain/role route map is introduced.
- No external project `.agentpal/` thick binding is introduced.
- No scanner, validator program, connector, marketplace, daemon, installer, auto
  sync, or auto execution behavior is introduced.
- No credential, token, cookie, password, API key, or secret value is introduced.

## Fail Conditions

- A shared entry claims that official Pals are already upgraded to `pal.json`
  v0.5 or already have real `asset-manifest.json` files when current evidence
  does not show that.
- R101 edits `official/pals/**`.
- R101 edits `workspace/organization/contacts/**`.
- R101 creates route maps or keyword dispatch tables.
- R101 copies Pal metadata or Pal assets into external project `.agentpal/`
  directories by default.
- R101 adds code or dependency manifests for scanner / validator / connector /
  marketplace behavior.
