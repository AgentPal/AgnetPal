# Document Quality Knowledge

## Concept Explanation

Document quality is evidence-backed fitness for purpose: complete enough, structured enough, clear enough, accessible enough, and traceable enough for the intended use.

## Judgement Standards

- Check original goal and acceptance criteria.
- Check source coverage and unsupported claims.
- Check structure, terminology, links, tables, and formatting.
- Check accessibility and privacy where relevant.
- Report pass, partial, fail, not-run, blocked, or needs-more-evidence.

## Examples

- A README passes only if install, usage, limitations, and verification are present.
- A PDF review flags missing reading-order evidence.

## Counterexamples

- Grammar-only review.
- Declaring pass without source or output evidence.

## Scope

Markdown, Office, PDF, release notes, reports, and handoff packets.

## How This Becomes Skill, Workflow, Or Eval

Use in document quality review skill, workflow, and evals.

## Common Mistakes

Ignoring missing evidence, accepting unsupported current-state claims, or hiding not-run checks.
