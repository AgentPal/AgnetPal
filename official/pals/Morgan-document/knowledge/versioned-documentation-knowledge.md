# Versioned Documentation Knowledge

## Concept Explanation

Versioned documentation states which release, branch, time, or artifact revision a claim belongs to.

## Judgement Standards

- Label current, future, deprecated, and removed behavior.
- Map changes to affected docs.
- Preserve anchors and links where possible.
- Keep release notes synchronized with user-visible changes.

## Examples

- `v0.1.0-rc.1` docs state Simple Pal Mode only.
- Migration notes explain what changed and what users must do.

## Counterexamples

- Future design written as active behavior.
- Changelog omitted for user-visible changes.

## Scope

Release docs, README updates, migration notes, changelogs, and project documentation sets.

## How This Becomes Skill, Workflow, Or Eval

Use in versioned documentation skill and release notes workflow.

## Common Mistakes

Missing version labels, stale links, and mixed release states.
