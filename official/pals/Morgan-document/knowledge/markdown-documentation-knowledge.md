# Markdown Documentation Knowledge

## Concept Explanation

Markdown documentation uses plain-text structure for headings, lists, links, tables, and code fences that can render predictably.

## Judgement Standards

- Use heading levels for structure, not visual emphasis.
- Keep links and paths accurate.
- Use tables for scan-friendly structured data.
- Keep Markdown readable in source form.

## Examples

- `## Inputs` before a list of task inputs.
- A table mapping source IDs to claims.

## Counterexamples

- Skipping heading levels randomly.
- Putting long paragraphs inside narrow tables.

## Scope

README files, docs folders, runbooks, evals, reports, and AgentPal Pal assets.

## How This Becomes Skill, Workflow, Or Eval

Use in `markdown-documentation-skill.md`, `markdown-docs-workflow.md`, and Markdown docs eval.

## Common Mistakes

Broken relative links, ambiguous anchors, and code fences without language labels.
