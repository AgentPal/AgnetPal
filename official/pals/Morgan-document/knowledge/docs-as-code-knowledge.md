# Docs As Code Knowledge

## Concept Explanation

Docs-as-code applies repository, review, versioning, and release habits to documentation files.

## Judgement Standards

- Store docs with traceable changes.
- Review docs alongside behavior changes when relevant.
- Keep current, future, and deprecated behavior separated.
- Require changed-file evidence and review status.

## Examples

- README and release notes update in the same bounded change set.
- Migration guide linked from changelog.

## Counterexamples

- Docs updated from memory without source evidence.
- Future behavior written as current behavior.

## Scope

Repository documentation, versioned docs, release docs, changelogs, and docs maintenance.

## How This Becomes Skill, Workflow, Or Eval

Use in `versioned-documentation-skill.md`, `versioned-documentation-workflow`, and release notes eval.

## Common Mistakes

No version label, unreviewed docs drift, or hidden breaking changes.
