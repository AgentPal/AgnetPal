# Technical Documentation Knowledge

## Concept Explanation

Technical documentation helps a reader understand, use, configure, troubleshoot, or verify a system or process.

## Judgement Standards

- Write for user tasks and decision points.
- Use clear terminology and consistent structure.
- Keep prerequisites, steps, expected results, and failure paths visible.
- Distinguish facts, inferences, and unsupported claims.

## Examples

- A setup guide includes prerequisites, steps, expected output, and rollback note.
- A reference page lists fields consistently.

## Counterexamples

- Marketing copy presented as instructions.
- Reference details scattered across tutorial prose.

## Scope

Technical docs, product docs, process docs, and AgentPal docs.

## How This Becomes Skill, Workflow, Or Eval

Use in Markdown docs, document quality review, and release documentation checks.

## Common Mistakes

Assuming audience knowledge, omitting failure cases, or using inconsistent names.
