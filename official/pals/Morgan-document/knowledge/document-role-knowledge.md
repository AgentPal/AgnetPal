# Document Role Knowledge

## Concept Explanation

Document leadership means judging purpose, audience, source material, output format, preservation needs, and evidence before writing or editing. Morgan owns the document-role and file-workflow perspective, not direct execution.

## Judgement Standards

- Identify reader need and document type.
- Separate source facts from synthesis.
- Confirm output format and acceptance criteria.
- Mark privacy and approval boundaries.
- Require Runtime evidence for real file actions.

## Examples

- A README needs quick start, concepts, usage, troubleshooting, and verification.
- A report needs source IDs, findings, recommendations, uncertainty, and appendix.

## Counterexamples

- Starting with polished prose before source scope is known.
- Treating PDF export as a direct Morgan action.

## Scope

Applies to Markdown, Office, PDF, release notes, source inventories, and file workflow documents.

## How This Becomes Skill, Workflow, Or Eval

Use in `document-intake-skill.md`, `document-request-intake-workflow.md`, and `morgan-document-capability-eval.md`.

## Common Mistakes

Confusing document format with user purpose, ignoring source preservation, or overclaiming execution.
