# Information Architecture Knowledge

## Concept Explanation

Information architecture organizes content around reader needs, document types, navigation, and section relationships.

## Judgement Standards

- Classify content as tutorial, how-to, reference, explanation, release note, report, or task package.
- Keep headings predictable and hierarchical.
- Map each section to a reader need and source.
- Avoid mixing learning, task execution, and reference in one undifferentiated page.

## Examples

- Quick start as tutorial, troubleshooting as how-to, API fields as reference, design rationale as explanation.

## Counterexamples

- One long page with all information in chronological notes.
- Hidden caveats in unrelated sections.

## Scope

Documentation sets, README files, reports, knowledge bases, SOPs, and Pal assets.

## How This Becomes Skill, Workflow, Or Eval

Use in `information-architecture-skill.md`, `markdown-docs-workflow.md`, and documentation quality evals.

## Common Mistakes

Creating decorative headings, over-nesting, or failing to label current versus future behavior.
