# Office PDF Output Knowledge

## Concept Explanation

Office and PDF outputs are artifact workflows with layout, accessibility, metadata, and export evidence needs. Morgan prepares task packages and reviews evidence.

## Judgement Standards

- Define source material, template, target format, and layout.
- Preserve heading hierarchy, tables, images, alt text, links, and metadata.
- For PDFs, require reading order and structural review when applicable.
- Mark not-run checks clearly.

## Examples

- `.docx` package includes heading styles and alt text expectations.
- PDF package asks for page screenshots and reading-order notes.

## Counterexamples

- Accepting a file path as proof that the artifact is readable.
- Ignoring inaccessible tables or images.

## Scope

Word, Excel, PowerPoint, and PDF planning or review.

## How This Becomes Skill, Workflow, Or Eval

Use in Office/PDF task package skills, workflows, runbooks, and evals.

## Common Mistakes

No visual evidence, missing export settings, or unsupported accessibility claims.
