# Source Material Organization Skill

## Purpose

Turn raw materials into a traceable source inventory before content is rewritten, summarized, or exported.

## When to use

Use when sources include pasted text, meeting notes, PDFs, Office files, screenshots, CSVs, links, or mixed project files.

## Inputs

Source list, allowed paths, file types, timestamps, authorship if known, target document, and preservation needs.

## Required context

Allowed read scope, privacy level, copyright boundary, and desired source-reference style.

## Process

1. Inventory each source with ID, type, status, and permission boundary.
2. Classify material as facts, instructions, examples, decisions, constraints, open questions, or artifacts.
3. Group related materials into sections.
4. Mark conflicts, duplicates, stale items, and missing context.
5. Prepare a source-to-output map.

## Output

Source inventory, grouping table, preservation notes, missing-source list, and proposed output mapping.

## Quality bar

Every major output section must point back to one or more source IDs or be labeled as inference.

## User confirmation points

Confirm reading private files, using proprietary material, discarding duplicates, or summarizing sensitive content.

## No-code boundary

No automated scanning or extraction is added to AgentPal.

## Common mistakes

Starting with a polished draft before source IDs exist, merging conflicting sources silently, or over-sharing raw content.

## Example

Morgan labels meeting notes as S01, product brief as S02, screenshot as S03, and maps them to overview, workflow, risks, and appendix.
