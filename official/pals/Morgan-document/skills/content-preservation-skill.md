# Content Preservation Skill

## Purpose

Preserve important source details while transforming material into a new document or Pal asset.

## When to use

Use for long source material, user-provided SOPs, interviews, requirements, legal-ish drafts, technical notes, or any document where detail loss is risky.

## Inputs

Source inventory, target output, preservation priority, allowed compression level, and sensitive-content rules.

## Required context

Content preservation protocol, source IDs, must-keep facts, examples, terminology, decisions, and exclusions.

## Process

1. Identify must-preserve claims, terms, examples, decisions, warnings, and edge cases.
2. Separate source facts from synthesis.
3. Keep a traceability note for each major transformed section.
4. Mark omissions explicitly.
5. Ask for user confirmation before aggressive shortening.

## Output

Preservation checklist, transformation map, omitted-content note, uncertainty list, and review questions.

## Quality bar

A reader should be able to see what was preserved, transformed, omitted, and why.

## User confirmation points

Confirm deletion of detail, anonymization, sensitive-source sharing, and final wording for high-stakes material.

## No-code boundary

This skill plans preservation and review. It does not run extraction or comparison tools.

## Common mistakes

Summarizing away edge cases, changing terminology without marking it, or hiding uncertainty.

## Example

For a 30-page SOP, Morgan preserves roles, steps, approval gates, exceptions, examples, and evidence requirements before drafting a shorter runbook.
