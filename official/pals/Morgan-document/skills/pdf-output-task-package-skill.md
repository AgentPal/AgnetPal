# PDF Output Task Package Skill

## Purpose

Prepare PDF output, export, or review task packages with accessibility and source-preservation requirements.

## When to use

Use when the final artifact is PDF, when a PDF must be reviewed, or when source material needs PDF export planning.

## Inputs

Source material, desired PDF purpose, source format, reading order needs, accessibility needs, metadata, and review expectations.

## Required context

PDF source type, text/scanned/mixed status if known, target audience, export path, and evidence requirement.

## Process

1. Identify whether the PDF is generated, converted, reviewed, extracted, or remediated.
2. Define reading order, tags, headings, tables, links, images, metadata, and page checks.
3. Mark privacy and copyright constraints.
4. Require Runtime output and accessibility/review evidence.
5. Record not-run items when evidence is absent.

## Output

PDF Runtime Task Package with source map, export/review constraints, accessibility checks, privacy boundary, and evidence checklist.

## Quality bar

The package must distinguish visual appearance from structural readability and review evidence.

## User confirmation points

Confirm external PDF sharing, OCR of private scans, publication, redaction, and irreversible changes.

## No-code boundary

Morgan does not convert, OCR, remediate, or validate PDFs directly.

## Common mistakes

Judging a PDF only by appearance, ignoring reading order, or claiming accessibility without tool evidence.

## Example

Morgan asks the Runtime to export a report to PDF, return output path, page screenshots, tag/reading-order notes, and any accessibility checker failures.
