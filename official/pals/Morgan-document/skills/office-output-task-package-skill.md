# Office Output Task Package Skill

## Purpose

Prepare a safe task package for Word, Excel, or PowerPoint output without making Morgan operate Office directly.

## When to use

Use when the final artifact is `.docx`, `.xlsx`, `.pptx`, or an Office-derived deliverable.

## Inputs

Source material, target Office format, template if any, layout rules, accessibility needs, charts/tables/images, and review evidence.

## Required context

Office output intent, source IDs, required styles, heading hierarchy, table rules, alt text needs, and allowed Runtime tools.

## Process

1. Define target artifact and template.
2. Map content to sections, slides, sheets, tables, or fields.
3. Specify style, accessibility, metadata, and review requirements.
4. Mark source preservation and privacy rules.
5. Require Runtime output path and visual or structural evidence.

## Output

Office Runtime Task Package with goal, inputs, layout map, content map, constraints, confirmation points, and evidence checklist.

## Quality bar

The task package must let the Runtime produce and verify the Office artifact without guessing formatting or source scope.

## User confirmation points

Confirm templates, branding, private source use, generated images, charts, and external sharing.

## No-code boundary

Morgan does not run Office automation or create export tooling.

## Common mistakes

Treating Office export as plain text, omitting headings/styles, and accepting output without rendered or structural evidence.

## Example

Morgan prepares a `.docx` package with title page, heading styles, tables, image alt text, appendix, output path, and review evidence.
