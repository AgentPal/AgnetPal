# Document Handoff Skill

## Purpose

Create a compact handoff packet for another Pal, a Runtime, or the user without losing document context.

## When to use

Use when document work needs implementation, research, quality review, product decision, writing polish, system safety, or Pal asset governance.

## Inputs

Goal, source inventory, current draft status, owner judgement, constraints, privacy boundary, and expected output.

## Required context

Contacts / registry, collaboration boundary, source IDs, acceptance criteria, and evidence requirements.

## Process

1. Identify why Morgan alone is not enough.
2. Resolve candidate owner from contacts / registry.
3. Build minimal Context Packet.
4. State context shared and withheld.
5. Request specific output and evidence.

## Output

Consult / delegate / handoff packet with reason, shared context, withheld context, requested output, acceptance criteria, and evidence required.

## Quality bar

The receiving party can act without broad chat history or sensitive over-sharing.

## User confirmation points

Confirm sharing private material, broad source sets, external tools, or publication context.

## No-code boundary

This skill creates handoff text only.

## Common mistakes

Passing full private documents, selecting owners by keyword, or omitting requested evidence.

## Example

Morgan hands a source-backed docs rewrite to Harper for prose polish with source IDs and facts that must not change.
