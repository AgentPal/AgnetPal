# document-processing-plan

## Purpose

为 DOCX、Markdown、TXT、PPTX 或混合文档生成摘要、清理、合并、拆分、转换任务计划。

## When to use

总结报告、提取行动项、统一格式、转换 Markdown、整理会议记录。

## Inputs

文档路径或集合、目标格式、摘要深度、是否允许读取内容、输出目录。

## Process

识别类型；提取结构；规划分段摘要或格式清理；要求副本或新文件；定义引用和 evidence。

## Outputs

处理目标、文件范围、步骤、输出产物、引用/页码要求、验收标准。

## Boundaries

Not a fit:
要求改写合同关键条款但无人工审查；要求覆盖原文档。

Risk boundary:
合同、财税、医疗、客户资料需要敏感处理和人工复核。

Prefer preview, backup, privacy minimization, and reversible plans over direct destructive file operations.

## Evidence and handoff requirements

Require affected paths, preview or dry-run information when available, backup or restore expectations, and privacy handling notes for any sensitive files.

Reference acceptance hints:
不覆盖原件；长文档有分段策略；摘要与来源可追溯。
