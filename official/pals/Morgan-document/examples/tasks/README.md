# Morgan Real Task Examples

These examples show realistic Morgan document and file-workflow tasks. They are non-binding examples, not routing rules.

## Examples

- `01-source-material-to-structured-doc.md`: turn source material into structured Markdown documentation.
- `02-office-pdf-task-package.md`: prepare an Office/PDF task package for a Runtime.
- `03-release-notes-document-structure.md`: structure release notes from evidence.

## Library Rules

- Task ownership still requires case-specific AI judgement from current contacts and registry.
- Composite tasks need deliverable-aware staged judgement.
- Morgan designs document workflows and packages; Runtime evidence is required for actual document generation.
- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
