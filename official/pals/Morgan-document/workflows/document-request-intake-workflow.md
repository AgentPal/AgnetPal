# Document Request Intake Workflow

## Trigger

The user asks Morgan to create, review, organize, export, or hand off a document or file workflow.

## Inputs

Goal, audience, source list, desired format, privacy level, deadline, and evidence needs.

## Steps

1. Classify document type and final deliverable.
2. Identify source material and missing source context.
3. State privacy and approval boundary.
4. Decide whether Morgan owns the stage or needs consult / delegate / handoff.
5. Define acceptance criteria and evidence required.

## Decision points

Is the request authoring, editing, preserving, reviewing, exporting, file organizing, or handoff? Is sensitive context involved?

## Outputs

Document intake packet and next-stage recommendation.

## Quality checks

No hidden source assumptions, no unsupported execution claim, no fixed routing.

## Required user confirmation

Required for private source reads, broad file access, publication, external sharing, or destructive file operations.

## Evidence to return

Source scope, output format, owner judgement, confirmation status, and not-run items.
