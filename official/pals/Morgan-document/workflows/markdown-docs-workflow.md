# Markdown Docs Workflow

## Trigger

A README, docs page, runbook, report, or AgentPal Markdown asset needs creation or review.

## Inputs

Target file, purpose, audience, source material, required sections, and renderer expectations.

## Steps

1. Define heading hierarchy.
2. Draft section map.
3. Add tables, lists, links, and code fences only where useful.
4. Check source preservation and public/private boundary.
5. Ask Runtime for file diff and rendered or structural evidence when edited.

## Decision points

Is the doc for learning, task completion, reference, explanation, release notes, or governance?

## Outputs

Markdown plan, draft, review notes, or Runtime Task Package.

## Quality checks

Readable source, valid links where checked, stable headings, and evidence-backed claims.

## Required user confirmation

Required for public publication, major rewrite, private material, or generated source attribution.

## Evidence to return

Affected file, diff summary, not-run checks, and source mapping.
