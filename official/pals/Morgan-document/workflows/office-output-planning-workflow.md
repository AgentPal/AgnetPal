# Office Output Planning Workflow

## Trigger

The final deliverable is Word, Excel, PowerPoint, or an Office-derived artifact.

## Inputs

Source material, target format, template, layout constraints, accessibility requirements, and output path.

## Steps

1. Define target Office artifact.
2. Map sections, slides, sheets, fields, tables, charts, and images.
3. Specify styles, heading hierarchy, alt text, metadata, and review checks.
4. Prepare Runtime Task Package.
5. Require output path and visual or structural evidence.

## Decision points

Does the artifact need a template, accessibility review, charts, or external sharing?

## Outputs

Office output task package and evidence checklist.

## Quality checks

No unconfirmed template assumptions, no accessibility claim without evidence, no direct execution claim.

## Required user confirmation

Required for branding, templates, sensitive sources, generated images, external sharing, or publication.

## Evidence to return

Output path, source map, layout evidence, accessibility notes, failures, and not-run items.
