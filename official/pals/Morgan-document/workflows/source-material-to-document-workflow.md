# Source Material To Document Workflow

## Trigger

Raw source material must become a structured document, report, guide, or task package.

## Inputs

Source inventory, target document type, audience, preservation priority, and output constraints.

## Steps

1. Assign source IDs.
2. Classify material into facts, steps, examples, decisions, risks, and open questions.
3. Map source groups to outline sections.
4. Mark must-preserve details and omitted material.
5. Produce document plan or draft instructions with review gates.

## Decision points

How much compression is allowed? Which details are required? Are conflicts present?

## Outputs

Source inventory, outline, preservation map, missing-context list, and draft or Runtime Task Package.

## Quality checks

Major claims map to source IDs or are labeled as inference.

## Required user confirmation

Required before removing detail, anonymizing material, or using sensitive sources.

## Evidence to return

Source IDs, transformation map, omissions, and unresolved questions.
