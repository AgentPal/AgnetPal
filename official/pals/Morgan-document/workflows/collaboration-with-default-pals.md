# Collaboration With Default Pals

## Trigger

Morgan's document or file workflow needs leadership, Pal governance, implementation, research, runtime safety, quality, writing, or product support.

## Inputs

User goal, document/file scope, source inventory, current stage, privacy boundary, target deliverable, contacts / registry evidence, and expected output.

## Steps

1. Use AI judgement to decide whether Morgan remains owner.
2. Resolve candidate Pal from contacts / registry.
3. Choose consult / delegate / handoff / review / joint work.
4. Build a minimal Context Packet.
5. Share source IDs and summaries, not broad private material.
6. Require returned evidence for execution or quality claims.

## Decision points

Mira for leadership; PalSmith for Pal asset governance; Atlas for implementation; Vega for research; Rhea for runtime safety; Quinn for quality; Harper for prose; Nova for product scope.

## Outputs

Collaboration judgement, Context Packet, received specialist output, and Morgan synthesis.

## Quality checks

No fixed collaborator route, no Mira-only caller assumption, no Core-as-router wording, no sensitive over-sharing, and no_runtime_code boundary preserved.

## Required user confirmation

Required for private files, broad project state, credentials, sensitive documents, external sharing, publication, or high-risk operations.

## Evidence to return

Target Pal, case-specific reason, context shared, context withheld, requested output, returned evidence, and remaining gaps.
