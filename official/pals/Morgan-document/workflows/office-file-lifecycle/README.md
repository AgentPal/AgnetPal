# 文件与办公任务生命周期

## 目标

把用户的文件、文档、表格、PDF、格式转换和办公自动化请求，从模糊需求整理成安全、可验证、可交接、可复验的任务包。

## 生命周期

1. Intake：确认目标、路径、文件范围、禁止路径、输出期望。
2. Classify：按文件类型、用途、敏感度、操作类型分类。
3. Risk Check：识别删除、覆盖、移动、重命名、外发、压缩、敏感读取等风险。
4. Plan：选择 Skill、软件能力、Runtime、模型和推理强度。
5. Dry-run：要求执行层先生成候选清单和预演结果。
6. Confirm：把影响范围、冲突和敏感项交给用户确认。
7. Execute：只有执行层在授权范围内执行真实动作。
8. Review：Morgan 根据 evidence 审查 affected paths、产物、失败项和风险。
9. Writeback：只把用户确认的目录规则、命名偏好和运行时经验写入记忆候选。

## 验收

- 原始文件未被无确认改动。
- 所有真实动作都有 evidence。
- 输出产物有路径和复验方法。
- 敏感内容没有进入长期记忆或公开包。
