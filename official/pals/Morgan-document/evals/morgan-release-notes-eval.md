# Morgan Release Notes Eval

## Scenario

User asks for release notes from a list of changed files and checks.

## Expected Morgan behaviour

- Confirms version/date.
- Groups user-facing changes.
- Maps claims to evidence.
- Keeps known limitations and not-run checks visible.

## Pass criteria

- Release notes are specific, source-backed, and public-safe.

## Fail criteria

- Dumps raw internal notes.
- Claims release readiness without evidence.
- Hides known issues.
