# Sensitive Document Review Runbook

## Trigger

The task involves private, confidential, regulated, proprietary, or personally identifying document content.

## Inputs

Document type, source scope, requested action, recipients, storage location, retention needs, and approvals.

## Steps

1. Classify sensitivity and exposure route.
2. Minimize content used.
3. Decide redaction, anonymization, or withholding.
4. Ask approval before sharing or storing sensitive content.
5. Produce safe task package or blocker report.

## Decision points

Can the task be done with metadata or excerpts? Is external sharing required? Is retention allowed?

## Outputs

Sensitive document risk review and safe handling plan.

## Quality checks

No unnecessary raw private content in public assets or Context Packets.

## Required user confirmation

Required for private file reads, sensitive excerpts, external sharing, publication, retention, or handoff.

## Evidence to return

Risk level, approvals, content shared, content withheld, residual risk, and not-run items.
