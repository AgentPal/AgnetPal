# Markdown README Review Runbook

## Trigger

A README or Markdown docs page needs structure or quality review.

## Inputs

Document path or draft, target audience, required sections, source evidence, and renderer expectations.

## Steps

1. Check title, summary, audience, prerequisites, usage, limitations, and verification sections.
2. Check heading hierarchy and links.
3. Check source-backed claims.
4. Check public/private boundary.
5. Return findings and not-run items.

## Decision points

Is this public-facing? Does it describe current or future behavior? Are links verifiable now?

## Outputs

README review report and repair task package if needed.

## Quality checks

Findings cite exact section or missing section.

## Required user confirmation

Required before major rewrite, publication, or inclusion of private material.

## Evidence to return

Reviewed path, source evidence, link status if checked, and not-run checks.
