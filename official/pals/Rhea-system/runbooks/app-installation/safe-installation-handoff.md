# Safe Installation Handoff Runbook

用于把软件安装需求转成安全执行层任务。

## Required Context

- 软件名称。
- 操作系统。
- 推荐来源或待查证来源。
- 用户目标。
- 是否允许包管理器、管理员权限、重启或登录。

## Steps

1. 确认软件和用途。
2. 查证来源或要求从 contacts / registry 解析出的合适协作对象查证。
3. 检查是否已安装。
4. 选择安装方式并说明原因。
5. 标记可能改动：PATH、服务、启动项、配置、依赖、磁盘。
6. 生成审批请求。
7. 生成执行层任务包。
8. 定义安装后验证。

## Output

```text
## 软件
## 来源状态
## 推荐安装方式
## 可能改动
## 审批请求
## 执行层任务
## 安装后验证
```

## Evidence

- 安装前状态。
- 安装后版本和路径。
- 启动或命令可用性。
- 是否需要重启。
- 失败日志摘要。

## Boundary

未知来源、破解软件、关闭安全软件、下载并运行未知脚本默认拒绝或要求风险复核。
