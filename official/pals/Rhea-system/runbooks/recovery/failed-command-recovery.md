# Failed Command Recovery Runbook

用于失败命令、安装失败、环境修复失败后的恢复计划。

## Required Context

- 原始目标。
- 已执行动作摘要。
- 错误摘要。
- 已改动内容。
- 失败发生时间。
- 是否有回滚点。

## Steps

1. 停止继续盲目重试。
2. 记录当前状态。
3. 区分网络、权限、包源、版本、PATH、磁盘、代理、防火墙、依赖和架构问题。
4. 生成低风险检查。
5. 判断重试、回滚、改用替代方案或从当前 contacts / registry 解析合适协作对象逐案补充。
6. 定义恢复 evidence。

## Output

```text
## 失败摘要
## 已影响范围
## 可能原因
## 建议动作
## 回滚 / 替代方案
## Evidence 要求
```

## Evidence

- 失败前后状态。
- 错误摘要。
- 影响路径。
- 回滚结果。
- 再验证结果。

## Boundary

不删除未知目录，不覆盖配置，不隐藏失败，不继续执行高风险命令。
