# command-plan-review

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：命令计划审查

## 用途

把执行层计划运行的命令或动作翻译成人能理解的影响、风险、审批点和证据要求。

## 适用场景

- Runtime 准备运行命令。
- 用户贴来脚本或命令想判断是否安全。
- 需要比较只读检测和写入修复。

## 不适合场景

- 要求 Rhea 生成危险命令并鼓励直接执行。
- 混淆、未知、不可解释的脚本。

## 输入信息

- 命令摘要或动作计划。
- 目标系统和工作目录。
- 预计影响路径。
- 来源和执行理由。

## 处理流程

1. 解释命令大概做什么。
2. 区分只读、写入、安装、删除、外发数据。
3. 标记风险和审批需求。
4. 建议更低风险替代方案。
5. 定义执行后 evidence。

## 输出格式

```text
## 这会做什么
## 会影响什么
## 风险等级
## 建议是否允许
## Evidence 要求
```

## 验收标准

- 用户能理解命令影响。
- 高风险动作不被轻描淡写。
- 未知来源脚本默认不建议执行。

## 风险边界

不直接运行、不代替审批、不输出一键破坏性命令。

## 与其他 Skill / Runbook 的关系

与 `permission-risk-review`、`approval-request-writer`、`system-evidence-review` 连用。
