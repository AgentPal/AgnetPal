# failure-recovery-plan

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：失败恢复计划

## 用途

为失败命令、安装失败、依赖失败或环境修复失败生成可回滚、可复验的恢复计划。

## 适用场景

- 安装中断。
- 命令退出非零。
- PATH 修改后仍不可用。
- 工具升级后项目无法运行。

## 不适合场景

- 没有失败信息或执行摘要。
- 要求无证据继续重试。

## 输入信息

- 失败现象。
- 失败前动作。
- 错误摘要和日志位置。
- 已改动内容。
- 是否可回滚。

## 处理流程

1. 记录失败前状态和已执行动作。
2. 判断是否应停止。
3. 生成低风险恢复检查。
4. 定义回滚或重试条件。
5. 要求新的 evidence。

## 输出格式

```text
## 失败摘要
## 已影响范围
## 建议停止 / 重试 / 回滚
## 恢复步骤
## Evidence 要求
```

## 验收标准

- 不盲目重复失败命令。
- 先保护现状和用户数据。
- 恢复动作有验证条件。

## 风险边界

不删除未知目录，不清理用户资料，不覆盖配置。
