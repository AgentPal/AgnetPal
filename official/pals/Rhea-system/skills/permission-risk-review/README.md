# permission-risk-review

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：权限与风险审查

## 用途

审查任务是否涉及管理员权限、sudo、系统目录、注册表、服务、启动项、安全软件、外发数据或敏感路径。

## 适用场景

- 安装、卸载、删除、修改环境变量。
- 命令要求管理员权限。
- 任务涉及服务、驱动、注册表、系统目录。

## 不适合场景

- 纯只读检测且无敏感数据。
- 用户要求绕过审批。

## 输入信息

- 计划执行动作。
- 影响路径和权限要求。
- 是否可回滚。
- 是否涉及凭据或用户数据。

## 处理流程

1. 识别权限级别。
2. 标记影响范围。
3. 判断是否必须审批。
4. 要求备份或回滚计划。
5. 输出允许、暂停或拒绝建议。

## 输出格式

```text
## 风险等级
## 影响范围
## 必须确认
## 回滚要求
## 建议
```

## 验收标准

- 高风险动作被明确标记。
- 不绕过用户审批。
- 有回滚或停止条件。

## 风险边界

不批准权限，不要求关闭安全机制，不指导破解或规避方法。

## 与其他 Skill / Runbook 的关系

常与 `command-plan-review` 和 `approval-request-writer` 配合。
