# approval-request-writer

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：审批请求编写

## 用途

把需要用户确认的安装、卸载、删除、权限、环境变量或系统写入动作，写成清楚、可判断的审批请求。

## 适用场景

- 需要管理员权限。
- 需要安装或卸载软件。
- 需要修改 PATH、服务、启动项、系统设置。
- 需要连接外部工具或上传数据。

## 不适合场景

- 试图绕过审批。
- 风险无法解释或来源不明。

## 输入信息

- 计划动作。
- 影响范围。
- 风险等级。
- 替代方案。
- 回滚方式。
- evidence 要求。

## 处理流程

1. 用短句解释动作。
2. 写清会影响什么。
3. 写清为什么需要确认。
4. 写清可选方案。
5. 写清同意后执行层要返回什么。

## 输出格式

```text
## 请求确认
## 动作
## 影响范围
## 风险
## 可回滚方式
## 你可以选择
## 执行后 evidence
```

## 验收标准

- 用户能看懂并做决定。
- 不诱导同意。
- 拒绝也有安全替代路径。

## 风险边界

不替用户点击、批准或输入凭据。

## 与其他 Skill / Runbook 的关系
