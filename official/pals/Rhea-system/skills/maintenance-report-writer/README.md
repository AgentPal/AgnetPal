# maintenance-report-writer

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：维护报告编写

## 用途

把系统维护、安装配置、故障排查和执行层结果整理成用户可读的维护报告。

## 适用场景

- 一次系统任务结束。
- 需要记录修复路径、风险和后续建议。
- 需要从当前 contacts / registry 解析合适协作对象逐案补充 或用户继续处理。

## 不适合场景

- 任务还没有 evidence。
- 高风险操作未确认。

## 输入信息

- 任务目标。
- 执行层摘要。
- evidence 审查结论。
- 未完成项、风险和后续建议。

## 处理流程

1. 概括任务和结果。
2. 列出证据。
3. 标注未完成项和风险。
4. 给出是否建议写入记忆。
5. 给出下一步或协作 handoff。

## 输出格式

```text
## 维护结论
## 做了什么
## Evidence
## 未完成项
## 风险
## 记忆候选
## 下一步
```

## 验收标准

- 用户能看懂当前状态。
- 证据和结论一一对应。
- 不夸大完成程度。

## 风险边界

不包含密钥、私人日志全文或敏感路径细节。

## 与其他 Skill / Runbook 的关系

通常在 `system-evidence-review` 之后使用。
