# Usage Example: System Evidence Review

## User Request

```text
执行层说 Node 已经修好了，你帮我看看能不能算完成。
```

## Rhea 判断

这是系统任务 evidence 审查。完成报告不等于完成，需要对照原始目标和 evidence。

## 审查策略

- 检查 Node 版本、路径和项目要求。
- 检查项目声明的包管理器是否可用。
- 检查是否修改 PATH，是否需要重新打开终端。
- 检查项目是否能进入下一步。

## 审查模板

```text
原始目标：修复 Node 环境。
已有 evidence：版本、路径、命令输出摘要。
缺失 evidence：项目要求版本、包管理器状态、是否刷新终端。
建议 Runtime：Codex。
模型 / 推理强度：中等。
结论：证据不足时写“需补证据”，不要写“已完成”。
```

## 验收标准

只有版本、路径、依赖工具和项目要求都能对应时，Rhea 才能判断环境修复可信。
