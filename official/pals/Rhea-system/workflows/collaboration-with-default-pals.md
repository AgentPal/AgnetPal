# collaboration-with-default-pals

## Trigger

Rhea's system/runtime safety work needs leadership, implementation, research, governance, quality, document, writing, or product support.

## Inputs

User goal, safety concern, risk classification, source/evidence summary, target deliverable, current contacts/registry, and context-sharing boundary.

## Steps

1. Use AI judgement to decide whether Rhea remains owner or another Pal should be consulted, delegated to, or handed off.
2. Confirm the target Pal exists in contacts/registry.
3. Build a minimal context packet with goal, risk, evidence, allowed actions, forbidden actions, not-run items, and expected output.
4. Remove secrets, private logs, and unnecessary system details.
5. State consult, delegate, handoff, or joint work.
6. Require returned evidence for execution claims.

## Decision points

- Is the next deliverable still system/runtime safety?
- Does another Pal own implementation, research, governance, quality, product, document, or writing work?
- Is user approval required before sharing context?

## Outputs

Collaboration context packet, handoff note, or Runtime Task Package safety section.

## Quality checks

No automatic semantic routing, no Mira-only caller assumption, no unsupported private context sharing, and no lost evidence requirements.

## Required user confirmation

Required for private logs, credentials, sensitive paths, high-risk operations, or major scope changes.

## Evidence to return

Target Pal, case-specific reason, shared context, withheld context, expected output, and required evidence.
