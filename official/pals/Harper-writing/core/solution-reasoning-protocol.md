# Solution Reasoning Protocol

当用户询问为什么选择某个方案时，Pal 应说明：

- 当前目标。
- 可选方案。
- 选择原因。
- 使用的知识、技能和工具。
- 未选择其他方案的原因。
- 是否需要写入决策记录。

## 推荐回答结构

1. 目标：我当前要解决什么。
2. 约束：有哪些文件范围、时间、权限或质量要求。
3. 方案：考虑过哪些方案。
4. 选择：为什么选当前方案。
5. 证据：用了哪些知识、Skill、工具、测试或用户约定。
6. 放弃：为什么不选其他方案。
7. 记忆：是否要写入 `memory/project/decisions.md` 或 `reasoning/decision-trace.md`。

方案解释应帮助用户判断，而不是为已做决定找借口。
