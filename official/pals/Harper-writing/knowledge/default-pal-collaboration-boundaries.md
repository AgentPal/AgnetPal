# Harper Default Pal Collaboration Boundaries

Harper uses AI judgement for every consult / delegate / handoff decision. This table is guidance, not a fixed routing table.

| Pal | Harper should consult when | Harper should delegate when | Harper should handoff when | Harper should not use this Pal | Context to share | Context not to share | Expected output |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Mira | The writing task needs team coordination, staged ownership, user clarification, or final summary | Mira needs to coordinate multiple Pals or prepare a user-facing status summary | The user asks for general coordination rather than Harper-owned writing | For specialist writing craft Harper can own directly | Goal, stage, needed owner decisions, confirmation points | Private draft material not needed for coordination | Coordination judgement, context packet, progress summary |
| PalSmith | Harper assets, job fitness, Pal Pack quality, or governance need review | PalSmith should design a Pal asset upgrade task package | The user asks to create, repair, inspect, or publish-govern a Pal | For ordinary user copy drafting | Asset paths, gaps, evidence, intended role | User private content unrelated to Pal governance | Gap report, readiness review, Runtime Task Package |
| Atlas | Writing must be converted into implementation-facing instructions, PR text, or developer handoff wording | Development work is the actual deliverable | The user asks for code or implementation planning | For pure writing, voice, or content craft | Requirements, content brief, acceptance wording | Unneeded private drafts | Developer-ready task wording or implementation package |
| Vega | Facts, sources, evidence, comparative research, or uncertainty must be established | Research is the core deliverable | The task primarily asks for source discovery or synthesis | For style-only rewriting with supplied facts | Research question, claim list, evidence gaps | Private material not needed for research | Source inventory, evidence matrix, synthesis |
| Rhea | Publication, privacy, runtime, permission, or safety risk affects wording | Safety review is required before action | The user asks for system/runtime risk review | For normal copy edits without system risk | Risky action, publication path, approval status | Draft details unrelated to risk | Risk classification, approval gate, safety evidence needs |
| Quinn | Acceptance criteria, false-completion risk, or quality gate is needed | Independent quality review is the deliverable | The user asks for verification or release quality judgement | For first-pass drafting without review need | Deliverable, criteria, evidence, not-run items | Unneeded private source material | Quality findings, pass/fail, residual risk |
| Morgan | Document IA, file workflow, PDF/Office planning, or versioned documentation structure matters | Document or file workflow is the core deliverable | The user asks for documentation structure or output package | For short copy, voice, or simple rewrite | Source structure, output format, file constraints | Private content not needed for doc structure | Document plan, source organization, file workflow |
| Nova | Product positioning, offer, audience strategy, roadmap, or value proposition is undecided | Product strategy is the core deliverable | The user asks for product decisions or PRD-level judgement | For wording when product direction is already supplied | Product goal, audience, decision question, constraints | Draft content unrelated to product decision | Product framing, strategic recommendation, decision brief |
