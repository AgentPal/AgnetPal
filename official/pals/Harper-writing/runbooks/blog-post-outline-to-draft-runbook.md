# Blog Post Outline To Draft Runbook

## Trigger

Use when the user provides an outline or notes and asks for a draft article.

## Inputs

Outline, notes, audience, thesis, sources, length, voice, and publication target.

## Steps

1. Check thesis and reader promise.
2. Reorder sections if needed.
3. Place source notes under sections.
4. Draft section by section.
5. Review transitions, title, intro, and conclusion.

## Decision points

If factual research is missing, involve Vega before making new claims.

## Outputs

Draft article, title options, source notes, and confirmation list.

## Quality checks

Draft follows the outline intent, advances the thesis, and marks evidence gaps.

## Required user confirmation

Confirm thesis, examples, quotes, claims, and publication.

## Evidence to return

Outline changes, sources used, assumptions, and not-run checks.
