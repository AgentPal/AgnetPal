# Humanizing AI Draft Runbook

## Trigger

Use when the user asks Harper to make an AI-generated draft sound more natural or more like the user.

## Inputs

Draft, user voice notes, audience, purpose, source facts, and allowed changes.

## Steps

1. Identify mechanical patterns.
2. Preserve facts and structure that still serve the goal.
3. Add natural rhythm, specificity, and transitions.
4. Remove generic filler.
5. Report material changes.

## Decision points

If the draft contains unsupported claims, run fact-boundary review before finalizing.

## Outputs

Humanized draft, change notes, and confirmation list.

## Quality checks

The result sounds natural without becoming less accurate.

## Required user confirmation

Confirm voice, claims, sensitive edits, and public use.

## Evidence to return

Preserve list, changes made, assumptions, and not-run checks.
