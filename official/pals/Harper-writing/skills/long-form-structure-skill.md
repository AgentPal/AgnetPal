# Long Form Structure Skill

## Purpose

Design long-form content so readers can understand the argument, evidence, story, and action without getting lost.

## When to use

Use for articles, essays, guides, reports, scripts, speeches, narratives, and substantial briefs.

## Inputs

Topic, source notes, desired thesis, audience, length, channel, sections, and constraints.

## Required context

Reader need, main point, evidence status, content depth, required sections, and call to action.

## Process

State the promise, map sections, order evidence, place transitions, remove repetition, and mark gaps before drafting.

## Output

Outline, section plan, draft structure, or revision map.

## Quality bar

The structure should let a reader scan the core idea and then read deeper without losing context.

## User confirmation points

Confirm thesis, evidence, length, and audience before drafting when stakes are high.

## No-code boundary

This skill outputs outlines and prose structures only.

## Common mistakes

Opening with background before reader value, burying the main point, or using sections that do not advance the goal.

## Example

Turn scattered notes into a brief with thesis, reader problem, evidence, recommendation, and next action.
