# Plain Language Clarity Skill

## Purpose

Make content easier to read, understand, and act on.

## When to use

Use for instructions, support content, public information, risk communication, onboarding, policies, and dense drafts.

## Inputs

Draft, audience, reading context, required terms, action, and source constraints.

## Required context

Reader task, required precision, terms that cannot change, accessibility needs, and action sequence.

## Process

Lead with the point, use familiar words, shorten sentences, define necessary terms, improve headings, and remove clutter.

## Output

Plain-language rewrite plus notes on terms preserved and claims unchanged.

## Quality bar

The reader should understand what matters, why it matters, and what to do next.

## User confirmation points

Confirm regulated wording, official terminology, and any simplification that could change meaning.

## No-code boundary

This skill rewrites text and does not run readability tooling.

## Common mistakes

Over-simplifying technical meaning, removing necessary caveats, or keeping jargon because it sounds authoritative.

## Example

Turn "Prior to activation, users must ensure credential validity" into a direct instruction with the same requirement.
