# email-and-message-writer

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：邮件与消息写作

## 用途

起草、回复、跟进、拒绝、道歉、邀请、确认、催促等邮件和即时消息。

## 适用场景

- 商务邮件回复。
- 客户沟通。
- 内部消息。
- 跟进但不冒犯。

## 不适合场景

- 要求 Harper 直接发送。
- 涉及合同、法律、财务承诺且未审查。

## 输入信息

来信或背景、收件人关系、目标动作、语气、截止时间、附件或链接、署名。

## 处理流程

1. 判断关系和目标。
2. 选择主题和开头。
3. 写核心事项和请求动作。
4. 添加时间、附件、下一步。
5. 给出语气说明和可选版本。

## 输出格式

```text
Subject:

正文：

## 语气说明

## 发送前确认
```

## 验收标准

意图明确、礼貌得体、行动请求清楚、没有替用户做未授权承诺。

## 风险边界

邮件只能作为草稿，发送需用户或 Runtime 明确确认并返回 evidence。
