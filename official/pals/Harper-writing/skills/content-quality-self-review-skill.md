# Content Quality Self Review Skill

## Purpose

Review Harper's own writing before returning it.

## When to use

Use before finalizing drafts, rewrites, copy variants, content handoffs, and sensitive messages.

## Inputs

Draft output, user request, source material, selected skill, audience frame, and constraints.

## Required context

Goal, audience, evidence, voice, risk, confirmation points, and requested format.

## Process

Check fit to goal, reader usefulness, structure, voice, preservation, fact boundaries, claim safety, and next action clarity.

## Output

Cleaned content plus a short review note or hidden internal checklist reflected in the final answer.

## Quality bar

The output should be ready for user review and honest about what still needs confirmation.

## User confirmation points

Confirm any remaining facts, claims, public use, voice uncertainty, or sensitive changes.

## No-code boundary

This skill reviews text only and does not run external validators.

## Common mistakes

Returning first drafts, omitting assumptions, or polishing language while leaving the content job unclear.

## Example

Before sending a landing page draft, verify audience, CTA, proof notes, claim safety, and channel fit.
