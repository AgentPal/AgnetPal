# Writing Intake Skill

## Purpose

Classify the writing job and gather only the context needed to produce or review useful content.

## When to use

Use for any new Harper task before drafting, rewriting, editing, structuring, adapting, or handing off.

## Inputs

User request, source material, target channel, audience, goal, constraints, deadline, voice notes, and evidence availability.

## Required context

Audience, intended action, content type, length, source status, risk level, and whether user approval is needed before external use.

## Process

Identify whether the task is create, revise, structure, adapt, review, or handoff. Separate facts from preferences. Note missing evidence and confirmation points. Choose the smallest relevant Harper skill or workflow.

## Output

A compact writing brief with task type, audience, goal, channel, voice, evidence status, risks, next skill, and expected deliverable.

## Quality bar

The brief should be short enough to act on and complete enough to prevent avoidable wrong turns.

## User confirmation points

Confirm sensitive audience, public use, unsupported claims, major voice shifts, commitments, and required facts.

## No-code boundary

This skill creates briefs and Markdown task packages only.

## Common mistakes

Starting a draft before knowing the reader, treating assumptions as facts, or asking a long questionnaire when two focused questions would work.

## Example

Input: "Make this announcement warmer." Output: "Task: revise. Audience: existing users. Goal: reassure and explain. Need confirmation on whether the date is final."
