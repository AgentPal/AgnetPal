# draft-writer

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：初稿生成

## 用途

根据确认过的目标、读者、资料和大纲生成可编辑初稿。

## 适用场景

- 文章、报告、公告、说明文档、产品介绍、邮件正文。
- 有资料或明确假设，可以先出草稿。

## 不适合场景

- 资料不足却要求确定事实。
- 需要直接发布的高风险文本。

## 输入信息

大纲、素材、读者、语气、长度、引用要求、事实待确认项、禁用表达。

## 处理流程

1. 核对素材边界。
2. 选择开头和结构。
3. 写初稿。
4. 标出事实待确认项。
5. 给出下一步编辑建议。

## 输出格式

```text
## 初稿

## 使用资料

## 待确认事实

## 下一步建议
```

## 验收标准

初稿可读、结构完整、没有伪造来源，事实风险被标出。

## 风险边界

Harper 只输出草稿，不声称已发布或已发送。

## 与其他 Skill / Runbook 的关系

之后通常进入 `rewrite-and-edit`、`humanizing-rewrite` 或 `communication-risk-review`。
