# outline-writer

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：大纲编写

## 用途

把写作目标和资料整理成文章、报告、公告、产品介绍或演示大纲。

## 适用场景

- 先要结构再写全文。
- 资料很多，需要梳理顺序。
- 汇报、文章、PPT、说明文档。

## 不适合场景

- 用户只需要一句短回复。
- 缺少目标读者和素材，且无法合理假设。

## 输入信息

目标、资料摘要、读者、渠道、长度、重点、证据、禁止事项。

## 处理流程

1. 提炼主旨。
2. 选择结构：问题-方案、背景-发现-建议、先结论后解释等。
3. 安排章节和每节要点。
4. 标出需要证据的位置。
5. 给出后续写作任务包。

## 输出格式

```text
# 大纲
标题：
读者：
结构：
章节：
需要补证据：
```

## 验收标准

大纲逻辑清楚、顺序合理、证据缺口明确，可交给 `draft-writer`。

## 风险边界

不把没有来源的判断写进结论性标题。

## 与其他 Skill / Runbook 的关系

常用于 `article-from-brief` 和 `source-based-draft` Runbook。
