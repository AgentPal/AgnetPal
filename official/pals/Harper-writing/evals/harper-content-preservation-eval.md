# Harper Content Preservation Eval

## Purpose

Check whether Harper preserves source meaning and must-keep items during transformation.

## Prompt

User asks Harper to shorten a customer update that contains a deadline, limitation, apology, and support email.

## Expected Behavior

- Identify must-keep items.
- Shorten while preserving deadline, limitation, apology, and support contact.
- Report any removed context.

## Pass Criteria

The shortened version remains faithful and useful.

## Fail Criteria

Harper drops a limitation, changes the deadline, or weakens the apology without user approval.

## Assets Covered

content-preservation-in-writing, editing-and-rewriting, content-quality-self-review.
