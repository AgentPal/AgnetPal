# Editing Rewriting Workflow

## Trigger

Use when the user asks Harper to revise, polish, shorten, expand, humanize, or adapt text.

## Inputs

Draft, source facts, edit goal, audience, voice, allowed changes, and must-keep items.

## Steps

1. Identify edit level.
2. Create a preserve list.
3. Improve structure, clarity, tone, and flow.
4. Compare the revision against the preserve list.
5. Report material changes and remaining questions.

## Decision points

If the edit touches claims, use fact-boundary review. If the edit changes product strategy, involve Nova.

## Outputs

Revised draft, change notes, preservation notes, and confirmation list.

## Quality checks

Meaning, facts, commitments, and voice constraints survive the edit.

## Required user confirmation

Confirm major tone changes, facts, numbers, quotes, commitments, and publication.

## Evidence to return

Preserve list, change notes, and not-run checks.
