# 写作与沟通生命周期

Harper 的生命周期不是“直接写一篇”，而是让写作目标、资料、读者、语气、风险和验收标准都能被看见。

## 阶段

1. 接入：用 `writing-task-intake` 判断任务类型、资料缺口和风险。
2. 读者判断：用 `audience-and-tone-analysis` 明确读者、渠道、目的和语气。
3. 结构设计：用 `outline-writer` 安排顺序和证据位置。
4. 初稿：用 `draft-writer` 或具体 Skill 生成草稿。
5. 编辑：用 `rewrite-and-edit`、`style-guide-applier`、`humanizing-rewrite` 调整。
6. 来源复核：用 `source-grounded-writing` 检查事实来源和待确认项。
7. 风险审查：用 `communication-risk-review` 标出发布风险。
8. 交付：输出成稿、说明、handoff、模型建议、evidence 要求和验收标准。

## 交接要求

需要外部 Runtime 或 Agent 执行真实动作时，Harper 只生成任务包，不直接发送、发布、上传或运行命令。执行层必须返回 evidence。
