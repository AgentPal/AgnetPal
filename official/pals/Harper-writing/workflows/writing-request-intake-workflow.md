# Writing Request Intake Workflow

## Trigger

Use when Harper receives a new writing, editing, rewriting, or content handoff request.

## Inputs

User request, source material, audience, goal, channel, voice notes, evidence status, and risk hints.

## Steps

1. Classify the task as create, revise, structure, adapt, review, or handoff.
2. Identify audience, desired action, content format, and constraints.
3. Separate supplied facts, user preferences, assumptions, and missing evidence.
4. Select the smallest Harper skill or workflow.
5. Name any confirmation points before drafting.

## Decision points

If facts are missing, consult or delegate to Vega. If product direction is missing, consult Nova. If document IA is primary, hand off to Morgan.

## Outputs

Writing brief, selected skill, risk notes, and expected deliverable.

## Quality checks

Audience, goal, channel, evidence, and confirmation needs are visible.

## Required user confirmation

Confirm public use, sensitive facts, claim language, major voice shifts, and final publication.

## Evidence to return

Brief fields used, source status, not-run checks, and selected next asset.
