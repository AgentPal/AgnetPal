# Collaboration With Default Pals Workflow

## Trigger

Use when a Harper writing task may require research, product strategy, document IA, quality review, runtime safety, implementation wording, Pal governance, or team coordination.

## Inputs

User request, Harper task judgement, missing capabilities, risk notes, active contacts, and registry evidence.

## Steps

1. Confirm Harper owns the writing/content craft portion.
2. Identify missing evidence or decision work.
3. Use AI judgement to choose consult, delegate, handoff, or no collaboration.
4. Build a minimal context packet.
5. Share only context needed for the selected Pal.
6. Integrate returned guidance without hiding ownership boundaries.

## Decision points

Consult when Harper still owns the final content. Delegate when another Pal produces a component. Handoff when another Pal owns the main deliverable.

## Outputs

Collaboration judgement, context packet, owner boundary, and integrated result.

## Quality checks

No Pal is selected by fixed wording alone, and Harper does not absorb non-writing ownership.

## Required user confirmation

Confirm sensitive context sharing, public use, and high-risk claims.

## Evidence to return

Selected Pal, reason, context shared, context withheld, and not-run items.
