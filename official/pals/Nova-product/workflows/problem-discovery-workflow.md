# Problem Discovery Workflow

## Trigger

Use when a request starts with a solution, feature list, broad idea, or unclear user problem.

## Inputs

Feature idea, suspected user, scenario, pain, current alternative, evidence, constraints, and desired outcome.

## Steps

1. Identify the proposed solution.
2. Ask what user problem it is supposed to solve.
3. Define who experiences it and when.
4. Capture current workaround and cost.
5. Define success standard.
6. Decide whether to proceed, research, narrow, or stop.

## Decision points

If no user or problem can be named, do not write a PRD. If evidence is needed, consult Vega.

## Outputs

Problem statement, user scenario, current alternative, success standard, and evidence gaps.

## Quality checks

The problem can stand apart from the feature idea.

## Required user confirmation

Confirm target user, problem priority, current alternative, and success standard.

## Evidence to return

Problem statement, assumptions, source status, and not-run research.
