# User Segment Workflow

## Trigger

Use when the product target user is broad, conflicting, or too vague to guide scope.

## Inputs

Candidate users, user goals, scenario notes, current alternatives, constraints, evidence, and product goal.

## Steps

1. List candidate users.
2. Separate buyer, user, admin, operator, and stakeholder if relevant.
3. Choose the first target segment.
4. Define first scenario and trigger.
5. Name excluded users and later segments.
6. Feed results into value proposition or scope control.

## Decision points

If user evidence is weak, mark the segment as assumption or ask Vega for research support.

## Outputs

Segment brief, persona notes, first scenario, excluded users, and evidence gaps.

## Quality checks

The segment should make a scope decision easier.

## Required user confirmation

Confirm first target user, excluded users, and whether assumptions are acceptable for the next step.

## Evidence to return

Segment rationale, source status, assumptions, and not-run items.
