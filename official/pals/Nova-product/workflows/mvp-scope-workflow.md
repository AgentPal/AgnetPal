# MVP Scope Workflow

## Trigger

Use when the user asks for MVP, V1, pilot, beta, first release, or scope cut.

## Inputs

Problem, first user, value proposition, candidate features, constraints, success metric, risks, and timeline.

## Steps

1. Define what the release must prove.
2. Identify the shortest core user path.
3. Sort features into must-have, later, explicit non-scope, and icebox.
4. Define success metric and feedback loop.
5. Mark risks and owner handoff.

## Decision points

If the release cannot prove value, narrow or return to problem framing.

## Outputs

MVP/release scope, non-goals, success metric, risk notes, and next owner.

## Quality checks

The release is small, testable, and meaningful.

## Required user confirmation

Confirm scope cuts, launch/pilot boundary, success metric, and public use.

## Evidence to return

Scope table, non-goals, assumptions, and not-run checks.
