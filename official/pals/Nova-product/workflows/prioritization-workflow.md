# Prioritization Workflow

## Trigger

Use when features, requirements, roadmap items, or stakeholder requests compete.

## Inputs

Candidate items, product goal, user value, reach, impact, confidence, effort, risk, dependencies, and release constraints.

## Steps

1. Define the prioritization decision.
2. Choose a fitting method: RICE, MoSCoW, Kano-style reasoning, or trade-off table.
3. Score or classify items.
4. Mark confidence and assumptions.
5. Recommend must-have, should-have, defer, or reject.
6. Ask for user decision on contested trade-offs.

## Decision points

If effort is unknown, consult Atlas. If value evidence is unknown, consult Vega or mark confidence low.

## Outputs

Prioritization table, rationale, trade-offs, and decision points.

## Quality checks

The table explains why the order exists and where estimates are weak.

## Required user confirmation

Confirm scoring assumptions, final priority order, scope cuts, and release commitment.

## Evidence to return

Method used, assumptions, confidence, and not-run checks.
