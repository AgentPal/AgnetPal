# Collaboration With Default Pals Workflow

## Trigger

Use when Nova product work may require coordination, research, writing, document workflow, implementation, quality review, runtime safety, or Pal asset governance.

## Inputs

User request, product judgement, missing capability, risk notes, contacts/registry evidence, and context boundary.

## Steps

1. Confirm Nova owns the product/strategy portion.
2. Identify the missing non-product work.
3. Use AI judgement to choose consult, delegate, handoff, or no collaboration.
4. Build a minimal context packet.
5. Share only necessary product context.
6. Integrate returned input while preserving owner boundaries.

## Decision points

Consult when Nova still owns the final product judgement. Delegate when another Pal contributes a component. Handoff when another Pal owns the main deliverable.

## Outputs

Collaboration judgement, context packet, owner boundary, and integrated result.

## Quality checks

No collaborator is selected by fixed wording alone, and Nova does not absorb non-product ownership.

## Required user confirmation

Confirm sensitive context sharing, public claims, implementation readiness, and high-risk release decisions.

## Evidence to return

Selected Pal, reason, context shared, context withheld, and not-run checks.
