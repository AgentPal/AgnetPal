# Product Discovery Lifecycle

This workflow is Nova's default path from fuzzy idea to execution-ready product handoff.

## Lifecycle

1. Intake: classify the request and maturity with `idea-intake`.
2. Problem: define user, scenario, current alternative, pain, and problem statement.
3. Scenario: map the first user and first real use case.
4. Scope: define V1, non-scope, later candidates, and icebox.
5. Version path: choose validation, lightweight complete working version, platform version, or long-term productized version.
6. Prioritize: rank features and cut anything that does not serve the current proof.
7. PRD: write a structured product document.
8. Acceptance: create observable pass/fail criteria.
9. Risk review: inspect assumptions, overbuild, privacy, compliance, and evidence gaps.
10. Handoff: prepare suitable implementation collaborator or runtime package.
11. Evidence review: check returned evidence before claiming completion.
12. Decision memory: record reusable, non-sensitive decisions when approved.

## Required Outputs

- Product judgment.
- Problem definition.
- Scenario map.
- Scope and non-scope.
- PRD or lighter task package.
- Acceptance criteria.
- Risk / assumption review.
- Handoff package when execution is needed.

## Stop Conditions

Stop and ask for clarification when the target user, core scenario, scope, or acceptance criteria are missing. Stop and request specialist review for regulated, privacy-heavy, payment, or public-commitment decisions.
