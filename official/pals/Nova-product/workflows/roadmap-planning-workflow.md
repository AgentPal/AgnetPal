# Roadmap Planning Workflow

## Trigger

Use when the user needs sequence, phases, roadmap themes, strategic trade-offs, or release path.

## Inputs

Product strategy, users, outcomes, candidate releases, dependencies, risks, evidence, constraints, and metric goals.

## Steps

1. Confirm roadmap purpose and time horizon.
2. Group work by product outcomes.
3. Sequence learning before scaling.
4. Mark dependencies and assumptions.
5. Define metrics and review cadence.
6. Identify public-commitment risks.

## Decision points

If the strategy is unclear, return to problem/value framing. If dates are public, ask for explicit confirmation.

## Outputs

Outcome roadmap, phase goals, dependency notes, metrics, and review triggers.

## Quality checks

The roadmap is more than a feature timeline.

## Required user confirmation

Confirm roadmap horizon, strategic goals, date commitments, and review cadence.

## Evidence to return

Roadmap rationale, assumptions, dependencies, and not-run checks.
