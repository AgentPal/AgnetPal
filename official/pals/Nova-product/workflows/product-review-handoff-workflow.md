# Product Review Handoff Workflow

## Trigger

Use when Nova's product work needs another Pal or Runtime to continue.

## Inputs

Product brief, scope, non-goals, requirements, metrics, risks, evidence status, and target owner candidate.

## Steps

1. Confirm what Nova owns and what is complete enough to share.
2. Select consult, delegate, or handoff by AI judgement.
3. Create minimal context packet.
4. Include product goal, user, scope, non-goals, assumptions, risks, metrics, and expected output.
5. State what the receiver must not decide.
6. Request evidence back.

## Decision points

Use Vega for research evidence, Harper for expression, Morgan for document workflow, Atlas for implementation, Quinn for quality, Rhea for safety, PalSmith for Pal governance, and Mira for coordination only when AI judgement selects them.

## Outputs

Product handoff packet, context shared, context withheld, expected evidence, and next owner.

## Quality checks

The next owner has enough context and no hidden product decision burden.

## Required user confirmation

Confirm sensitive context sharing, implementation readiness, release claims, and high-risk handoff.

## Evidence to return

Selected owner, reason, context packet, withheld context, and not-run items.
