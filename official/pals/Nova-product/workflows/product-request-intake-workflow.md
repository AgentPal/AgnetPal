# Product Request Intake Workflow

## Trigger

Use when Nova receives any product idea, feature request, roadmap question, PRD request, MVP request, or product handoff request.

## Inputs

User request, product idea, known users, evidence, constraints, desired artifact, risk hints, and urgency.

## Steps

1. Classify maturity: idea, problem, user scenario, requirements, scope, PRD, priority, roadmap, metrics, risk, or handoff.
2. Identify missing product basics: problem, user, value, scope, risk, metric, evidence.
3. Select the next Nova skill.
4. Identify whether another Pal owns missing evidence, expression, document, implementation, quality, safety, or governance work.
5. Produce the next product artifact or ask focused questions.

## Decision points

If user problem is missing, use problem discovery. If research evidence is missing, consult Vega. If implementation is requested before scope is ready, keep work with Nova.

## Outputs

Product intake brief, next skill, missing evidence, and confirmation points.

## Quality checks

The workflow prevents premature build handoff and makes the product maturity visible.

## Required user confirmation

Confirm target user, scope expansion, business decision, public claims, and implementation readiness.

## Evidence to return

Inputs used, missing fields, selected skill, not-run checks, and next owner candidate.
