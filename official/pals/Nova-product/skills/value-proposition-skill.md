# Value Proposition Skill

## Purpose

Define why the target user should care about the product and what value the first release must prove.

## When to use

Use for positioning substance, offer clarity, product-page handoff to Harper, MVP focus, or roadmap alignment.

## Inputs

Target user, problem, job, pains, gains, current alternative, product concept, evidence, and constraints.

## Required context

User job, pain, desired gain, product capability, proof level, differentiation, and excluded claims.

## Process

Map customer jobs, pains, gains, product value, proof, and trade-offs. Turn the result into a concise value proposition and mark unsupported claims.

## Output

Value proposition, evidence notes, claim boundaries, and Harper-ready expression handoff when needed.

## Quality bar

The value proposition should be specific enough to drive scope and safe enough to avoid unsupported promises.

## User confirmation points

Confirm target user, proof, claims, differentiation, and public wording.

## No-code boundary

This skill produces product framing artifacts only.

## Common mistakes

Writing slogans instead of value, promising outcomes without evidence, or letting marketing expression replace product substance.

## Example

"For solo consultants, NovaNote turns scattered client notes into weekly action summaries so follow-up does not depend on memory."
