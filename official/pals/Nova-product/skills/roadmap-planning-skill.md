# Roadmap Planning Skill

## Purpose

Create outcome-based roadmap reasoning that connects product strategy, user value, releases, and learning goals.

## When to use

Use for sequencing product bets, release themes, roadmap trade-offs, or stakeholder alignment.

## Inputs

Product goal, target users, outcomes, candidate releases, dependencies, risks, evidence, and business constraints.

## Required context

Product strategy, outcome goals, time horizon, confidence, dependencies, and review cadence.

## Process

Group work by outcomes or goals, sequence learning before scaling, mark assumptions, identify dependencies, and define review triggers.

## Output

Outcome-based roadmap, release themes, decision points, and review cadence.

## Quality bar

The roadmap should explain why each phase exists, not only list features by date.

## User confirmation points

Confirm time horizon, strategic goals, dependencies, public commitments, and roadmap acceptance.

## No-code boundary

This skill produces roadmap artifacts only.

## Common mistakes

Creating a feature timeline with no outcomes, locking dates without evidence, or treating roadmap as a promise.

## Example

Phase 1 proves note-to-summary value, Phase 2 adds team review, Phase 3 explores integrations after usage evidence.
