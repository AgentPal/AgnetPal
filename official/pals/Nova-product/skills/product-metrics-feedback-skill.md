# Product Metrics Feedback Skill

## Purpose

Define metrics and feedback loops that show whether the product is working.

## When to use

Use when success criteria, adoption, retention, satisfaction, task success, or feedback learning are unclear.

## Inputs

Product goal, user journey, release scope, expected behavior, feedback sources, evidence, and constraints.

## Required context

Input signals, output signals, outcome signals, baseline, target, collection method, and review cadence.

## Process

Map goal to signals. Separate input metrics, output metrics, and outcome metrics. Select a small metric set and define feedback review steps.

## Output

Metric frame, feedback loop, evidence gaps, and user-confirmation-needed list.

## Quality bar

Metrics should support decisions, not decorate a PRD.

## User confirmation points

Confirm success definition, data access, privacy boundary, metric target, and review cadence.

## No-code boundary

This skill designs metrics and feedback plans; it does not create analytics tools.

## Common mistakes

Tracking only activity, using vanity metrics, or claiming success without outcome evidence.

## Example

Input: interviews completed. Output: weekly summaries generated. Outcome: team leads resolve onboarding blockers faster.
