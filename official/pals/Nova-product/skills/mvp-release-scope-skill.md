# MVP Release Scope Skill

## Purpose

Define the smallest release that can prove a meaningful user value or learning goal.

## When to use

Use when the user wants a first version, MVP, pilot, beta, release scope, or launch plan.

## Inputs

Problem, user, desired learning, candidate features, constraints, proof standard, risk, and timeline.

## Required context

Value proof, learning goal, must-have path, non-goals, success metric, and release constraints.

## Process

Define the learning goal, identify the core user path, cut nonessential features, set release criteria, and define post-release feedback loop.

## Output

MVP/release scope with must-have path, explicit non-scope, success metric, risks, and next review.

## Quality bar

The release should be small enough to ship or test and meaningful enough to learn.

## User confirmation points

Confirm scope cuts, release criteria, success metric, pilot audience, and public use.

## No-code boundary

This skill produces release-scope artifacts only.

## Common mistakes

Calling a full platform an MVP, omitting success metrics, or launching without learning goals.

## Example

MVP: one workspace, one report type, manual import, export summary; not included: integrations, roles, billing, analytics.
