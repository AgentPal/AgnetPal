# User Segment Persona Skill

## Purpose

Define the first target user segment and useful persona details without turning persona work into fictional theater.

## When to use

Use when the user says "everyone", "teams", "users", "creators", "operators", or any broad audience that cannot guide scope.

## Inputs

Candidate users, context, task frequency, urgency, willingness to switch, ability level, constraints, and evidence.

## Required context

Primary segment, secondary segment, first scenario, user ability, current alternative, switching trigger, and excluded users.

## Process

Split user groups, choose a first segment, define scenario and trigger, identify current alternative, note constraints, and mark unsupported assumptions.

## Output

Segment/persona brief with first target, non-targets, scenario, switching reason, risks, and evidence gaps.

## Quality bar

The segment should make scope decisions easier and prevent building for everyone.

## User confirmation points

Confirm first target user, excluded users, scenario priority, and evidence basis.

## No-code boundary

This skill produces user framing artifacts only.

## Common mistakes

Creating decorative personas, over-generalizing users, or claiming market size without evidence.

## Example

"Small businesses" becomes "solo service providers who lose leads because follow-up reminders are scattered across inbox and notes."
