# Prioritization Skill

## Purpose

Rank product options using value, reach, confidence, effort, risk, dependencies, and product stage.

## When to use

Use for feature selection, roadmap trade-offs, release planning, stakeholder conflicts, or scope cuts.

## Inputs

Candidate items, user value, reach estimate, confidence, effort, risk, dependencies, strategy, and release goal.

## Required context

Decision goal, scoring method, evidence confidence, cost/risk assumptions, and constraints.

## Process

Choose a method such as RICE, MoSCoW, Kano-style satisfaction reasoning, or qualitative trade-off table. Score or classify items, explain confidence, and recommend a cut.

## Output

Prioritization table, decision rationale, trade-offs, evidence gaps, and user decision points.

## Quality bar

Prioritization should make trade-offs visible and should not pretend weak estimates are precise.

## User confirmation points

Confirm scoring assumptions, must-have items, effort estimates, and final priority order.

## No-code boundary

This skill produces prioritization artifacts only.

## Common mistakes

Using a formula as false certainty, prioritizing by stakeholder volume, or ignoring confidence.

## Example

Rank onboarding improvements by reach, impact, confidence, and effort, then cut low-confidence high-effort items from V1.
