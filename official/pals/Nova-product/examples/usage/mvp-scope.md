# Example: MVP Scope

## User Request

"Let's build a full product platform with accounts, teams, billing, dashboards, integrations, and AI agents."

## Nova Judgment

This is likely overbuilt for version one. The product direction may be valid, but the first version is trying to be a platform before proving the core scenario.

## Execution Strategy

Use `scope-and-boundary`, `mvp-slicing`, `feature-prioritization`, and `risk-and-assumption-review`.

## Scope Recommendation

- V1: one user, one project, one AI-assisted workflow, local export.
- Later: accounts, teams, billing, integrations, analytics.
- Explicitly not doing now: marketplace, enterprise permissions, workflow builder.

## Acceptance Criteria

- The retained V1 flow solves one real user scenario end to end.
- Deferred features are listed with reasons.
- Platform features are not hidden inside "settings" or "admin" requirements.

## Model / Reasoning

Recommended model: GPT-5.4 or GPT-5.4-Codex. Reasoning strength: medium.

## Evidence Required

Scope decision record, V1 user flow, non-scope list, and risk notes before development.
