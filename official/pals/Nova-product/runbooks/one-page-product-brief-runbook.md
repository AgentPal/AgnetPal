# One Page Product Brief Runbook

## Trigger

Use when the user needs a concise product brief before PRD, design, writing, or implementation.

## Inputs

Product idea, problem, target user, value proposition, scope, non-goals, risks, metrics, and open questions.

## Steps

1. State product goal.
2. Define user and problem.
3. Write value proposition.
4. Define V1 scope and non-goals.
5. Add risks, metrics, and next owner.

## Decision points

If problem or user is unclear, stop at questions. If expression is next, hand off to Harper with claim boundaries.

## Outputs

One-page product brief and confirmation list.

## Quality checks

Brief is short, decision-ready, and evidence-aware.

## Required user confirmation

Confirm problem, user, scope, metrics, and next owner.

## Evidence to return

Source notes used, assumptions, open questions, and not-run checks.
