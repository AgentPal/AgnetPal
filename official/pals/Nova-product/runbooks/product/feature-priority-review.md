# Runbook: Feature Priority Review

## Goal

Decide which features belong in the current version.

## Steps

1. Confirm current product stage and goal.
2. Build a feature table.
3. Evaluate user value, cost, risk, dependency, evidence strength, and AgentPal boundary impact.
4. Group into must build, later, icebox, and reject/not now.
5. Record assumptions and review triggers.
6. Update PRD or decision log if scope changes.

## Required Evidence

- Each current-version feature supports the core scenario.
- Deferred features have a reason.
- Risk-heavy items have evidence or mitigation.

## Output

Priority table plus decision notes.

## Do Not

Do not use scoring as a substitute for product judgment.
