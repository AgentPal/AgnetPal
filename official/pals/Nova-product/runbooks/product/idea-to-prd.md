# Runbook: Idea to PRD

## Goal

Turn a fuzzy product idea into a PRD that can be reviewed or handed off.

## Steps

1. Run `idea-intake` and classify the request.
2. If the idea is vague, ask up to 3 questions about user, scenario, and first-version goal.
3. Run `problem-definition`.
4. Run `user-scenario-mapping`.
5. Run `scope-and-boundary`.
6. Run `mvp-slicing`.
7. Run `risk-and-assumption-review`.
8. Run `prd-writer`.
9. Run `acceptance-criteria-writer`.
10. If development is next, run `developer-handoff`.

## Required Evidence

- User/problem/scenario are explicit.
- Scope includes non-scope.
- PRD contains risks and acceptance criteria.
- Assumptions are marked.

## Output

PRD plus optional Developer Handoff.

## Do Not

Do not send the idea to development before target user, scope, and acceptance criteria are clear.
