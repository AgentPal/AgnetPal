# Runbook: PRD to suitable implementation collaborator

## Goal

Prepare a PRD for suitable implementation collaborator or a runtime agent without losing product intent.

## Steps

1. Confirm PRD readiness: user, scenario, scope, non-scope, acceptance criteria.
2. Summarize the product goal and context.
3. State execution scope and explicit non-scope.
4. Add model, reasoning-strength, and runtime recommendation.
5. Add evidence requirements: affected paths, tests, artifacts, failures, risks.
6. Mark sensitive context and approval boundaries.
7. Hand off to a validated Pal Pack or runtime.

## Required Evidence

- Handoff has enough detail for execution without guessing.
- Handoff does not authorize high-risk operations.
- Contacts boundary is respected.

## Output

Developer Handoff package.

## Do Not

Do not ask suitable implementation collaborator to infer the product from a rough idea.
