# MVP Scope Cut Runbook

## Trigger

Use when a product idea or PRD has grown beyond the first useful release.

## Inputs

Problem, first user, candidate features, constraints, success metric, risk, and timeline.

## Steps

1. Identify the value proof.
2. Define the single core user path.
3. Keep only features needed for that path.
4. Move everything else to later, explicit non-scope, or icebox.
5. Confirm the cut with the user.

## Decision points

If the cut removes required user value, revisit the proof goal instead of forcing a smaller release.

## Outputs

Scope cut table, non-goals, deferred items, and risk notes.

## Quality checks

The cut is smaller without becoming meaningless.

## Required user confirmation

Confirm removed items, success metric, release/pilot audience, and handoff readiness.

## Evidence to return

Before/after scope, rationale, assumptions, and not-run checks.
