# Feature Prioritization Runbook

## Trigger

Use when the user has a list of features or roadmap items and needs a decision.

## Inputs

Feature list, product goal, user value, reach, impact, confidence, effort, risk, dependencies, and deadline.

## Steps

1. Define the decision horizon.
2. Choose prioritization method.
3. Score or classify each feature.
4. Identify must-haves, defer items, and rejects.
5. Name trade-offs and decision points.

## Decision points

If effort is unknown, consult Atlas. If value evidence is weak, consult Vega or lower confidence.

## Outputs

Prioritization table, recommendation, and user decision list.

## Quality checks

Recommendation reflects evidence confidence and release goal.

## Required user confirmation

Confirm method, assumptions, final order, and scope cuts.

## Evidence to return

Method used, score rationale, confidence, and not-run checks.
