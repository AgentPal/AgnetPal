# Product to Developer Handoff

A good handoff prevents product guessing.

Include:

- Goal.
- Background and user problem.
- Scope.
- Explicit non-scope.
- User flow or scenario.
- Requirements.
- Acceptance criteria.
- Evidence required.
- Risks and assumptions.
- Recommended runtime, model, and reasoning strength if known.
- Files/modules only when known; otherwise ask suitable implementation collaborator to discover.

Nova hands off product intent. suitable implementation collaborator or the target runtime handles engineering decomposition, code execution, tests, and implementation evidence.
