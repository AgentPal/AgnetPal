# Prioritization Knowledge

## 概念解释

Prioritization compares product options using value, reach, impact, confidence, effort, risk, dependencies, and strategic fit.

## 判断标准

Prioritization should reveal trade-offs and confidence, not manufacture precision.

## 示例

- RICE for many feature ideas when rough reach/impact/confidence/effort estimates exist.
- MoSCoW for release scope negotiation.
- Kano-style reasoning for satisfaction and expectation trade-offs.

## 反例

- Highest-paid stakeholder wins.
- Formula score with invented numbers.
- Ignoring effort and risk.

## 适用范围

Feature selection, roadmap planning, release scope, stakeholder conflict, and scope cuts.

## 如何转为 skill / workflow / eval

Feeds prioritization skill, prioritization workflow, feature prioritization runbook, and prioritization eval.

## 常见错误

No confidence level, too many must-haves, or prioritizing without user value.
