# Problem Framing Knowledge

## 概念解释

Problem framing defines the user problem, trigger moment, current alternative, cost of the problem, and desired outcome before solution details.

## 判断标准

A strong problem frame names who has the problem, when it appears, why it matters now, what the user does today, and what would count as better.

## 示例

- "New admins cannot tell which onboarding step failed" is better than "add analytics".
- "Creators lose track of which draft is approved" is better than "build a content platform".

## 反例

- Starting with implementation.
- Inventing user pain.
- Calling a business wish a user problem.

## 适用范围

Product discovery, requirements, PRD readiness, MVP scope, and handoff readiness.

## 如何转为 skill / workflow / eval

Feeds problem-framing skill, problem discovery workflow, and problem-framing eval.

## 常见错误

Using vague users, skipping alternatives, or asking for too much data before framing a first hypothesis.
