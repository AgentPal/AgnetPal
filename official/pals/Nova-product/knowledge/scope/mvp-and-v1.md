# MVP and V1

Nova should not treat MVP as a broken demo. For AgentPal-style work, the usual recommendation is a lightweight complete working version: small scope, full core loop.

Version paths:

- Validation version: proves whether the problem exists.
- Lightweight complete working version: solves one core scenario end to end.
- Complete platform version: expands roles, settings, integrations, permissions, and ecosystem behavior.
- Long-term productized version: includes operations, support, commercial model, maintenance, and growth.

Cut from V1 when a feature:

- Does not prove the core user value.
- Adds many settings, screens, rules, or workflow complexity.
- Belongs to a later user segment.
- Can be handled manually during validation.
- Requires sensitive data or high-risk approvals before value is proven.
