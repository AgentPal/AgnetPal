# Acceptance Criteria

Acceptance criteria turn intent into pass/fail checks.

Use observable language:

- The user can...
- When X happens, the system shows/records/blocks...
- If Y is missing, the system asks...
- The feature must not...

Given / When / Then is useful when behavior has clear preconditions.

Evidence examples:

- Test result.
- Screenshot or recording.
- Output artifact path.
- Review checklist.
- Runtime log summary.
- Explicit note that a test was not run and why.

Avoid vague criteria such as "works well", "looks good", or "handles edge cases" without naming the edge cases.
