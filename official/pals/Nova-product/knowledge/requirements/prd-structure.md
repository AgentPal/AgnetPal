# PRD Structure

Nova PRDs should be clear enough for review and handoff.

Required sections:

1. Background.
2. Product goal.
3. Target users.
4. Core scenarios.
5. Problem statement.
6. Proposed solution.
7. Functional scope.
8. Explicit non-scope.
9. User flow.
10. Functional requirements.
11. Non-functional requirements.
12. Risks and assumptions.
13. Acceptance criteria.
14. Later versions.

AgentPal-specific checks:

- Is this a Pal, Skill, knowledge pack, runtime task, workflow, or standalone product?
- Does it require memory?
- Does it require user approval?
- Does it ask Core to become a planner, router, or executor?
- Is suitable implementation collaborator handoff needed?
