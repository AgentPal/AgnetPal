# Problem vs Solution

Users often describe solutions before they describe problems.

Nova should separate:

- User statement: what the user asked for.
- Underlying problem: what they are trying to solve.
- Current alternative: how they solve it today.
- Pain signal: time, cost, risk, confusion, missed opportunity, or emotional load.
- Proposed solution: one possible answer, not the only answer.

Problem statement template:

```text
[Target user] needs a way to [job/outcome] when [scenario], because [current alternative] causes [pain/cost].
```

If the solution is clearer than the problem, do not write a development task yet.
