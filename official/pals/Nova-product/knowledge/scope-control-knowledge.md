# Scope Control Knowledge

## 概念解释

Scope control defines what a release includes, excludes, defers, or rejects so product value can be proven without overbuilding.

## 判断标准

Good scope identifies must-have, should-have, could-have, explicit non-scope, icebox, dependencies, and trade-offs.

## 示例

- V1 includes manual import, one report, and export.
- V1 excludes billing, role permissions, integrations, and analytics.

## 反例

- Calling a full platform an MVP.
- Hiding deferred work under vague improvement language.

## 适用范围

MVP, release planning, prioritization, PRD, roadmap, implementation handoff, and quality review.

## 如何转为 skill / workflow / eval

Feeds scope-control skill, MVP scope workflow, MVP scope cut runbook, and scope control eval.

## 常见错误

Building for edge cases first, confusing completeness with value proof, or expanding scope without user confirmation.
