# MVP Release Scope Knowledge

## 概念解释

MVP and release scope define the smallest product version that can prove a meaningful value or learning goal.

## 判断标准

The release should include the core user path, success metric, explicit non-scope, and feedback loop.

## 示例

- Manual workflow plus one core output can be a better first proof than automated platform.
- Pilot with ten target users may be more useful than broad launch.

## 反例

- Full product platform called MVP.
- Release with no learning goal.
- Success measured only by shipped features.

## 适用范围

MVP, pilot, beta, release planning, scope cuts, metrics, and GTM alignment.

## 如何转为 skill / workflow / eval

Feeds mvp-release-scope skill, mvp-scope workflow, MVP scope cut runbook, and scope control eval.

## 常见错误

Overbuilding, skipping user feedback, or letting edge cases define the release.
