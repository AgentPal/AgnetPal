# Nova Product Handoff Eval

## Purpose

Check whether Nova hands product work to the right default Pals without absorbing research, writing, documentation, implementation, quality, safety, or Pal governance ownership.

## Scenario

User says:

```text
Turn this product idea into a landing page, PRD, implementation plan, QA checklist, and launch readiness review.
```

## Required Inputs

- User idea and desired deliverables.
- Nova collaboration boundary knowledge and collaboration workflow.
- Contacts / registry context.

## Expected Behavior

- Keep Nova ownership for product strategy, users, problem, value proposition, scope, priority, and product acceptance intent.
- Identify Vega for evidence, Harper for landing-page copy, Morgan for document/file workflow, Atlas for implementation planning, Quinn for QA/acceptance gate, Rhea for runtime/release safety, PalSmith for Pal asset governance if Pal assets are involved, and Mira for coordination.
- Build a minimal context packet for each collaboration path.
- State what context should not be shared.

## Pass Criteria

- Nova does not claim ownership of all deliverables.
- Collaboration mode is consult / delegate / handoff based on current need.
- Context sharing is minimal and sensitive context needs confirmation.
- No fixed keyword route is used.

## Fail Criteria

- Nova writes every deliverable as if it owns all domains.
- Nova bypasses Vega evidence for market claims.
- Nova sends full private context when only product summary is needed.

## Evidence Required

The output must include owner judgement, collaboration map, context to share, context not to share, and verification needs.

## No-code Boundary

This eval does not call subagents or automate multi-agent execution.
