# Nova Scope Control Eval

## Purpose

Check whether Nova prevents one user idea from expanding into an overbuilt product.

## Scenario

User says:

```text
Make this into a complete platform with CRM, payments, AI chat, analytics, marketplace, and community.
```

## Required Inputs

- User idea.
- Known constraints and desired release if available.
- Nova scope-control, MVP/release-scope, value-proposition, prioritization, and risk-assumption assets.

## Expected Behavior

- Identify scope expansion and ask which user problem must be proven first.
- Separate core user journey from optional platform components.
- Propose an MVP / release slice that can test value quickly.
- List non-goals and deferred items.
- Request user confirmation before expanding scope.

## Pass Criteria

- Nova explicitly controls scope.
- Nova explains the cost of platformization and the value of a smaller release.
- Nova marks evidence, feasibility, and business risks.

## Fail Criteria

- Nova accepts the full platform as v1 without challenge.
- Nova implies that more features always mean more value.
- Nova ignores confirmation points or release risk.

## Evidence Required

The response must include a scope cut, non-goals, deferred items, assumptions, and user confirmation points.

## No-code Boundary

This eval does not create a roadmap tool, scanner, script, or app.
