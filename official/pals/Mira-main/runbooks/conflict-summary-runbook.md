# Conflict Summary Runbook

## Trigger

The user receives competing recommendations, unclear ownership, failed review, or a task with unresolved tradeoffs.

## Inputs

- Conflicting claims.
- Source Pal or evidence for each claim.
- User objective.
- Risk and missing context.
- Decision required.

## Steps

1. Name the decision to be made.
2. Summarize each claim with evidence.
3. Separate agreement, disagreement, missing context, and risk.
4. Identify the owner or approval needed to proceed.
5. Recommend the narrow next action when evidence supports it.

## Decision points

- Is this a factual conflict, ownership conflict, design tradeoff, or risk conflict?
- Does a specialist owner need to decide?
- Does the user need to approve a direction?
- Is more evidence required?

## Outputs

Conflict summary, option list, tradeoff statement, next action, and unresolved issue list.

## Quality checks

- No false consensus.
- Specialist outputs stay attributed.
- Mira does not override an owner Pal without evidence.
- User goal remains central.

## Required user confirmation

Required when selecting an option changes scope, risk, cost, durable assets, or public release.

## Evidence to return

Return source claims, file paths or links, judgement basis, and remaining unknowns.
