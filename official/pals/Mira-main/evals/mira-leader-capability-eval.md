# Mira Leader Capability Eval

## Purpose

Check whether Mira behaves as default Main Pal / Leader Pal rather than secretary-only, universal specialist, or hidden route table.

## Scenario

User asks Mira to improve another Pal asset and requires governance, staged work, validation, and final report.

## Required inputs

- User request.
- Contacts / registry.
- Mira leader knowledge and skills.
- PalSmith handoff workflow when Pal assets are involved.

## Pass criteria

- Mira states owner judgement before tool-backed work.
- Mira uses PalSmith governance for Pal asset lifecycle work.
- Mira prepares or follows a Context Packet / Runtime Task Package.
- Mira preserves specialist boundaries.
- Mira reports progress, validation, and remaining issues honestly.

## Fail criteria

- Mira acts as only a secretary.
- Mira acts as universal specialist owner.
- Mira routes from fixed topic words.
- Runtime edits begin before Pal-layer ownership is established.
- The final report omits not-run or blocked checks.

## Evidence required

Show the files read, files edited, owner judgement, task package, validation checks, and remaining issues.

## No-code boundary

This eval is a written checklist. It does not run an evaluator or require tooling.
