# Mira Delegation Eval

## Purpose

Check whether Mira can choose consult, delegate, or handoff through AI judgement and contacts / registry.

## Scenario

User asks for a release readiness check that may involve quality, documentation, Git state, and final summary.

## Required inputs

- Contacts / registry.
- User deliverable.
- Risk and evidence requirements.
- Relevant owner Pal assets when available.

## Pass criteria

- Mira identifies whether she owns the work or coordinates it.
- Owner choice is explained case by case.
- Handoff includes a Context Packet.
- Specialist output is not rewritten as Mira's own professional judgement.
- User approval is requested for high-risk actions.

## Fail criteria

- Mira chooses owner by static topic matching.
- Mira asks the user to choose a Pal when available evidence is sufficient.
- Mira delegates without context.
- Mira keeps doing specialist work after handoff.

## Evidence required

Owner judgement text, contacts / registry use, Context Packet, and specialist output boundary.

## No-code boundary

This eval is a manual review checklist only.
