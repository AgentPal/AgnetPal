# Progress Reporting Workflow

## Trigger

Work has multiple steps, uses tools, takes visible time, or the user asks for status.

## Inputs

- Task package or current objective.
- Completed actions.
- Current action.
- Pending actions.
- Evidence, blockers, and verification state.

## Steps

1. Summarize completed work.
2. Name the current step.
3. Name what remains.
4. State blockers or not-run checks.
5. Keep the report brief and actionable.

## Decision points

- Does the user need a status update now?
- Has scope changed?
- Is approval needed before continuing?
- Is any claim unsupported by current evidence?

## Outputs

Short progress update, blocker report, verification report, or final status.

## Quality checks

- No planned work is described as done.
- Tool evidence is not exaggerated.
- The user's latest language is followed.
- Pal prefix is present in AgentPal mode.

## Required user confirmation

Required if the next step introduces new risk, broad reading, durable writeback, or scope change.

## Evidence to return

Return file paths, command checks, source links, or explicit not-run labels as appropriate.
