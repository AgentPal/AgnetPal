# Pal Team Roles Knowledge

## 概念解释

Mira uses contacts / registry as the source of available Pal collaborators. Capability notes help judgement but are not routes. The current request, deliverable, risk, and evidence needs decide ownership.

## 判断标准

- Check contacts / registry when a Pal owner matters.
- Do not maintain a private fixed list of specialist owners inside Mira knowledge.
- Explain owner choice briefly and specifically.
- Report an owner gap when no registered Pal fits.

## 示例

- Good: "PalSmith fits because this is Pal asset lifecycle governance."
- Good: "No registered Pal clearly owns this; Mira can provide an honest fallback."
- Good: "Atlas may execute file edits only after a Pal-layer owner task package."

## 反例

- Bad: Static domain table says every code word means Atlas.
- Bad: Mira claims only she can call PalSmith.
- Bad: Contacts are ignored and a non-existent Pal is invented.

## 适用范围

Applies to registered Pal discovery and owner judgement in AgentPal v0.1.

## 如何转为 skill / workflow / eval

Use with `ai-judgement-delegation-skill.md`, `default-pal-collaboration-map.md`, and `mira-delegation-eval.md`.

## 常见错误

- Treating examples as binding routes.
- Failing to update reasoning after contacts change.
- Hiding owner uncertainty.
