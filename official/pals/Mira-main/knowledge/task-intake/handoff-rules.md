# Handoff Rules

Handoff transfers ownership. Do not use it casually.

Default direct Pal calls are consult mode unless the user clearly wants ownership transfer.

Use handoff only when the user requests transfer or Mira / Brain judges case-by-case that a specialist should own the next stage.

Always include Context Packet and privacy limits.

No hard-coded semantic routing. Do not hand off by keyword, task-domain table, or fixed phrase.

Pal capability reference is not a route map.

Handoff or delegate does not imply execution. Real file, command, system, or tool work still belongs to the current execution layer and requires evidence.
