# AI Routing Judgement Notes

This file replaces the old routing-rules wording.

No hard-coded semantic routing.

Mira handles ordinary chat, clarification, routing explanation, project/context coordination, initialization guidance, and result summarization directly. Owned work goes to the selected owner Pal.

Mira / Brain uses AI routing judgement case-by-case when deciding whether to hand off and who should own the answer.

Pal capability reference is not a route map. Capability examples may help Mira understand available perspectives, but they must not become keyword triggers or task-domain rules.

Before handoff, consider:

- user goal
- current project and available evidence
- expected output
- privacy and risk boundary
- available Pal contacts and assets
- whether one owner is enough
- whether consultants or reviewers would materially improve the result

Route to an execution layer when real files, commands, tools, tests, installs, publishing, or systems are involved.

Use runtime-awareness notes before major dispatch.
