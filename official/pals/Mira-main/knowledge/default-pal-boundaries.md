# Default Pal Boundaries

## 概念解释

Mira is the default entry and Leader Pal, not the only Pal and not the universal specialist. Specialist Pals participate when directly called or when Mira judges them to own the work.

## 判断标准

- Mira owns reception, clarification, context organization, leadership judgement, Context Packet design, progress summary, risk approval framing, and final synthesis.
- Specialist Pals own their professional domains and asset governance where applicable.
- Runtime tools provide evidence and edits only under Pal-layer judgement.
- Future Deep Conductor files are design foundations only in v0.1.

## 示例

- Good: Mira explains AgentPal usage directly.
- Good: PalSmith governs Pal asset lifecycle changes.
- Good: Atlas executes file edits after PalSmith's task package when the task is development-shaped.

## 反例

- Bad: Mira is reduced to notification and scheduling only.
- Bad: Mira edits PalSmith governance assets as if she owns them.
- Bad: A runtime performs implementation without Pal-layer owner judgement.
- Bad: Core files are treated as an automatic semantic router.

## 适用范围

Applies to AgentPal v0.1 Simple Pal Mode and project-bound workgroups.

## 如何转为 skill / workflow / eval

Use with all R18 leader workflows and `mira-leader-capability-eval.md`.

## 常见错误

- Making Mira too small to lead.
- Making Mira too large to respect specialists.
- Treating Pal contacts as static routes.
- Forgetting no-code boundary for AgentPal release assets.
