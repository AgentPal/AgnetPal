# Conflict Summary Skill

## Purpose

Help Mira summarize competing recommendations, unclear ownership, or conflicting constraints without erasing specialist perspectives.

## When to use

Use when Pals disagree, evidence conflicts, user goals conflict, or a task has unresolved tradeoffs.

## Inputs

- Competing claims or recommendations.
- Source Pal or evidence for each claim.
- User goal and constraints.
- Risks and missing information.

## Required context

Use only the relevant Pal outputs or evidence. Do not expand into unrelated research or hidden assumptions.

## Process

1. List the decision to be made.
2. Summarize each position with its evidence.
3. Name agreement, disagreement, and missing context.
4. Identify the approval or owner needed to proceed.
5. Recommend the next narrow step if evidence supports it.

## Output

A concise conflict summary with options, tradeoffs, and next action.

## Quality bar

The summary must make unresolved tension clearer. It must not falsely force consensus or present Mira's preference as specialist proof.

## User confirmation points

Ask when choosing between options would change scope, risk, cost, schedule, identity, release readiness, or data exposure.

## No-code boundary

This skill produces written synthesis only. It does not arbitrate through hidden automation.

## Common mistakes

- Hiding disagreement to sound decisive.
- Overriding specialist owners without evidence.
- Turning every disagreement into a long plan.
- Forgetting the user's actual requested outcome.

## Example

Atlas wants to edit Mira immediately, while PalSmith requires governance first. Mira summarizes that PalSmith governance must precede Atlas execution for Pal asset quality.
