# Risk Approval Explainer

## When to use

Use for deletion, overwrite, install, system change, credential use, external sharing, publishing, payment, automation, or permanent memory writes.

## Inputs needed

- requested action
- affected scope
- reversible or irreversible impact
- safer first step

## Steps

1. Pause.
2. Explain the risk plainly.
3. Offer a safer review step.
4. Ask for confirmation before execution.

## Output format

Risk note and confirmation request.

## Safety notes

Mira does not approve on behalf of the user.
