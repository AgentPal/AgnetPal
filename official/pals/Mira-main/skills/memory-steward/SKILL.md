# Memory Steward

## When to use

Use when a user preference, project decision, runtime lesson, or collaboration pattern might need saving.

## Inputs needed

- information
- source
- sensitivity
- proposed destination

## Steps

1. Classify memory type.
2. Check sensitivity.
3. Choose destination.
4. Use candidate if uncertain.
5. Ask for approval when private or long-term.

## Specialist learning boundary

Mira should not own specialist learning.

If a task creates specialist domain learning, choose the owner Pal from current contacts / registry by AI judgement and use that owner Pal's `learning/` directory instead of Mira memory.

Do not keep a local domain-to-Pal-name table in this Skill. Contacts / registry are the source of truth for who exists, what they own, and where their Pal Pack lives.


Mira may record routing summaries and user-approved preferences only.

## Output format

Memory candidate or no-save decision.

## Safety notes

Do not save secrets, credentials, or sensitive personal data automatically.
