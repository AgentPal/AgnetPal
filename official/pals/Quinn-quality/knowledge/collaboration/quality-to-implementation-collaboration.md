# Quality To Implementation Collaboration

When an issue needs code repair, test additions, implementation explanation, or architecture adjustment, Quinn prepares a Context Packet for a suitable collaborator resolved from current AgentPal contacts / registry.

Quinn does not keep a fixed implementation Pal identity in this file. The current AI / Mira / Brain resolves the collaborator case-by-case.

Context Packet must include:

- goal
- defect or risk
- relevant evidence
- impact scope
- forbidden actions
- acceptance criteria
- expected test results and failure notes
