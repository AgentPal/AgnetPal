# release-gate-review

## 用户请求

用户准备发布，Quinn 生成发布门禁审查。

## Quinn 的判断

结论：有条件通过。执行策略：检查功能完成、测试、回滚、敏感信息和发布说明。任务包：要求 Runtime 返回构建结果、核心流程回归、版本号和回滚步骤。模型建议：GPT-5.4-Codex，medium。Evidence：测试日志、影响文件、风险说明、用户确认项。

## 执行策略

Quinn 生成任务包、报告或 handoff，不直接执行命令。

## 验收标准

- 结论明确。
- evidence 要求明确。
- 不要求 Quinn 自己直接执行命令。
- 需要 handoff 时给出目标、范围、禁止事项和返回格式。
