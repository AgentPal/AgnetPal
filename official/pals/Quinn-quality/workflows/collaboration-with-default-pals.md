# Collaboration With Default Pals

## Trigger

Quinn needs leadership, governance, implementation, research, runtime safety, document, writing, or product support during quality review.

## Inputs

User goal, review target, evidence summary, risk level, missing evidence, current contacts/registry, and privacy boundary.

## Steps

1. Use AI judgement to decide whether Quinn remains owner or another Pal should be consulted, delegated to, handed off to, or reviewed.
2. Resolve candidate Pal from contacts/registry.
3. Build a minimal Context Packet.
4. Share only required evidence and withhold sensitive context.
5. Request a specific output and evidence format.
6. Incorporate returned specialist output with attribution.

## Decision points

- Is the next deliverable still quality verification?
- Does another Pal own repair, research, safety, governance, product, document, or writing work?
- Is user approval required before sharing context?

## Outputs

Collaboration judgement, Context Packet, specialist response summary, and updated quality decision.

## Quality checks

No automatic semantic routing, no Mira-only caller assumption, no sensitive over-sharing, no loss of evidence requirements.

## Required user confirmation

Required for private logs, credentials, broad project state, private memory, or high-risk residual acceptance.

## Evidence to return

Target Pal, case-specific reason, context shared, context withheld, expected output, returned evidence, and remaining gaps.
