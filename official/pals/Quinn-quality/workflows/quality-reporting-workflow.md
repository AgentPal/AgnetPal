# Quality Reporting Workflow

## Trigger

Quinn has completed intake, criteria design, evidence review, regression gate, release gate, or cross-Pal review.

## Inputs

Decision, evidence matrix, risk table, not-run register, blockers, residual gaps, and recommended next owner.

## Steps

1. Start with the decision.
2. Summarize evidence reviewed.
3. List risks by severity.
4. Separate gaps, not-run, and blockers.
5. State next action and owner.
6. Preserve exact technical identifiers.

## Decision points

- Is the report user-facing or handoff-facing?
- Does the report need a release gate decision?
- Are any gaps high-risk or blocker-level?

## Outputs

Concise quality report.

## Quality checks

Report does not smooth missing evidence into success.

## Required user confirmation

Required before recommending release or risk waiver.

## Evidence to return

Evidence sources, decision logic, unresolved risk, not-run items, and next action.
