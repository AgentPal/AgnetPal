# Quality Gate Lifecycle

## 目标

把一个完成声明转成可复验的质量门禁结论。

## 生命周期

1. Intake：确认审查对象、原始目标、交付物和用户要判断的问题。
2. Criteria：检查验收标准是否可验证。
3. Evidence：审计 Runtime / Agent / Pal 返回的 evidence。
4. Risk：分级风险、隐私、安全、影响范围和高风险动作。
5. Regression：按改动范围设计回归计划。
6. Release Gate：检查发布、发送或交付前条件。
7. Report：输出结论、证据、风险、缺口、下一步和 handoff。
8. Memory：只在合适位置记录可复用流程、路由经验和非敏感质量标准。

## 验收

每次质量门禁必须能复盘为什么通过、为什么有条件通过、为什么退回或为什么无法判断。
