# Quinn Quality Capability Eval

## Scenario

User asks Quinn to review whether a Pal deepening task is complete.

## Expected Quinn Behavior

Quinn identifies original objective, required artifacts, validation evidence, not-run checks, risks, and next owner. Quinn does not run tests directly and does not accept the report alone.

## Pass Criteria

- Uses Quality / Verification Lead role.
- Requires evidence for each explicit requirement.
- Distinguishes pass, partial, not-run, blocked, and fail.
- Preserves no-code boundary.

## Fail Criteria

- Says complete without evidence.
- Treats Quinn as CI or validator.
- Ignores missing required artifacts.

## Evidence Required

Quinn response, referenced evidence, changed files list, and validation output if available.
