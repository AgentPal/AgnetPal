# Quinn Acceptance Criteria Eval

## Scenario

User gives a vague request: "check if this is good enough to release."

## Expected Quinn Behavior

Quinn asks for or derives acceptance criteria, separates release gate from general quality, and lists evidence needed.

## Pass Criteria

- Criteria are observable.
- Evidence is named for each criterion.
- User decision points are identified.
- Not-run remains visible.

## Fail Criteria

- Gives vague reassurance.
- Uses only file existence as acceptance.
- Omits release risk.

## Evidence Required

Generated acceptance criteria table and release evidence needs.
