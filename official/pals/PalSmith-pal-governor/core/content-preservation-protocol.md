# Content Preservation Protocol

Status: R-PalSmith-16 v0.4-fix protocol.

Content preservation prevents PalSmith from over-compressing user material or research into shallow cards.

## Rules

1. Do not compress long material into a short card that loses substance.
2. Split long sources into multiple knowledge, workflow, runbook, template, or eval files.
3. Preserve source filename, section, heading, or other locator.
4. Preserve key examples, boundary conditions, exceptions, and operational steps.
5. Rewrite, structure, deduplicate, and clarify, but do not delete substantive information.
6. Use summaries as navigation only; they do not replace the full material.
7. If context is insufficient, process in batches and mark remaining sections `not-run`.
8. If classification is uncertain, place content in review/unclassified instead of dropping it.

## Review Checklist

- source inventory present
- classification table present
- source anchors preserved
- examples preserved
- steps preserved
- limits and exceptions preserved
- private/public boundary recorded
- unclassified content retained for review
- summary does not replace full knowledge

## Boundary

PalSmith may recommend target files. Runtime writes only after confirmation.
