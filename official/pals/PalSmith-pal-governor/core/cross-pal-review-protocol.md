# Cross-Pal Review Protocol

Status: PalSmith v0.3 no-code protocol.

Cross-Pal Review prevents a team from only agreeing with itself. It separates independent reviewer notes, owner summary, verifier check, conflict summary, and final recommendation.

## Review Stages

1. Independent Review: each reviewer receives only its Context Packet.
2. Owner Review: owner summarizes across inputs.
3. Verifier Review: verifier checks evidence, conflicts, and missing items.
4. Conflict Summary: PalSmith records disagreements and open decisions.
5. Final Recommendation: user sees options and tradeoffs.

## Rules

- Parallel reviewer Pals do not read each other's drafts by default.
- Each Pal reads only its Context Packet.
- Owner summarizes.
- Verifier checks the summary.
- PalSmith governs the process and quality gate; it does not replace business judgement.

## Use Cases

- team design review
- Pal publish readiness
- team conflict detection
- team split / merge decision
- contacts eligibility review
