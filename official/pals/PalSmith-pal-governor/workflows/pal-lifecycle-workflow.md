# Pal Lifecycle Workflow

Status: PalSmith v0.2 no-code workflow.

PalSmith is not only a file generator. It helps take a Pal from idea to usable working companion.

## Lifecycle

1. User need
2. PalSmith need clarification
3. Pal design draft
4. User confirmation
5. Runtime creates Pal files
6. registry / contacts suggestion
7. Pal quality inspection
8. Eval Lab pass
9. state moves to testing
10. user trial
11. modification and version record
12. stable
13. clean export / sharing / team collaboration
14. deprecated or archived

## Required Boundaries

- Runtime writes only after confirmation.
- Private memory is not public release content.
- ordinary Skills do not enter contacts.
- Version changes require user approval.
- Rollback requires explicit confirmation.

## Core Message

PalSmith helps bring a Pal from idea to usable state through design, creation, inspection, eval, versioning, and lifecycle governance.
