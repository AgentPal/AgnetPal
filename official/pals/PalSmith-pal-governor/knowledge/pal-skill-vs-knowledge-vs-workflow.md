# Skill Vs Knowledge Vs Workflow

## Concept

- Knowledge = facts, rules, background, cases, domain context, and reference material a Pal needs to know.
- Skill = a recurring capability the Pal can perform or guide.
- Workflow = a multi-step task sequence.
- Runbook = a stable repeated operation manual.
- Template = a reusable output format.
- Eval = acceptance and test standard.

## Judgement Standards

Ask:

- Is this a fact or rule? Put it in knowledge.
- Is this an ability? Put it in skill.
- Is this a sequence? Put it in workflow.
- Is this a stable operation? Put it in runbook.
- Is this an output shell? Put it in template.
- Is this a test? Put it in eval.

## Examples

- Knowledge: "Clean export excludes memory/user and state."
- Skill: "Plan clean Pal export."
- Workflow: "Inspect, package, manifest, privacy review, report."
- Runbook: "Monthly Pal health review."
- Template: "Pal Quality Report table."
- Eval: "Reject ordinary Skill from contacts."

## Counterexamples

- A 10,000-word SOP compressed into one skill sentence.
- A process hidden inside knowledge without steps.
- A template treated as domain expertise.

## Scope

Use for creation, material ingestion, research conversion, quality inspection, and self-health.

## Convert To Assets

Classify each source section and write it to the smallest asset type that preserves its use.

## Common Errors

- mixing all content into one knowledge card
- making every file a skill
- omitting evals for important abilities
