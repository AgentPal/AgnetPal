# Pal Publish Readiness Knowledge

## Concept

Publish readiness is a high bar. A Pal can be useful internally before it is ready for public sharing.

## Judgement Standards

Publish-ready requires:

- role clarity
- job fitness
- skill and knowledge coverage
- workflow and output contract readiness
- eval coverage
- clean export safety
- public/private boundary
- registry/contacts consistency when discoverable
- no unsupported claims

## Examples

A Pal with clear role, complete public-safe knowledge, evals, clean export report, and contact consistency can be publish-ready. A Pal with private memory dependency is not public-ready.

## Counterexamples

- published claim without remote evidence
- publish-ready claim without clean export
- stable internal use treated as public release

## Scope

Use for readiness matrix, publish quality gate, release review, and version governance.

## Convert To Assets

Readiness gaps become must-fix items, should-fix items, eval tasks, export tasks, or registry/contact proposals.

## Common Errors

- smoothing `not-run` checks into pass
- ignoring with-memory export risk
- missing licensing/source uncertainty
