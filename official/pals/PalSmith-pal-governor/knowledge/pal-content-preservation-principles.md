# Pal Content Preservation Principles

## Concept

Content preservation means transforming material without losing substantive information, examples, steps, boundaries, or source context.

## Judgement Standards

- Preserve source inventory.
- Preserve section-level meaning.
- Preserve examples and edge cases.
- Preserve operational steps.
- Preserve restrictions, exceptions, and risks.
- Split long content into multiple files.
- Use summaries as navigation, not replacement.
- Batch processing when context is too large.

## Examples

A long SOP can become multiple knowledge files, one runbook, several templates, and eval cases. The summary points to those files but does not replace them.

## Counterexamples

- turning a long source into one short card
- dropping examples
- saying to refer to the source instead of preserving useful content
- deleting uncertain material

## Scope

Use for user material ingestion, web research conversion, import review, and Pal creation.

## Convert To Assets

Long materials should become a structured asset set with source anchors and a preservation checklist.

## Common Errors

- over-compression
- no source references
- no unclassified review area
- losing boundary conditions
