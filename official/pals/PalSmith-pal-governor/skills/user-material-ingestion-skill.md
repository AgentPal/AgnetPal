# User Material Ingestion Skill

## Purpose

Absorb user-provided files and notes into a Pal Pack while preserving meaning, detail, source location, and privacy boundaries.

## When To Use

Use when a user uploads documents, examples, SOPs, style guides, transcripts, templates, research notes, or prior outputs for a Pal.

## Inputs

- approved material paths or pasted text
- target Pal or team
- privacy/publication intent
- desired usage
- classification uncertainty from the user

## Required Context

Read user-material ingestion protocol, content preservation protocol, content preservation knowledge, target Pal role, and target Pal file structure.

## Process

1. Ask permission to read each source.
2. Create Source Inventory with file/section/date/scope.
3. Classify each section as Knowledge, Skill, Workflow, Runbook, Template, Eval, Example, Reference, Persona/Voice, Boundary, or Decision.
4. Preserve key examples, steps, edge cases, limitations, and source anchors.
5. Split long material into multiple files instead of over-compressing.
6. Put uncertain material in review/unclassified.
7. Prepare a Runtime Task Package for confirmed writes.
8. Review output with a content preservation checklist.

## Output

- Material Ingestion Plan
- Source Inventory
- Classification Table
- file plan
- unclassified review list
- preservation checklist
- Runtime Task Package

## Quality Bar

The ingested result should remain useful without the user having to reconstruct missing details from memory.

## User Confirmation Points

- each source read
- privacy classification
- file write targets
- handling of uncertain or sensitive material

## No-Code Boundary

Runtime reads and writes files. PalSmith classifies, plans, and reviews.

## Common Mistakes

- compressing long sources into tiny summaries
- losing examples or exceptions
- failing to preserve source location
- merging private material into public knowledge

## Example

A user's 10,000-word operations manual becomes a source inventory, several knowledge files, a workflow/runbook, templates, eval cases, and a review list for ambiguous sections.
