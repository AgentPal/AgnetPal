# Multi Agent Development Knowledge

## Concept explanation

Development tasks often include product, research, environment, quality, document, writing, or Pal asset stages. Atlas coordinates development-shaped stages with other Pals, but AgentPal v0.1 remains Simple Pal Mode and does not run active multi-agent execution by itself.

## Judgement standards

- Use case-specific AI judgement, not fixed route tables.
- Atlas owns implementation planning and evidence review stages.
- Other Pals may own research, product scope, environment, quality, documents, writing, or Pal asset governance.
- Runtime execution happens only under a Pal-layer task package.

## Examples

- Good: Vega researches an API, Atlas writes task package, Quinn reviews release readiness.
- Good: PalSmith governs Pal Pack changes before Atlas prepares runtime edits.
- Good: Mira remains leader for composite cross-Pal synthesis.

## Counterexamples

- Bad: Atlas claims all stages because final output includes code.
- Bad: Active parallel agents are claimed when no runtime evidence exists.
- Bad: A static collaborator list becomes a route map.

## Scope

Use for composite development tasks and default Pal collaboration.

## How it becomes skill / workflow / eval

Use with `multi-agent-development-coordination-skill.md`, `multi-agent-development-workflow.md`, `collaboration-with-default-pals.md`, and `atlas-developer-capability-eval.md`.

## Common mistakes

- Treating agent research docs as active AgentPal behavior.
- Sharing too much context.
- Forgetting user approval for sensitive context.
