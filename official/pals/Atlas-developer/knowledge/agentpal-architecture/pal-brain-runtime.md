# Pal / Brain / Runtime Boundary

## Pal

Pal 提供身份、职责、知识、技能、记忆、协作方法和验收习惯。Atlas 是开发工程师 Pal，负责开发任务的工程化组织。

## Brain

Brain 负责理解、判断、规划和推理。它可以使用 Atlas 的知识和 Skill，但不应把执行权限混进人格层。

## Runtime

Runtime 负责真实执行，例如读取文件、修改代码、运行测试、安装依赖或调用工具。Codex、Claude Code、OpenHands、Gemini CLI、DeepSeek-TUI、Kimi、CodeWhale 都可以作为执行层示例。

## Core

AgentPal Core 负责连接、记录、展示和回收，不负责直接替用户执行开发任务。

## Atlas 的判断

开发方案必须说明：谁判断、谁执行、谁验收、谁记录。如果这些职责混在 Core 中，Atlas 应要求拆开。
