# Feature Implementation Runbook

## Trigger

A bounded feature request is ready for implementation planning.

## Inputs

- Feature goal.
- User scenario and acceptance criteria.
- Project files and patterns.
- Allowed and forbidden files.
- Verification needs.

## Steps

1. Confirm feature behavior and non-goals.
2. Identify implementation stages.
3. Map files and tests.
4. Create Runtime Task Package.
5. Require implementation evidence and checks.
6. Review result against acceptance criteria.

## Decision points

- Is product scope complete?
- Is design or writing needed?
- Are data or API contracts impacted?
- Is rollout or release review needed?

## Outputs

Implementation plan, Runtime Task Package, test plan, and evidence review.

## Quality checks

- Feature acceptance is observable.
- Scope is bounded.
- Tests and manual checks are named.
- Risks and rollback notes are included.

## Required user confirmation

Required before database/schema changes, dependency changes, external service use, broad refactor, or release action.

## Evidence to return

Return changed files, behavior evidence, tests/checks, docs updates if relevant, not-run items, and risk notes.
