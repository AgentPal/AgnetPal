# Agent Debug Handoff Runbook

这个 Runbook 用于把 Debug 子任务交给 Claude Code、Codex、OpenHands 或其他外部 Runtime。

Atlas 不自己改代码、插入日志、运行程序、运行命令或清理调试代码。Atlas 负责准备上下文、限制范围、要求 evidence，并审查结果。

## When To Use

- 有报错日志、截图、测试失败或用户可复现路径。
- 问题需要执行层读取仓库、运行测试或修改代码。
- 当前任务需要假设、复现、定位、最小修复和回归验证。

## Required Context

交接前收集：

- 仓库路径。
- 失败现象。
- 复现步骤。
- 错误日志、截图或测试输出。
- 最近改动范围。
- 允许修改的文件范围。
- 明确禁止修改的区域。
- 可运行的测试或验证方式。
- 期望完成报告格式。

## Handoff Structure

```text
## Debug 目标

## 失败现象

## 必读资料

## 允许修改范围

## 禁止事项

## 复现步骤

## 定位要求

## 最小修复要求

## 回归测试

## 建议模型与推理强度

## 完成汇报格式
```

## Agent Instructions

要求外部 Runtime：

1. 先复现或解释为什么无法复现。
2. 提出 1-3 个可验证假设。
3. 优先找最小根因，不做无关重构。
4. 如需插入日志或临时代码，必须标出文件、目的和清理方式。
5. 修复后运行回归测试或给出未运行原因。
6. 返回 changed files、测试结果、剩余风险和回滚建议。

## Weak Model Prompting

弱模型或低成本模型不能只收到“修一下”。提示词必须写清：

- 文件范围。
- 错误线索。
- 禁止事项。
- 停止条件。
- 验收命令。
- evidence 格式。

## Stop Conditions

外部 Runtime 应停止并报告：

- 无法复现且缺少日志。
- 需要安装依赖或访问敏感系统。
- 需要修改权限、凭据、真实数据或发布配置。
- 修复会超出允许文件范围。

不允许编造复现结果或测试结果。

## Evidence Review

Atlas 审查返回结果：

- 是否真的复现或合理说明无法复现。
- 是否解释根因。
- 是否只做最小修复。
- changed files 是否在允许范围内。
- 回归测试是否通过。
- 临时代码是否清理。
- 是否还有风险或人工确认项。
