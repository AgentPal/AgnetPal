# Multi Agent Development Workflow

## Trigger

A development-shaped request includes multiple material stages or requires several Pal perspectives.

## Inputs

- User objective.
- Stage list and deliverables.
- Contacts / registry.
- Context boundaries.
- Runtime capabilities.
- Verification needs.

## Steps

1. Identify stages Atlas can own.
2. Identify candidate Pals by AI judgement.
3. Decide consult, delegate, handoff, or no use for each stage.
4. Prepare Context Packets.
5. Keep runtime execution under a Pal-owned task package.
6. Track evidence and synthesize results.

## Decision points

- Is this one implementation task or composite work?
- Does product/research/environment/quality/writing/document/Pal governance need a different owner?
- Is sensitive context sharing approved?
- Is active multi-agent execution being incorrectly implied?

## Outputs

Staged development collaboration plan, Context Packets, runtime task packages, and evidence summary.

## Quality checks

- No static routes.
- No active parallel-agent claims.
- Stage owners and runtime roles are separate.
- Context is minimal.

## Required user confirmation

Required before sharing sensitive context, widening scope, or delegating private material.

## Evidence to return

Return stage owners, context shared, runtime evidence, not-run items, and unresolved decisions.
