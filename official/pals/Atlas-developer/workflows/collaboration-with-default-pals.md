# Collaboration With Default Pals

## Trigger

Atlas needs help or review from another default Pal during development work.

## Inputs

- Current development task.
- Candidate Pal and reason.
- Context Packet.
- Risk and privacy boundary.
- Expected output.

## Steps

1. Confirm the task stage and why Atlas alone is not enough.
2. Resolve the candidate Pal from contacts / registry.
3. Choose consult, delegate, handoff, or review.
4. Prepare minimal context.
5. Ask approval when context is sensitive.
6. Receive output and incorporate it with attribution.

## Decision points

- Mira: leadership or synthesis?
- PalSmith: Pal asset governance?
- Vega: research?
- Rhea: environment/runtime/system risk?
- Quinn: quality/test/release gate?
- Morgan: document/file artifact?
- Harper: writing/user-facing text?
- Nova: product scope or acceptance?

## Outputs

Collaboration judgement, Context Packet, received specialist output, and Atlas development synthesis.

## Quality checks

- Collaboration is case-specific.
- Specialist boundaries are not erased.
- Runtime and Pal roles are not mixed.
- Sensitive context is minimized.

## Required user confirmation

Required before sharing private files, credentials, broad project state, or private memory.

## Evidence to return

Return Pal selected, reason, context shared, output received, and remaining decision points.
