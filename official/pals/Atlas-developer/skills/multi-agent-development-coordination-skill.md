# Multi Agent Development Coordination Skill

## Purpose

Help Atlas coordinate development-shaped work with other Pals and runtimes without pretending AgentPal v0.1 has active multi-agent execution.

## When to use

Use when a development task has research, product, environment, quality, Pal asset, documentation, or writing stages outside Atlas's direct planning scope.

## Inputs

- Composite user goal.
- Material stages and final deliverables.
- Contacts / registry.
- Runtime capability and execution limits.
- Context boundaries and verification needs.

## Required context

Read contacts / registry and the smallest relevant owner Pal guidance. Do not load every Pal Pack. Treat all collaborators as case-specific candidates.

## Process

1. Split the task into material stages.
2. Identify which stages Atlas owns: implementation planning, task package, file boundary, review, verification design.
3. Identify candidate Pals: Mira for coordination, Vega for research, Rhea for environment, Quinn for quality, PalSmith for Pal assets, Nova for product, Morgan for documents, Harper for writing.
4. Prepare Context Packets for consult, delegate, or handoff.
5. Keep runtime execution under the stage owner task package.
6. Synthesize development evidence without erasing other owners.

## Output

A staged collaboration plan, Context Packet, or development coordination report.

## Quality bar

Atlas must not claim active parallel child agents or route by keywords. Collaboration choices must fit the current task and registry.

## User confirmation points

Ask before sharing sensitive context, broad project files, private memory, or changing owner scope.

## No-code boundary

This skill is Pal-layer coordination guidance. It does not create subagents or workflows in the runtime layer.

## Common mistakes

- Treating all implementation-shaped work as Atlas-only.
- Making a fixed collaborator map into a route table.
- Claiming concurrent agent execution.
- Passing too much context to another Pal.

## Example

For a Pal Pack code/docs update, Atlas consults PalSmith for asset governance, Quinn for quality gates, and keeps only the runtime edit package and evidence review stage.
