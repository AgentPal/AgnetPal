# requirement-to-agent-task

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：需求转 Agent 任务

## 用途

把需求、Bug 或技术问题转成 Codex、Claude Code、OpenHands、Kimi、DeepSeek、Gemini CLI 等执行层可复制执行的开发任务提示词。

## 适用场景

- 用户已有清楚目标，希望交给执行层开发。
- 产品说明需要转成工程任务。
- Bug 分析后需要形成修复任务。
- 弱模型或低成本模型需要更完整的任务约束。

## 不适合场景

- 产品范围未定。
- 没有仓库或文件范围。
- 用户要求绕过审批或直接执行高风险动作。

## 输入信息

- 目标。
- 背景和必读资料。
- 仓库路径。
- 修改范围。
- 禁止事项。
- 验收标准。
- 可用 Runtime、模型、推理强度、Skill、插件和工具。

## 处理流程

1. 压缩目标为清楚的开发目标。
2. 列出必读资料和文件范围。
3. 明确本轮只做什么、不做什么。
4. 选择建议 Runtime、模型和推理强度。
5. 写入测试建议、风险、完成报告格式和 evidence 要求。
6. 对弱模型补充更具体的步骤、文件范围和停止条件。

## 输出格式

```text
## 目标
## 必读资料
## 修改范围
## 禁止事项
## 实现建议
## 验收标准
## 测试建议
## 建议执行层
## 建议模型与推理强度
## 任务长度
## 完成汇报格式
```

## 验收标准

- 执行层不需要再猜目标。
- 文件范围和禁止事项明确。
- 验收标准可操作。
- 明确要求 evidence。

## 风险边界

任务提示词不得要求执行层越权、绕过审批、安装未知依赖、删除文件或直接发布到远程仓库。高风险动作必须拆成计划和审批步骤。
