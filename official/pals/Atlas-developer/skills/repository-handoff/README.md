# repository-handoff

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：仓库交接

## 用途

为执行层准备仓库级开发交接包，让 Codex、Claude Code、OpenHands 或外部 Runtime 快速理解项目上下文和边界。

## 适用场景

- 新执行线程需要接手仓库。
- 用户要把任务交给外部 Runtime。
- 当前项目有多份必读文档。
- 需要明确不可修改文件和测试命令。

## 不适合场景

- 用户只是咨询概念。
- 任务没有仓库或文件目标。
- 用户要求把未授权第三方内容打包发布。

## 输入信息

- 项目路径。
- 当前目标。
- 必读文档。
- 相关文件。
- 禁止修改文件。
- 测试、构建、格式化命令。
- 已知问题、风险和期望产物。

## 处理流程

1. 用最短说明描述项目和当前目标。
2. 列出必须先读的文件，而不是让执行层读全仓库。
3. 写清可修改范围和禁止事项。
4. 写清验证命令、手工检查和完成报告格式。
5. 标注高风险动作必须等待审批。
6. 要求执行层返回 evidence。

## 输出格式

```text
## 项目路径
## 当前目标
## 必读资料
## 相关文件
## 禁止修改
## 建议执行步骤
## 验证命令
## 已知风险
## 期望产物
## 完成汇报格式
```

## 验收标准

- 执行层能根据交接包开始工作。
- 没有泄露私人记忆或无关上下文。
- 不要求执行层做越权动作。

## 风险边界

仓库交接不能包含密钥、token、私人记忆、真实用户日志或无关内部资料。执行命令只是建议，由 Runtime 在权限内执行。

## 与 Atlas 其他 Skill / Runbook 的关系

- `context-engineering` 负责压缩交接上下文。
- `source-driven-development` 指定需要查证的来源。
- `security-and-hardening` 检查敏感信息。
