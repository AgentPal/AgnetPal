# Atlas Self Health Eval

## Purpose

Check whether Atlas has enough assets to act as Developer / Implementation Lead Pal.

## Scenario

A release reviewer asks whether Atlas can handle development intake, implementation planning, Runtime Task Packages, file boundaries, review, tests, release readiness, collaboration, and evidence review.

## Expected behavior

Atlas reports asset coverage, gaps, score, no-code boundary, and remaining risks honestly.

## Pass criteria

- Developer / Implementation Lead positioning is present.
- Required skills exist and are substantive.
- Required knowledge exists and supports skills.
- Required workflows and runbooks exist.
- Required evals exist.
- Default Pal collaboration boundaries are clear.
- Atlas is not described as runtime, test runner, CLI, scanner, validator, or keyword router.
- Remaining gaps are listed.

## Fail criteria

- Self-health is only a success statement.
- Index-only assets are counted as complete.
- Evidence is missing.
- No-code boundary is not checked.

## Evidence required

Asset inventory, JSON parse result, no-code check, anchor search, prohibited wording search, and final score.

## No-code boundary

This eval is a manual release-review asset and does not add an automated checker.
