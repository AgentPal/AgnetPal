# Fact Checking Request Example

用户请求：

```text
网上有人说某个 AI 工具已经支持本地多 Runtime 调度，你帮我确认是否真实。
```

## Vega 输出示例

### 待核查断言

- 断言：某个 AI 工具已经支持本地多 Runtime 调度。
- 需要明确：工具名称、版本、平台、是否本地运行、multi-runtime 的定义。
- 时间要求：必须查当前来源。

### 需要查的来源

| 来源类型 | 目的 |
|---|---|
| 官方文档 | 是否声明本地多 Runtime 调度 |
| Release / Changelog | 功能是否已发布 |
| GitHub Issue / Discussion | 用户是否实际使用或遇到限制 |
| 第三方评测 | 是否有独立验证 |
| 反证来源 | 是否只是 roadmap、preview 或误读 |

### 事实 / 推断 / 建议

- 事实：只有来源直接支持的内容。
- 推断：如果多个来源暗示但未明确说明，标注为推断。
- 建议：如果证据不足，建议继续查证或逐案请从当前 contacts / registry 解析出的合适协作对象补充工程可行性评估。

### 不确定项

- “本地”是否指无云端服务。
- “多 Runtime 调度”是否只是多个子任务或真正协作调度。
- 功能是否只在某个版本、平台或付费计划可用。

### 结论格式

```text
## 核查结论
## 已确认事实
## 支持来源
## 反证 / 冲突
## 不确定项
## 建议下一步
```
