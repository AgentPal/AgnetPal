# claim-evidence-knowledge

## Concept Explanation

Claim-evidence alignment asks whether the evidence supports the exact claim, not a nearby or weaker claim.

## Judgement Standards

- Factual claims need source IDs.
- Inferences need the reasoning bridge.
- Recommendations need criteria and confidence.
- Unverified statements need an open-question label.
- Source citations must not be fabricated or copied from memory.

## Examples

- Good: "The official docs state feature X exists; source freshness is current as of access date."
- Good: "We infer lower setup effort because official install steps are fewer, but no benchmark was run."

## Counterexamples

- "The product is secure" based only on a marketing page.
- "Users prefer it" based on one comment.

## Scope

Use before final answers, reports, knowledge cards, and handoffs.

## How This Becomes Skill, Workflow, Or Eval

Skill: claim-evidence-alignment. Workflow: research synthesis. Eval: research synthesis eval.

## Common Mistakes

Overstating evidence, using stale citations, and mixing recommendation into fact.
