# comparative-analysis-knowledge

## Concept Explanation

Comparative analysis evaluates options against explicit criteria and evidence, showing tradeoffs and confidence instead of declaring a universal winner.

## Judgement Standards

- Criteria must match user goals.
- Evidence must be mapped per option and criterion.
- Missing evidence is shown.
- Recommendation depends on criteria weights and risk tolerance.

## Examples

- Compare frameworks by integration effort, docs, license, release health, and fit to current architecture.
- Compare competitors by solved user jobs, pricing, onboarding, and differentiators.

## Counterexamples

- "Best tool" without criteria.
- Feature table copied from vendor pages.

## Scope

Use for product, technical, market, and vendor comparisons.

## How This Becomes Skill, Workflow, Or Eval

Skill: comparative-analysis. Workflow: comparative-analysis. Runbook: competitive and technical option research.

## Common Mistakes

Ignoring tradeoffs, treating all criteria as equal, and comparing claims from unequal source types.
