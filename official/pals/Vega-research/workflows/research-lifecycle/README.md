# Research Lifecycle Workflow

这是 Vega 自有研究生命周期 Workflow，用于把模糊的信息问题推进到可验证、可追溯、可复用的研究结果。

## When To Use

- 用户提出技术调研、开源项目筛选、竞品分析或方案对比。
- 问题需要多个来源和时间判断。
- 研究结果需要协作时，应从当前中央 Pal roster 解析合适协作对象逐案补充。

## Workflow

### 1. Define Question

明确用户真正要判断的问题、输出用途和决策场景。问题太宽时先追问，不直接搜索泛词。

### 2. Scope Research

划定研究范围、时间范围、地区 / 版本限制、必须排除的内容和高风险领域。

### 3. Discover Sources

设计来源计划：官方来源、原始仓库、文档、Release、Issue、第三方评测、社区反馈、用户评价、反证来源。

### 4. Verify Sources

评估来源类型、发布时间、更新时间、作者或机构、商业利益、证据强度、交叉支持和局限。

### 5. Synthesize Findings

将材料整理为事实、推断、建议和不确定项。不能把单一来源或营销材料写成强结论。

### 6. Compare Options

按用户目标比较方案、项目或竞品。复杂技术方案的工程判断应由当前 owner 从中央 Pal roster 逐案解析合适协作对象。

### 7. Write Report

输出结论先行的研究简报：背景、主要发现、证据、反证、不确定点、风险、建议和下一步。

### 8. Create Knowledge Card

可复用结论生成知识卡候选，记录来源、更新时间、适用范围、不确定性和建议审核人。

### 9. Handoff To Another Pal

需要产品判断、工程判断、系统知识维护或风险验收时，生成 Context Packet，并只交给标准 Pal Pack。

## Multi-Step Research Orchestration

复杂研究任务可以拆成多个受控步骤：

1. 来源发现：中低成本模型或具备搜索能力的 Runtime 可做初筛。
2. 来源验证：可靠模型检查官方来源、时间、License、反证和冲突。
3. 综合分析：强模型处理多来源冲突、方案权衡和不确定性。
4. 报告写作：中等模型可整理结构，强模型做最终复验。
5. 知识卡：只沉淀来源清楚、时间清楚、适用范围明确的结论。

Vega 可以委托外部 Runtime 做资料收集，但不能让外部 Runtime 直接替用户下最终结论。弱模型只适合做来源初筛、格式整理和表格填充。

## Completion Criteria

- 研究问题清楚。
- 来源类型和时间要求明确。
- 事实、推断、建议、不确定项分开。
- 有证据表或来源摘要。
- 结论不过度外推。
- 知识卡候选保留来源和更新时间。
