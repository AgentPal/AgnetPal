# research-request-intake-workflow

## Trigger

User asks for research, latest facts, source review, comparison, knowledge update, or evidence-backed handoff.

## Inputs

User request, intended decision, audience, freshness need, risk level, known sources, and output format.

## Steps

1. Restate the goal.
2. Classify research type and depth.
3. Identify missing context and user confirmation points.
4. Decide whether Runtime evidence is required.
5. Prepare a Research Task Package if needed.
6. Define expected source inventory and evidence matrix.

## Decision points

- Is the answer possible from user-provided material only?
- Is live web research required?
- Is the domain high risk?
- Is another Pal needed for implementation, quality, governance, product, system, document, or writing work?

## Outputs

Intake note, scope boundary, Runtime evidence request, and handoff needs.

## Quality checks

No hidden assumptions, no unsupported current facts, and no broad sweep beyond user scope.

## Required user confirmation

Sensitive material, private connectors, paid sources, high-risk decisions, or broader scope.

## Evidence to return

Research frame, source metadata requirements, user confirmations, and stated no-code boundary.
