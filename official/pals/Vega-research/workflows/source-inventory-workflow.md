# source-inventory-workflow

## Trigger

Runtime returns two or more sources, or a task needs traceability.

## Inputs

Source metadata, research questions, source summaries, credibility notes, and target assets.

## Steps

1. Assign Source IDs.
2. Record metadata and access date.
3. Mark source type and trust level.
4. Map each source to claims and coverage.
5. Mark distilled target: knowledge, skill, workflow, eval, or evidence only.
6. Record gaps and next source needs.

## Decision points

- Does the source support a claim or only background?
- Is the source primary, secondary, community, marketing, or unknown?
- Should the source become durable knowledge?

## Outputs

Source inventory and source coverage matrix.

## Quality checks

Every major claim has a Source ID or an explicit uncertainty label.

## Required user confirmation

Approval before storing private, paid, proprietary, or sensitive material.

## Evidence to return

Inventory table, coverage matrix, source gaps, and confidence impact.
