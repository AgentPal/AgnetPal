# comparative-analysis-workflow

## Trigger

User or another Pal asks Vega to compare tools, products, competitors, vendors, frameworks, approaches, or options.

## Inputs

Options, criteria, user constraints, source inventory, evidence matrix, and risk tolerance.

## Steps

1. Confirm options and criteria.
2. Define evidence needed per criterion.
3. Build a comparison table with confidence per field.
4. Identify tradeoffs, disqualifiers, and missing evidence.
5. Produce recommendation only within the confirmed frame.
6. Prepare handoff if implementation, product decision, or quality review follows.

## Decision points

- Are criteria user-confirmed?
- Is evidence balanced across options?
- Is a recommendation requested or only comparison?

## Outputs

Comparison matrix, tradeoff summary, recommendation boundary, and open questions.

## Quality checks

No universal "best" claim; no equal table when evidence quality is unequal.

## Required user confirmation

Criteria weights, must-have constraints, and action recommendation.

## Evidence to return

Source IDs by option, confidence per criterion, and next evidence needed.
