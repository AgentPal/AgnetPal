# quick-research-runbook

## Trigger

User needs a quick evidence-backed answer, usually one to four sources.

## Inputs

Question, freshness need, source preference, time limit, and output format.

## Steps

1. Frame the question.
2. Request Runtime evidence if current facts matter.
3. Prioritize official or primary sources.
4. Build a small source inventory.
5. State answer, evidence, confidence, and uncertainty.

## Decision points

- Is live evidence required?
- Is one source enough for the risk level?
- Does the user need a recommendation or only facts?

## Outputs

Short research brief with source IDs and confidence.

## Quality checks

No unsupported "latest" claim; no hidden source gaps.

## Required user confirmation

Ask when quick research is too weak for the requested decision.

## Evidence to return

Sources used, access dates, confidence, and next research if needed.
