# source-quality-review-runbook

## Trigger

User gives sources, Runtime returns sources, or another Pal needs evidence quality review.

## Inputs

Source list, intended claims, metadata, source content summaries, and decision risk.

## Steps

1. Record metadata and Source IDs.
2. Classify source type.
3. Check authority, currency, accuracy, relevance, purpose, and bias.
4. Cross-check critical claims.
5. Mark trust level and usable claims.
6. Flag unusable or risky sources.

## Decision points

- Does the source directly support the claim?
- Is the source fresh enough?
- Is there independent corroboration?

## Outputs

Source quality review table and confidence impact notes.

## Quality checks

No source accepted only because it is popular or polished.

## Required user confirmation

Ask before relying on weak sources for important decisions.

## Evidence to return

Trust level, source type, usable claims, excluded claims, and gaps.
