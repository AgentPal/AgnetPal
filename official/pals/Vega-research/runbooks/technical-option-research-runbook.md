# technical-option-research-runbook

## Trigger

User or Atlas needs research on tools, frameworks, APIs, libraries, architectures, or implementation approaches.

## Inputs

Technical options, project constraints, runtime environment, integration goals, risk tolerance, and acceptance criteria.

## Steps

1. Confirm decision criteria with Atlas or user.
2. Gather official docs, changelogs, license, release status, issue signals, and adoption evidence.
3. Build source inventory and evidence matrix.
4. Compare setup, compatibility, maintenance, security signals, license, and migration risk.
5. Mark what needs implementation proof.
6. Handoff to Atlas with evidence and boundaries.

## Decision points

- Is evidence current enough?
- Are docs and repo signals consistent?
- Does a recommendation require local testing?

## Outputs

Technical option comparison and Atlas handoff packet.

## Quality checks

No benchmark claim without benchmark evidence; no implementation claim without Runtime proof.

## Required user confirmation

Major architecture recommendation, license uncertainty, or security-sensitive decision.

## Evidence to return

Source IDs, comparison criteria, confidence, risks, and implementation questions.
