# knowledge-card-curator

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：知识卡沉淀

## 用途

把研究结果转成可写入系统层、Pal 层或项目层的知识卡候选。

## 适用场景

- 调研结果未来会复用。
- 技术选型结论需要保留。
- GitHub 项目评估可作为后续参考。
- 研究结果需要由当前 AI / Mira / Brain 从 contacts / registry 解析合适审核对象。

## 不适合场景

- 临时搜索结果。
- 未验证单一来源。
- 版权资料全文。
- 用户私人资料或敏感资料。

## 输入信息

- 研究简报。
- 来源摘要。
- 适用范围。
- 不确定项。
- 建议写入层级。

## 处理流程

1. 判断写入位置：系统层、Pal 层或项目层。
2. 写标题和摘要。
3. 记录来源、更新时间、适用范围。
4. 写关键结论和不确定性。
5. 标注维护建议和审核人。

## 输出格式

```text
# Knowledge Card Candidate
## 建议写入位置
## 标题
## 摘要
## 适用范围
## 来源
## 更新时间
## 关键结论
## 不确定性
## 建议审核人
```

## 验收标准

- 不保存版权全文。
- 来源和更新时间清楚。
- 适用范围明确。
- 不确定性未被抹掉。

## 风险边界

知识卡候选不是自动写入正式知识库。系统知识建议由当前 AI / Mira / Brain 按 contacts / registry 逐案选择审核对象。
