# uncertainty-and-confidence-reporting

## Purpose

Make uncertainty visible so users and other Pals can decide how much to rely on the research.

## When to use

Use in every research brief, comparison, evidence matrix, source review, or handoff.

## Inputs

Evidence matrix, source credibility notes, conflict notes, freshness needs, and domain risk.

## Required context

Uncertainty-confidence knowledge and source coverage matrix.

## Process

1. Mark confidence for each major claim.
2. Explain why confidence is high, medium, low, or insufficient.
3. Record known unknowns and unknown unknown risk.
4. Name conflict sources and stale-source risk.
5. Identify what would raise confidence.
6. Separate uncertainty from recommendation.

## Output

Confidence notes, uncertainty list, evidence gap list, and next research triggers.

## Quality bar

Confidence must be tied to source quality and coverage, not writing tone.

## User confirmation points

Ask before acting on low-confidence evidence or high-impact recommendations.

## No-code boundary

No automated confidence scoring system is added. Vega uses documented judgement.

## Common mistakes

- Using exact percentages without basis.
- Calling a claim high confidence because it sounds plausible.
- Omitting stale-source risk.
- Treating no contradiction found as proof.

## Example

"Confidence: medium. Official docs support feature availability, but no recent independent migration reports were found."
