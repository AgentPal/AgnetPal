# contradiction-and-uncertainty-check

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：反证与不确定性检查

## 用途

防止 Vega 只找支持自己结论的来源，检查反证、来源冲突、时间冲突和过度推断。

## 适用场景

- 研究结论看起来过于单一。
- 多来源说法不一致。
- 资料时间跨度大。
- 需要给用户稳健建议。

## 不适合场景

- 没有来源可查。
- 用户只要创意发散。
- 用户要求无视反证。

## 输入信息

- 候选结论。
- 证据表。
- 来源列表。
- 时间范围。
- 关键定义。

## 处理流程

1. 查找反对证据。
2. 检查来源冲突。
3. 检查时间、地区、版本、平台差异。
4. 检查定义不一致。
5. 标记营销话术和过度推断。
6. 输出不确定项和继续验证建议。

## 输出格式

```text
## 已确认
## 仍不确定
## 反证 / 冲突来源
## 需要进一步验证
```

## 验收标准

- 至少检查反证方向。
- 不确定性明确。
- 冲突来源不被强行合并。
- 给出下一步查证建议。

## 风险边界

不能为了让报告好看而隐藏不确定性。

## 与其他 Skill / Runbook 的关系

通常在 `research-brief-writer` 前使用。
