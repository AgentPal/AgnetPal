# comparative-analysis

## Purpose

Compare options using explicit criteria, evidence, confidence, tradeoffs, and decision boundaries.

## When to use

Use for product, market, competitor, technical, tool, framework, vendor, or approach comparisons.

## Inputs

Options, decision criteria, user constraints, source inventory, evidence matrix, and risk tolerance.

## Required context

Comparative analysis knowledge, evidence matrix, and collaboration boundary.

## Process

1. Confirm options and decision criteria.
2. Define scoring labels or qualitative comparison fields.
3. Map evidence to each criterion.
4. Separate measurable facts from judgement.
5. Identify tradeoffs, unknowns, and disqualifiers.
6. Produce recommendation only within the confirmed decision frame.

## Output

Comparison table, tradeoff notes, confidence by criterion, recommendation, and open questions.

## Quality bar

The comparison must not use equal-looking tables to hide unequal evidence quality.

## User confirmation points

Confirm criteria weight, must-have constraints, and whether a recommendation is wanted.

## No-code boundary

Vega compares evidence. It does not implement, benchmark, or run tests.

## Common mistakes

- Comparing options on criteria the user did not value.
- Mixing vendor claims with measured evidence.
- Ignoring switching cost.
- Hiding missing data.

## Example

Option A wins on setup and docs; Option B wins on extensibility; confidence is medium because integration evidence is mostly official documentation.
