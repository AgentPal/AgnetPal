# evidence-matrix

## Purpose

Build a matrix that aligns claims with evidence, source IDs, confidence, contradictions, open questions, and next research.

## When to use

Use when a research answer will influence a recommendation, comparison, knowledge update, or handoff.

## Inputs

Claims, source inventory, credibility notes, user decision criteria, contradictions, and gaps.

## Required context

Evidence matrix knowledge, claim-evidence knowledge, and source coverage matrix.

## Process

1. List each claim as a testable statement.
2. Attach source IDs and evidence summaries.
3. Add confidence: high, medium, low, or not enough evidence.
4. Record contradictions and source conflict.
5. Record open questions.
6. Suggest next research needed.
7. Mark whether the claim is fact, inference, recommendation, or uncertainty.

## Output

Matrix columns: Claim, Evidence, Source ID, Confidence, Contradictions, Open questions, Suggested next research.

## Quality bar

The matrix must make unsupported claims visible instead of hiding them in fluent prose.

## User confirmation points

Ask before converting low-confidence claims into action recommendations.

## No-code boundary

This is an evidence structure, not a validator or scoring program.

## Common mistakes

- Writing conclusions first and searching support later.
- Using one source for all claims.
- Hiding contradictions.
- Treating confidence as certainty.

## Example

Claim: "Tool A has lower setup risk." Evidence: official install docs plus recent issue tracker data. Confidence: medium if issue data is mixed.
