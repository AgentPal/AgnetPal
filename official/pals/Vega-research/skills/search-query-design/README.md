# search-query-design

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：搜索式设计

## 用途

帮助 Vega 为 Runtime 或外部 Runtime 设计高质量搜索查询，而不是只搜索一个泛词。

## 适用场景

- 查 GitHub 项目。
- 查竞品、官方文档、最新发布、报错线索或维护状态。
- 需要反证搜索或时间限定搜索。

## 不适合场景

- 用户已经提供足够来源。
- 不允许使用外部搜索。
- 涉及用户隐私或内部资料，且未确认能外发。

## 输入信息

- 研究问题。
- 核心实体。
- 同义词或英文名。
- 来源类型。
- 时间范围。
- 反证方向。

## 处理流程

1. 抽取核心实体。
2. 补充同义词、中英文关键词和版本词。
3. 拆成事实查询、对比查询、反证查询、时间查询。
4. 按来源限定站点或类型。
5. 给每组查询标注目的。

## 输出格式

```text
## 查询组 A：官方来源
## 查询组 B：GitHub / 开源项目
## 查询组 C：第三方评测
## 查询组 D：用户反馈 / 争议
## 查询组 E：反证和风险
```

## 验收标准

- 查询组覆盖主要来源类型。
- 每个查询有明确目的。
- 包含反证和风险方向。
- 不泄露用户敏感资料。

## 风险边界

Vega 不直接执行搜索。搜索式只是交给 Runtime 或外部 Runtime 的任务包。

## 与其他 Skill / Runbook 的关系

由 `deep-research-plan` 调用，结果交给 `source-quality-check` 和 `evidence-table-builder`。
