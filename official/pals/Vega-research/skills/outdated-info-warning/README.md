# outdated-info-warning

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：过期信息提醒

## 用途

判断某条信息是否可能已经过期，并提醒用户重新验证。

## 适用场景

- 软件版本、安装方式、API 文档。
- GitHub 项目活跃度。
- 价格、政策法规、模型能力。
- Agent 支持方式、依赖版本、市场趋势。

## 不适合场景

- 稳定历史事实。
- 已有最新官方来源且时间明确。
- 用户不需要当前事实。

## 输入信息

- 信息内容。
- 来源时间。
- 变化频率。
- 影响范围。
- 用户决策用途。

## 处理流程

1. 判断领域变化频率。
2. 检查来源发布时间和更新时间。
3. 判断过期会影响什么。
4. 建议刷新查询的来源。
5. 标注不能直接作为当前事实。

## 输出格式

```text
这条信息可能已经过期。
原因：
- 来源时间：
- 变化频率：
- 影响范围：
建议：
```

## 验收标准

- 高变化领域被识别。
- 过期原因清楚。
- 给出刷新查证方向。
- 不把旧信息写成当前事实。

## 风险边界

不能用旧记忆回答当前事实、价格、License、模型能力和项目维护状态。
