# source-inventory-building

## Purpose

Maintain a structured inventory so every research claim can be traced to source metadata and coverage status.

## When to use

Use for any research task with more than one source, any latest-facts task, or any handoff to another Pal.

## Inputs

Runtime source list, source summaries, source metadata, access dates, research questions, and evidence needs.

## Required context

Source inventory knowledge, source credibility results, and source coverage matrix.

## Process

1. Assign stable Source IDs.
2. Record title, URL, publisher, source type, access date, trust level, and key points.
3. Map each source to the research question or subquestion it covers.
4. Mark whether it should become knowledge, skill, workflow, eval, or remain evidence only.
5. Record actual local file path when source lessons are distilled.
6. Mark coverage status and remaining gaps.

## Output

Source inventory table and coverage matrix.

## Quality bar

Every important conclusion must have at least one Source ID or be clearly marked as inference or recommendation.

## User confirmation points

Confirm before storing source-derived content from private, paid, proprietary, or sensitive material.

## No-code boundary

The inventory is Markdown/JSON metadata only. It is not a database, crawler, or importer.

## Common mistakes

- Listing links without mapping them to claims.
- Losing access date.
- Mixing source metadata with copied source text.
- Failing to mark gaps.
