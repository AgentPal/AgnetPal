# research-synthesis

## Purpose

Turn evidence into a concise research answer that separates fact, inference, recommendation, uncertainty, and user decision needs.

## When to use

Use after source inventory and evidence matrix exist.

## Inputs

Research question, evidence matrix, source credibility notes, contradictions, gaps, and desired output form.

## Required context

Research synthesis knowledge, uncertainty-confidence knowledge, and output contract.

## Process

1. Start with the answer in brief.
2. Separate facts from inferences and recommendations.
3. Explain source coverage and confidence.
4. Highlight contradictions and uncertainty.
5. State what the user must decide.
6. Suggest next research only where it changes the decision.

## Output

Research brief with sections: answer, facts, inferences, recommendation, uncertainty, source coverage, and next step.

## Quality bar

The synthesis must be more than a summary. It must show how evidence changes the answer.

## User confirmation points

Ask before making a recommendation in a high-risk domain or where evidence is low.

## No-code boundary

Vega writes synthesis from evidence returned by Runtime. It does not browse or automate collection itself.

## Common mistakes

- Listing source summaries without answering the question.
- Hiding weak evidence.
- Treating inferred fit as fact.
- Omitting source coverage.

## Example

"Recommendation: choose A for a prototype because docs and integration evidence are stronger. Uncertainty: production support claims need direct vendor confirmation."
