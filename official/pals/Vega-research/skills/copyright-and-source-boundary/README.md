# copyright-and-source-boundary

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：版权与来源边界

## 用途

防止 Vega 把外部资料全文、付费内容、未授权材料或敏感资料直接写入公开包或长期知识库。

## 适用场景

- 用户上传 PDF、报告、网页保存或文章。
- 准备把研究资料放进 `imports/`。
- 生成知识卡或研究简报。
- 准备公开发布 Pal Pack。

## 不适合场景

- 用户只问公开来源的简短摘要。
- 来源已经明确允许再分发，且仍需保留 License。

## 输入信息

- 资料来源。
- License 或授权方式。
- 内容类型。
- 是否付费或私密。
- 保存目标。

## 处理流程

1. 判断是否可公开再分发。
2. 区分可保存摘要与不可保存全文。
3. 标注原始 License、NOTICE 或来源要求。
4. 检查是否涉及隐私、凭据或商业机密。
5. 给出保存、索引、摘要或拒绝建议。

## 输出格式

```text
## 可保存
## 不应保存
## 需要保留的来源 / License 信息
## 建议处理方式
```

## 验收标准

- 无授权全文不进入公开包。
- 第三方 License 边界清楚。
- 私密资料不会发给外部 Runtime。
- 摘要和引用不过量。

## 风险边界

不能把第三方资料因为被 Vega 摘要就视为 MIT。无 License 或协议不明内容不应公开打包。

## 与其他 Skill / Runbook 的关系

为 `knowledge-card-curator`、`imports/` 和 `THIRD_PARTY_NOTICES.md` 提供边界。
