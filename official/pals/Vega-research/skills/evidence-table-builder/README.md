# evidence-table-builder

运行时 Skill 入口请看 SKILL.md；README.md 仅作为人类阅读说明。


中文名：证据表构建

## 用途

把研究材料整理成证据表，避免只给用户堆摘要或链接。

## 适用场景

- 多个来源结论不一致。
- 需要比较多个项目或方案。
- 需要判断技术方案是否靠谱。
- 需要输出可追溯研究简报。

## 不适合场景

- 没有可引用来源。
- 只是自由头脑风暴。
- 用户要求直接跳到结论。

## 输入信息

- 研究问题。
- 来源列表。
- 来源质量评估。
- 候选结论。
- 反证或冲突来源。

## 处理流程

1. 抽取候选结论。
2. 为每个结论列支持来源。
3. 查找反对来源或冲突来源。
4. 标注证据强度。
5. 写出不确定点。
6. 给出谨慎建议。

## 输出格式

```text
| 结论 | 支持来源 | 反对来源 | 证据强度 | 不确定点 | 建议 |
|---|---|---|---|---|---|
```

## 验收标准

- 每条结论可追溯到来源。
- 有反证或明确说明未找到反证。
- 证据强度不被夸大。
- 建议与证据强度匹配。

## 风险边界

不能把无来源结论写成事实。证据表不是最终专业意见。
