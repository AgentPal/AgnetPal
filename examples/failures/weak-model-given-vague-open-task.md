# Failure: Weak Model Given Vague Open Task

## Wrong Behavior

The Conductor sends a vague multi-stage research, writing, implementation, and verification goal to a weak/economy model candidate with no file boundaries or acceptance criteria.

## Why Wrong

Weak/economy candidates need detailed packages. Vague open tasks increase rework and false completion.

## Correct Behavior

Either increase reasoning candidate or provide a detailed package with stages, allowed context, forbidden context, steps, acceptance criteria, verification, and final report fields.
