# Failure Example: Context Bloat From All Pals

## Bad Pattern

A runtime reads all Pal Packs, all knowledge, all memory, all examples, and all evals before answering an ordinary task.

## Why It Fails

- creates a token sink
- mixes unrelated roles and memories
- increases privacy risk
- makes asset usage impossible to audit
- slows down useful work

## Better Pattern

Use:

- `RESOURCE_INDEX.md` as navigation
- Context Slicing
- Context Access List
- selected owner Pal assets
- Asset Loading Report when asked

Knowing a path exists is not the same as reading its content.
