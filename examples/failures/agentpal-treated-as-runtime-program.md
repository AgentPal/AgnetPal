# Failure: AgentPal Treated As Runtime Program

## Wrong Behavior

A roadmap says v0.4 will build the AgentPal CLI, scanner, validator, installer, daemon, desktop app, web app, and database so AgentPal can execute tasks itself.

## Why Wrong

AgentPal's default future boundary is no-code. It is a Pal layer, Pal Pack Standard, and AI team framework on top of host Agent Runtimes. Execution belongs to the host Runtime.

## Correct Behavior

The roadmap says v0.4 may expand runtime adapter protocols, host Runtime usage guides, Runtime-installed Skill orchestration examples, cross-runtime memory continuity, and Deep Conductor patterns. It does not make program features the default route.
