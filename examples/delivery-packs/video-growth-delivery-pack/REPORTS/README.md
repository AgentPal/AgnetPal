# Report Notes

## Purpose

This directory is for report shells and evidence placeholders produced during a real delivery.

## Public-safe example boundary

Reusable report templates may be stored here. Real task reports, raw platform analytics, customer materials, private meeting notes, and account-specific evidence do not belong in this public-safe example.

## Suggested report types

- weekly content planning report;
- publication review report;
- blocked material report;
- post-publication learning candidate report;
- training completion note.

## Memory boundary

Reports are not memory by default. A stable lesson can be proposed for reusable memory only after human review and customer-private material removal.

## Status terms

Use `pass`, `partial`, `missing`, `not-run`, `blocked`, and `requires-human-review` without smoothing weaker states into `pass`.
