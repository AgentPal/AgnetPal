# Pal Team Blueprint

## Purpose

This file describes a candidate Pal team for the Video Growth Delivery Pack.

Candidate Pals are AI judgement inputs only. They are not a route map, not keyword routing, not central roster edits, and not automatic task assignments.

## Candidate Pal team

| Candidate Pal | Responsibility | Required assets |
| --- | --- | --- |
| Research Pal | Research audience, competitor accounts, topics, and source uncertainty. | research intake, evidence table, competitor review checklist |
| Scriptwriter Pal | Draft scripts, hooks, outlines, CTAs, and short-video variants. | script style guide, script templates, rewrite method |
| Brand Guardian Pal | Protect tone, claim safety, and forbidden wording. | brand rules, claim boundary checklist, review rubric |
| Material Manager Pal | Track source materials, footage notes, usage rights, and private/public boundary. | material inventory, source map, privacy boundary |
| Video QA Pal | Review scripts, shot brief, captions, accessibility, and publishing readiness. | QA checklist, issue report, acceptance notes |
| Publishing Pal | Prepare platform handoff notes and publishing checklist. | publishing profile notes, handoff checklist |
| Growth Review Pal | Review performance notes and generate learning candidates. | review report template, metric interpretation boundary |

## Collaboration model

Faye designs the project delivery structure and asks PalSmith to create a Pal team skeleton. PalSmith creates or proposes Pal Pack skeletons only in a separate confirmed task with its own evidence.

## No routing rule

This blueprint must not be used as:

- keyword dispatch;
- domain dispatch;
- role dispatch;
- deterministic route table;
- central contacts replacement.

The current Brain / AI must still judge ownership from the user request, central roster, project context, Delivery Pack, Capability Inventory, Pal assets, risk, and evidence needs.

## PalSmith skeleton expectation

If a later approved PalSmith task creates skeletons, each candidate Pal should receive only public-safe placeholder assets until real customer material is classified. No candidate Pal in this file is automatically registered in central contacts.
