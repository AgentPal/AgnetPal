# Cross-Pal PalSmith Create AI Team

This public-safe example shows PalSmith designing an AI team with Mira as possible conductor pattern. It is a staged written package, not automatic Deep Conductor execution.

## User Request

"Create an AI team for a small education business: plan courses, research trends, write copy, manage docs, and verify launches."

## Owner Judgement

Mira may receive the ordinary request and judge that PalSmith is a strong owner candidate because the requested deliverable is an AI team / Pal asset design package. PalSmith then owns the Pal asset governance body.

## Staged Task Judgement

- domain_focus: AI team and Pal asset design.
- content_deliverables: team blueprint, role boundaries, example tasks, eval ideas.
- final_deliverables: Pal creation package and governance notes.
- work_stages:
  - goal clarification: PalSmith candidate.
  - team blueprint: PalSmith candidate.
  - product framing input: Nova consultant candidate.
  - research-source policy: Vega consultant candidate.
  - writing voice input: Harper consultant candidate.
  - document workflow input: Morgan consultant candidate.
  - verification plan: Quinn consultant candidate.
- runtime_candidates: file-editing Runtime only after the creation package is approved.
- verification_needs: no role overlap, no private data, no automatic execution claim.

## Context Access List

| Stage | Candidate | Can read | Cannot read | Need output |
| --- | --- | --- | --- | --- |
| Intake | PalSmith | user goal, constraints, desired team size | private materials not approved | clarification and asset plan |
| Product input | Nova | business goal summary | full user archive | product role needs |
| Research input | Vega | research goal summary | unrelated source lists | source policy |
| Writing input | Harper | audience and voice notes | confidential copy bank unless approved | voice guidance |
| Docs input | Morgan | document workflow needs | unrelated files | doc workflow guidance |
| Verification input | Quinn | proposed team assets | hidden drafts | readiness checks |

## Staged Task Package

- Goal: design an AI team package from the user's goal.
- Constraints: PalSmith does not auto-register assets; Runtime writes require confirmation; candidates are not fixed routes.
- Steps: clarify goal, design Main Pal pattern, define specialist candidate responsibilities, prepare example tasks, prepare eval plan, prepare creation package.
- Output: team blueprint, file plan, confirmation questions, readiness checklist.

## Good Response Example

PalSmith keeps the team design no-code, labels every role as a candidate capability holder, and prepares a controlled creation package with evidence requirements.

## Bad Response Example

PalSmith creates a permanent task assignment table, says the team will run automatically, or edits contacts without a separate confirmed package.

## Verification / Acceptance Criteria

- Team roles have clear boundaries.
- The package avoids static collaboration rules.
- Runtime writes are separated from PalSmith judgement.
- Current v0.2 remains Simple Pal Mode.
