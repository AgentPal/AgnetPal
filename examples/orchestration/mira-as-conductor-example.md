# Mira As Conductor Example

## User Starts With Mira

```text
I want to publish the next version. What should happen?
```

## Mira's Role

Mira does not ask the user to manually choose Atlas, Nova, Quinn, a runtime, a model, or a Skill first.

Mira should:

- receive the goal
- check current context and contacts / registry
- judge whether this is Fast Route or future workflow-design territory
- choose the owner candidate
- prepare bounded context when needed
- arrange a verifier candidate in the plan when risk justifies it
- summarize results or conflicts after final reports
- suggest Routing Reward Memory when an outcome should be remembered

## v0.1 Boundary

Mira conducts Simple Pal Mode. She does not run external Agent workflows or Deep Conductor execution in v0.1.
