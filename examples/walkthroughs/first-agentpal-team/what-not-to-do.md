# What Not To Do

Do not:

- paste real customer data;
- paste tokens, passwords, API keys, cookies, private keys, or secrets;
- connect a CRM, GitHub account, Lark workspace, payment system, or database;
- create a real external project `.agentpal/delivery-pack/` folder;
- copy Pals, memory, reports, Org Packs, FDE Packs, Delivery Packs, governance
  records, or capability inventory into an external project by default;
- treat recommended Pals as fixed routes;
- claim the local walkthrough proves remote publication;
- push, tag, or create a GitHub Release without a separate explicit
  authorization round.
