# Pal Capability Profiles

Pal capability profiles summarize selected Pal capabilities as judgement inputs. They are not replacements for `contacts/pals.json`, `registry/pal.index.json`, `PAL.md`, or the Pal's output contract.

## Examples

- `mira-main.example.json`
- `palsmith-pal-governor.example.json`
- `atlas-developer.example.json`
- `vega-research.example.json`
- `quinn-quality.example.json`

## Rules

- A Pal is a candidate capability holder, not an Agent process.
- A Pal does not directly execute runtime operations.
- A Pal profile is not a fixed route.
- The current owner Pal still performs Task Judgement.
- Use the smallest relevant Pal profile, not all Pal profiles by default.
