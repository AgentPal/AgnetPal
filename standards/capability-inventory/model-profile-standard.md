# Model Profiles

Use profiles to describe model strengths after evidence exists.

## Strong models

Best for:

- architecture-sensitive decisions
- complex debugging
- cross-agent planning
- prompt shaping for weaker models
- final verification

Prompt style: goal, context, constraints, tradeoffs, risks, and evidence requirements.

## Medium models

Best for:

- scoped implementation
- documentation updates
- template filling
- standard bug fixes

Prompt style: explicit goal, file scope, constraints, and acceptance criteria.

## Lower-cost or weaker models

Best for:

- repetitive edits
- deterministic formatting
- clear bounded tasks

Prompt style: strict steps, exact scope, forbidden changes, acceptance commands, and required final report.
