# Capability Consideration Matrix

This file is a capability consideration matrix, not a route map.

No hard-coded semantic routing.

AI routing judgement is case-by-case. Pal capability reference is not a route map.

| Situation | Possible consideration | Model strength | Prompt detail | Notes |
|---|---|---|---|---|
| AgentPal onboarding | Mira can often handle directly | low to medium | low | Explain usage and next step. |
| Direct Pal call | Explicit commands are deterministic | medium | medium | Use `/pal Name` and Context Packet. |
| Multi-perspective review | Owner Pal may consult or request review from other Pals in Simple Pal Mode | high | high | Same-response collaboration; no fixed Pal set. |
| Project workgroup binding | Mira coordination plus execution layer may be needed | medium to high | high | Confirm external path first. |
| Scoped implementation | Specialist owner or execution runtime may be useful | medium | high | Require files, constraints, acceptance. |
| Read-only system audit | A system-oriented perspective may help | high | high | Read-only first; no modification without approval. |
| High-risk operation | Pause for approval | high | high | Safety and availability guardrails are deterministic. |
| Release verification | Evidence-oriented execution is required | high | high | Evidence required. |

All rows are non-binding examples. They are not keyword triggers and do not force a fixed Pal.
