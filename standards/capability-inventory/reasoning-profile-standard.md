# Reasoning Strengths

## low

Use for formatting, simple docs cleanup, JSON validation, and deterministic edits.

## medium

Use for normal documentation, template filling, scoped feature work, and standard bug fixes.

## high

Use for multi-agent planning, architecture-sensitive changes, complex debugging, release blockers, and safety boundary design.

## xhigh

Use only for major architecture redesign, deep failure analysis, or difficult cross-runtime coordination.
