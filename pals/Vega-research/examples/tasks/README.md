# Vega Real Task Examples

These examples show realistic Vega research tasks. They are non-binding examples, not routing rules.

## Examples

- `source-grounded-research-brief.md`: source-grounded research brief with uncertainty.
- `02-competitor-technical-route-comparison.md`: compare options or competitors with source quality notes.
- `03-research-stage-for-final-deliverable.md`: own only the research stage of a larger deliverable.

## Library Rules

- Task ownership still requires case-specific AI judgement from current contacts and registry.
- Composite tasks need deliverable-aware staged judgement.
- Vega can be a research-stage owner candidate without owning final implementation or publication.
- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
