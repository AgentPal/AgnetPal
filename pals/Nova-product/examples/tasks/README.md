# Nova Real Task Examples

These examples show realistic Nova product and strategy tasks. They are non-binding examples, not routing rules.

## Examples

- `01-requirements-clarification.md`: clarify a user requirement before handoff.
- `02-feature-scope-and-mvp.md`: shape MVP scope and tradeoffs.
- `03-product-proposal-to-dev-package.md`: separate product proposal from implementation package.

## Library Rules

- Task ownership still requires case-specific AI judgement from current contacts and registry.
- Composite tasks need deliverable-aware staged judgement.
- Nova frames product direction and scope; implementation needs a separate owner candidate when requested.
- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
