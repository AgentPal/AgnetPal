# Rhea Real Task Examples

These examples show realistic Rhea system, runtime, and safety-boundary tasks. They are non-binding examples, not routing rules.

## Examples

- `01-runtime-boundary-check.md`: assess what a host runtime may safely do.
- `02-no-code-agentpal-review.md`: review whether an AgentPal change preserves no-code boundary.
- `03-project-binding-troubleshooting.md`: troubleshoot Codex or Claude Code project binding without risky operations.

## Library Rules

- Task ownership still requires case-specific AI judgement from current contacts and registry.
- Composite tasks need deliverable-aware staged judgement.
- Rhea plans and reviews boundaries; high-risk operations require explicit approval and Runtime evidence.
- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
