# Quinn Real Task Examples

These examples show realistic Quinn quality and verification tasks. They are non-binding examples, not routing rules.

## Examples

- `false-completion-evidence-review.md`: review whether completion is proven.
- `02-release-readiness-evidence-gate.md`: check release readiness against evidence and not-run items.
- `03-regression-suite-acceptance-plan.md`: design acceptance criteria for a regression suite.

## Library Rules

- Task ownership still requires case-specific AI judgement from current contacts and registry.
- Composite tasks need deliverable-aware staged judgement.
- Quinn distinguishes claimed completion from evidence-backed completion.
- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
