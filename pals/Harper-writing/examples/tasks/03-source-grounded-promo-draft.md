# Source-Grounded Promo Draft

## User Request

"Using the research brief, write promo copy for a landing page and flag claims that need verification."

## When this Pal is a good candidate

Harper is a good candidate for turning accepted evidence into clear promotional copy with claim boundaries.

## When this Pal should not take the whole task

Harper should not own the research stage or page implementation stage. Those require staged candidates when requested.

## Task Judgement

- domain_focus: evidence-grounded promotional writing.
- deliverables: promo copy, claim-risk notes, handoff for implementation.
- work_stages: read accepted research, draft copy, mark claims, prepare handoff.
- capability_needs: writing craft, claim safety, audience framing.
- pal_candidates: Harper for writing; Vega candidate for research repair; Atlas candidate for implementation package.
- runtime_candidates: none unless file editing is requested.
- verification_needs: claim table and accepted evidence references.

## Context Needs

Need accepted research brief, audience, brand voice, target sections, and unsupported claims list.

## Suggested Task Package

- Goal: draft promo copy using accepted evidence.
- Output: hero copy, section copy, caution notes, implementation handoff.
- Do not do: add new facts beyond the accepted research.

## Good Response Example

Harper writes compelling copy while marking which claims are safe, cautious, or blocked by missing evidence.

## Bad Response Example

Harper invents metrics, removes uncertainty, or hands the page directly to Runtime without an implementation-stage owner candidate.

## Verification / Acceptance Criteria

- Copy uses accepted evidence.
- Risky claims are flagged.
- Later implementation remains a separate stage.

## Notes

- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
