# Office PDF Task Package

## User Request

"Prepare a task package to generate a PDF summary from these sections and verify the layout."

## When this Pal is a good candidate

Morgan is a good candidate because the work is document-output planning, source organization, and layout verification requirements.

## When this Pal should not take the whole task

Morgan should not claim the PDF was rendered unless the Runtime actually generated and verified it.

## Task Judgement

- domain_focus: document artifact package.
- deliverables: PDF generation package and layout acceptance criteria.
- work_stages: source mapping, output spec, generation instructions, render verification.
- capability_needs: document workflow, Office/PDF task packaging, visual acceptance.
- pal_candidates: Morgan for document package; Quinn candidate for final evidence review.
- runtime_candidates: document-capable Runtime or relevant document Skill if available.
- verification_needs: generated artifact, rendered pages or equivalent evidence, layout issues.

## Context Needs

Need source sections, target format, page constraints, branding constraints, output path, and verification method.

## Suggested Task Package

- Goal: generate a PDF from approved sections.
- Output spec: sections, order, page style, filename.
- Evidence: generated file path, render/preview evidence, issues list.

## Good Response Example

Morgan gives the Runtime a precise document package and requires render evidence before saying the PDF is complete.

## Bad Response Example

Morgan says the PDF is done after writing Markdown, or omits layout acceptance criteria.

## Verification / Acceptance Criteria

- The package identifies source sections and output artifact.
- Layout evidence is required.
- Not-run render checks remain visible.

## Notes

- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
