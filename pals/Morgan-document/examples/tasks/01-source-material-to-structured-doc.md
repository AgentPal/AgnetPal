# Source Material To Structured Doc

## User Request

"Turn these scattered notes into a structured Markdown guide without losing the important constraints."

## When this Pal is a good candidate

Morgan is a good candidate because the task is document organization, information architecture, and content preservation.

## When this Pal should not take the whole task

Morgan should not invent research facts or write marketing copy beyond the source material. Research or writing stages need separate candidates when needed.

## Task Judgement

- domain_focus: document structure.
- deliverables: structured guide outline and document task package.
- work_stages: source inventory, information architecture, draft structure, preservation check.
- capability_needs: document workflow, content preservation, Markdown structure.
- pal_candidates: Morgan for document structure; Vega candidate if missing facts require research.
- runtime_candidates: file-editing Runtime if writing the file is approved.
- verification_needs: source coverage, omitted items, document acceptance criteria.

## Context Needs

Need supplied notes, target audience, output format, required sections, and private/public boundary.

## Suggested Task Package

- Goal: create a structured guide from approved source notes.
- Steps: inventory notes, group topics, draft outline, write document, verify coverage.
- Output: doc path, source coverage, not-run checks.

## Good Response Example

Morgan preserves constraints, turns notes into readable sections, and reports any missing information instead of filling it with guesses.

## Bad Response Example

Morgan rewrites source constraints into vague prose or adds unsupported facts to make the guide smoother.

## Verification / Acceptance Criteria

- Important source constraints remain visible.
- The structure fits the target audience.
- Missing information is marked, not invented.
