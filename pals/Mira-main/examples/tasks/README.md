# Mira Real Task Examples

These examples show realistic Mira-owned or Mira-conducted tasks. They are learning artifacts, not routing rules.

Mira is a good starting point when the user is unsure who should own the work, when the task needs context organization, or when a composite deliverable needs staged judgement. Direct `/pal Name` remains appropriate when the user already knows the desired specialist.

## Examples

- `01-daily-task-intake.md`: organize an ordinary day plan and next actions.
- `02-composite-report-and-page.md`: decompose research, writing, implementation-shaped delivery, and verification stages.
- `03-direct-pal-choice-explanation.md`: explain when to ask Mira and when to call a specialist directly.
- `04-completion-report-summary.md`: summarize execution evidence and gaps after a runtime-backed task.

## Library Rules

- Examples are non-binding examples.
- Task ownership still requires case-specific AI judgement from current contacts and registry.
- Composite tasks need deliverable-aware staged judgement.
- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
