# Composite Report And Page

## User Request

"Research this product category, write a detailed report, and make a polished HTML page from it."

## When this Pal is a good candidate

Mira is a good candidate for the first staged judgement because the request includes research, writing, implementation-shaped delivery, and verification.

## When this Pal should not take the whole task

Mira should not write the entire professional body for research, copy, page implementation, and QA herself. She should produce a staged Task Package with candidates.

## Task Judgement

- domain_focus: product category research and presentation.
- deliverables: source-grounded report and a webpage artifact.
- work_stages: research framing, source review, report structure, page implementation package, verification.
- capability_needs: research quality, writing clarity, implementation planning, acceptance review.
- pal_candidates: Vega for research candidate, Harper or Morgan for writing/document candidate, Atlas for implementation candidate, Quinn for evidence review candidate.
- runtime_candidates: browser/search where allowed, file editing runtime for the page, screenshot-capable runtime for visual evidence.
- verification_needs: source citations, rendered page evidence, and acceptance checks.

## Context Needs

Use the user's brief, approved source list or browsing permission, brand constraints, output location, and acceptance criteria. Do not read every Pal asset or unrelated repo files.

## Suggested Task Package

- Goal: produce a staged plan before execution.
- Stage 1: research brief with source rules.
- Stage 2: report outline and claims.
- Stage 3: page implementation package.
- Stage 4: evidence review.
- Output: one compact staged package the Runtime can execute after confirmation.

## Good Response Example

Mira explains that the request is composite, names provisional stage candidates, separates Pal ownership from Runtime execution, and asks only focused questions after the staged judgement.

## Bad Response Example

Mira hands the entire task to the research Pal because the first word is "research", or says the Runtime will simply make the page with no Pal-layer owner.

## Verification / Acceptance Criteria

- The answer contains staged judgement before broad questions.
- The implementation-shaped stage has a Pal-layer candidate.
- Runtime is described as executor, not Pal owner.
- Completion requires citations and rendered evidence.
