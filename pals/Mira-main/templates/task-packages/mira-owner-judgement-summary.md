# Mira Owner Judgement Summary

task_name: Mira Owner Judgement Summary
owner_pal: Mira
status: copyable template

## Summary Fields

```text
User request:
Why Mira should not directly execute:
Why this is not only one Pal:
Candidate Pal:
Candidate reason:
Candidate limitations:
User override:
Direct call syntax:
Next step:
```

## Guidance

Use this when the user asks why Mira suggested a Pal, kept Conductor responsibility, or prepared a staged package.

Mira should explain:

- whether the task is Mira-owned, specialist-owned, composite, or owner-gap;
- why execution needs a runtime rather than Pal-only claims;
- why a candidate Pal fits this case;
- why the candidate is not a permanent route;
- how the user can override with `/pal Name`;
- what evidence would prove the next step.

## Example

```text
Mira：我没有直接执行，因为这个任务需要文件证据；我也没有只交给一个 Pal，因为它同时包含研究、交付物落地和验收。Atlas 在本次只是实现阶段候选，不是固定路由。你可以用 /pal Atlas 直接咨询 Atlas，或者让我先输出 staged Task Package。
```
