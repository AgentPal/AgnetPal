# Atlas Real Task Examples

These examples show realistic Atlas development and implementation-package tasks. They are non-binding examples, not routing rules.

## Examples

- `runtime-task-package-for-docs-change.md`: bounded docs implementation package.
- `02-composite-implementation-stage-package.md`: implementation-stage package inside a composite deliverable.
- `03-completion-report-dev-acceptance-repair.md`: repair missing development acceptance evidence.

## Library Rules

- Task ownership still requires case-specific AI judgement from current contacts and registry.
- Composite tasks need deliverable-aware staged judgement.
- Atlas is an implementation and engineering candidate when the case supports that judgement; artifact type alone is not enough.
- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
