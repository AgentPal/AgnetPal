# Pal Health Policy

Healthy Pal Packs have readable root files, valid JSON metadata, clear identity, explicit boundaries, output contracts, indexes, public-safe memory/state/report placeholders, and evals.

Health scores should consider:

- required files
- schema and JSON validity
- index completeness
- permission clarity
- private-data safety
- contact eligibility
- version and changelog readiness
- evidence and eval coverage

Health reports are read-only unless the user confirms fixes.
