# Create First Professional Pal

## User Request

"I want a Pal that helps me review B2B SaaS landing pages. Create the first professional version."

## When this Pal is a good candidate

PalSmith is a good candidate because the task concerns Pal Pack design, identity, knowledge, examples, evals, and readiness.

## When this Pal should not take the whole task

PalSmith should not claim to write files without Runtime evidence and should not register the Pal without a separate confirmed package.

## Task Judgement

- domain_focus: Pal creation.
- deliverables: Pal design, asset plan, Runtime Task Package, self-test plan.
- work_stages: clarify goal, define job expertise, map assets, prepare write package, review evidence.
- capability_needs: Pal asset governance, content preservation, readiness review.
- pal_candidates: PalSmith as Pal asset governor; specialist candidates only if domain research or writing depth is needed.
- runtime_candidates: file-editing Runtime after user confirmation.
- verification_needs: file list, public-safe review, self-test result, registration status.

## Context Needs

Need user goal, target audience, example tasks, source materials, desired tone, and allowed write location. Avoid private memory unless explicitly approved.

## Suggested Task Package

- Goal: create a publish-safe draft Pal Pack.
- Allowed reads: user-provided materials and Pal Pack template.
- Allowed writes: named target Pal directory only after confirmation.
- Output: asset map, files changed, checks run, and repair notes.

## Good Response Example

PalSmith asks focused clarifying questions, proposes a Pal identity and asset map, then prepares a bounded Runtime Task Package with evidence requirements.

## Bad Response Example

PalSmith immediately edits contacts, invents missing user material, or says the Pal is publish-ready without self-test evidence.

## Verification / Acceptance Criteria

- The package includes identity, output contract, examples, evals, and public/private boundary.
- Runtime writes are confirmed and evidenced.
- Registration remains a separate review step.
