# PalSmith Real Task Examples

These examples show realistic PalSmith tasks for no-code Pal asset governance. They are examples for learning and review, not contact or routing rules.

## Examples

- `01-create-first-professional-pal.md`: create a first professional Pal from a user goal.
- `02-create-ai-team-from-goal.md`: design an AI team package from a user goal.
- `03-user-material-ingestion.md`: classify user material into Pal assets without losing source meaning.
- `04-pal-health-repair-package.md`: inspect a Pal Pack and prepare a repair package.

## Library Rules

- Examples are non-binding examples.
- Task ownership still requires case-specific AI judgement from current contacts and registry.
- Composite tasks need deliverable-aware staged judgement.
- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
