# Create AI Team From Goal

## User Request

"Build an AI team for launching a niche newsletter: research topics, write issues, manage sponsor copy, and check quality."

## When this Pal is a good candidate

PalSmith is a good candidate because the work is AI team design and Pal asset governance, not direct newsletter production.

## When this Pal should not take the whole task

PalSmith should not make a static team-to-task table or claim multiple Pals will execute automatically in current v0.2.

## Task Judgement

- domain_focus: AI team design.
- deliverables: team blueprint, candidate responsibilities, asset plan, governance notes.
- work_stages: clarify user goal, define capability areas, design Main Pal and specialist candidates, prepare creation package.
- capability_needs: Pal team design, conflict detection, no-code governance.
- pal_candidates: PalSmith for team blueprint; Mira candidate for Main Pal pattern if the team needs an entry conductor.
- runtime_candidates: file-editing Runtime after confirmation.
- verification_needs: no role overlap, no fixed routes, public-safe assets.

## Context Needs

Need newsletter goal, cadence, audience, source policy, tone, sponsor boundary, and desired team size. Do not use private project material without explicit approval.

## Suggested Task Package

- Goal: design a Pal team package, not run the team.
- Output: team roster proposal, boundaries, example prompts, eval plan, creation steps.
- Evidence required: generated assets and review notes if Runtime writes files.

## Good Response Example

PalSmith describes candidate roles, a Main Pal pattern, collaboration boundaries, and a creation package that the Runtime can execute after approval.

## Bad Response Example

PalSmith says each newsletter task will be automatically handled by a fixed member or treats the blueprint as an active multi-agent runtime.

## Verification / Acceptance Criteria

- The team blueprint is no-code and candidate-based.
- It includes conflict checks and eval ideas.
- It does not claim automatic multi-agent execution.

## Notes

- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
