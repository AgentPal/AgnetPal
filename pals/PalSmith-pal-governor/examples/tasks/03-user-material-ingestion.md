# User Material Ingestion

## User Request

"I pasted a long operating manual. Turn it into Pal knowledge, examples, and eval ideas without losing important instructions."

## When this Pal is a good candidate

PalSmith is a good candidate because it owns content-preserving material ingestion into Pal assets.

## When this Pal should not take the whole task

PalSmith should not summarize away source obligations, expose private content publicly, or write assets without approved boundaries.

## Task Judgement

- domain_focus: source material to Pal assets.
- deliverables: source inventory, asset mapping, extraction plan, Runtime Task Package.
- work_stages: inventory, classify, preserve critical instructions, map to assets, request evidence.
- capability_needs: content preservation, public/private boundary, Pal Pack structure.
- pal_candidates: PalSmith; Morgan as document-structure consultant candidate when source organization is the main issue.
- runtime_candidates: file-editing Runtime after confirmation.
- verification_needs: source coverage, exclusion list, public-safe check.

## Context Needs

Need the supplied material, target Pal, permitted asset locations, and sensitivity labels. Do not read unrelated memories or reports.

## Suggested Task Package

- Goal: convert approved material into traceable Pal assets.
- Steps: source inventory, extraction table, asset mapping, write package, coverage review.
- Output: changed files, omitted material list, follow-up repair items.

## Good Response Example

PalSmith preserves source meaning, marks sensitive material, proposes exact asset destinations, and requires evidence before saying ingestion is complete.

## Bad Response Example

PalSmith compresses the manual into a vague persona paragraph or copies sensitive operational details into public examples.

## Verification / Acceptance Criteria

- Important instructions remain traceable.
- Sensitive material is excluded or explicitly approved.
- The output includes source coverage and not-run checks.

## Notes

- Candidates are not fixed routes.
- Current v0.2 does not auto-run Deep Conductor.
