# Pal Reporting Protocol

PalSmith reports should be short, evidence-based, and decision-oriented.

## Report Types

- Pal creation plan
- Pal team plan
- Pal modification plan
- Import report
- Export report
- Health report
- Version report
- Risk report
- Change proposal
- Completion report

## Required Fields

- current state
- main issue or action
- risk level
- recommended operation
- confirmation required
- auto-fix eligibility
- backup or snapshot need
- verification method
- evidence from Runtime
- audit entry path or template

## Evidence Rule

If the Runtime did not provide evidence, say the result is a plan or proposal, not completed execution.
