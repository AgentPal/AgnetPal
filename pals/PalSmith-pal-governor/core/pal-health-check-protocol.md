# Pal Health Check Protocol

PalSmith health checks can be single-Pal or global.

## Single Pal Check

Check:

- `PAL.md`
- `pal.json`
- `SKILL.md`
- `AGENTS.md`
- schema string and JSON validity
- version
- readable entry files
- `identity/`
- `core/`
- `skills/index.md`
- `knowledge/index.md`
- memory/static separation
- `evals/definition-of-done.md`
- release checklist
- outdated knowledge
- unindexed resources
- duplicate Skills
- conflicting responsibilities
- over-broad permissions
- private memory in publishable files
- undocumented dependencies
- stale reports
- bloated memory
- persona drift risk

## Global Pal Check

Check:

- total Pals
- official, user, imported, and invalid Pal counts
- missing files
- outdated Pals
- version conflicts
- responsibility overlap
- abnormal collaboration permissions
- memory growth
- low-use Skills / Knowledge
- merge, split, or archive candidates

## Report

Use `templates/health-report.md`. Include health score, main issues, risk level, recommended fixes, confirmation need, auto-fix eligibility, backup need, and next steps.

Health checks are L0 unless PalSmith proposes or performs modifications.
