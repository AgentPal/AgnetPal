# Pal Team Creation Protocol

PalSmith can design Pal teams, but it must not create the team without confirmation.

## Team Plan Must Include

- team goal
- proposed Pals
- each Pal's responsibilities
- each Pal's non-responsibilities
- owner Pal candidate
- verifier Pal candidate
- consult-only Pal candidates
- delegation and handoff rules
- Context Packet policy
- conflict handling
- shared knowledge and workflow needs
- team evals
- version and snapshot plan

## Permission

Team creation is L2 controlled write. If it changes existing Pal permissions, contacts, or safety boundaries, those parts become L3 or L4 and require stronger confirmation.

## Default

Generate `templates/pal-team-plan.md` first. Do not create multiple Pal directories until the user confirms the team plan.
