# Verification

## Purpose

Use this directory for reusable verification checklists and PalBench-style
cases.

## What Belongs Here

- Acceptance checks.
- Boundary checks.
- Evidence requirements.
- Pass, fail, not-run, blocked, missing, and unknown handling rules.

## What Does Not Belong Here

- False pass claims.
- Private evidence.
- Runtime logs containing secrets.
- Executable validators.

## Relationship To Pal Assets

Verification may reference Pal-owned evals as candidates, but it does not
replace them.

## Relationship To Project Records

Project-specific verification results belong in private project records or
approved reports.

## Public Safety Boundary

Record reusable checks, not private proof.
