# Private Boundary

## Purpose

This file explains what must stay out of a reusable public Org Pack.

## Customer-Private Assets

Do not include:

- real customer data;
- customer credentials;
- private tokens, cookies, passwords, or API keys;
- real account names or system identifiers;
- private project memory;
- private project reports;
- private evidence;
- customer-specific business rules.

## Approved Targets

Customer-private materials belong in approved private project records,
customer-private delivery records, or explicitly approved evidence locations.

## Reusable Assets

Reusable Org Pack assets should be de-identified, generic, and safe to publish.

## External Project Boundary

Do not copy this Org Pack into an external project `.agentpal/` directory by
default.
