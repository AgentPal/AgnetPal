# Verification Checklist

Use this checklist before reusing, publishing, or adapting an FDE Pack.

| Check | Status | Evidence |
| --- | --- | --- |
| Required files exist | not-run | |
| `fde.json` parses | not-run | |
| Required boundary flags are false where required | not-run | |
| `requires_human_review=true` | not-run | |
| Reusable assets are public-safe | not-run | |
| Customer-private assets are excluded | not-run | |
| No credentials or secrets | not-run | |
| No connector / scanner / validator / marketplace program | not-run | |
| No keyword routing map | not-run | |
| Update policy exists | not-run | |
| High-risk domain human review boundary exists | not-run | |

Use `pass`, `fail`, `missing`, `not-run`, `blocked`, or `requires-human-review`.
