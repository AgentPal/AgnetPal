# MCP Profiles

MCP profiles describe MCP-hosted tools or resources when a runtime exposes them. They record tool/resource shape, access boundary, permissions, risks, and evidence needs.

AgentPal does not automatically probe MCP servers. A profile is illustrative unless the user or maintainer marks it as maintained for a specific environment.

## Rules

- Do not treat MCP tools as Pals.
- Do not include private endpoint data or credentials.
- Do not use MCP profiles as automatic invocation rules.
- MCP use requires current Runtime availability evidence and a bounded Task Package when actions matter.
