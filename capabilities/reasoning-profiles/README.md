# Reasoning Profiles

Reasoning profiles describe when different reasoning strengths may be useful. They are prompt-shaping inputs, not fixed model or runtime rules.

## Example

- `reasoning-strengths.example.json`

## Rules

- Reasoning level is a candidate judgement, not an automatic setting.
- Use lower reasoning for low-risk formatting or simple summaries when appropriate.
- Use higher reasoning for composite deliverables, evidence review, public release checks, or ambiguous task ownership.
- Current Runtime availability and user constraints still matter.
