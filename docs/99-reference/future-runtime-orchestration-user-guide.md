# Future Runtime Orchestration User Guide

This guide explains how to read future orchestration material without mistaking it for current runtime execution.

## Current User Expectation

In AgentPal v0.5:

- ordinary messages go to Mira
- `/pal Name` enters the selected Pal's working mode
- Deep Conductor may structure a no-code collaboration and mode decision
- no separate runtime workflow is started or claimed by AgentPal itself
- no runtime-mode metadata should be shown in normal answers

## Future Direction

A later AgentPal version may design runtime adapters that coordinate child workflows or external execution runtimes. Until a host runtime provides verified capability and the user authorizes it, those ideas remain host-dependent future work.
