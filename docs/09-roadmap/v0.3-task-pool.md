# AgentPal v0.3 Task Pool

This task pool turns the v0.3 orchestration prototype plan into execution-ready packages. Each task stays inside the no-code Pal layer boundary.

## P0-1: Context Packet Standard

Status: started / first implementation / R52.

Goal: make Context Packet the standard structure for controlled Pal handoff and review work.

Modification scope:

- `orchestration/context-packet-protocol.md`
- `templates/orchestration/context-packet-template.md`
- selected examples and evals that need packet references

Must read:

- `orchestration/context-access-list-protocol.md`
- `orchestration/context-packet-minimalism-protocol.md`
- `orchestration/task-package-output-contract.md`
- `pals/Mira-main/core/output-contract.md`

Do not do:

- do not create a message bus;
- do not copy full chat history into packets;
- do not include private memory or secrets in public examples.

Acceptance criteria:

- protocol and template exist;
- required fields include schema, packet id, from Pal, to Pal candidate, mode, goal, state, constraints, can-read / cannot-read, output contract, verification, privacy, and expiration;
- template states candidate is not a fixed route;
- template states current v0.3 is no-code only.

Suggested model: GPT-5.4-Codex.

Reasoning strength: high.

Task length: medium.
