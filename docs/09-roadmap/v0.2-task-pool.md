# AgentPal v0.2 Task Pool

This task pool turns the v0.2 development plan into execution-ready packages. Each task stays inside the no-code Pal layer boundary.

## P0-1: PalSmith End-To-End Productization

Goal: make PalSmith usable as a complete path from user goal to Pal or AI team creation package.

Status: started / first implementation / R40. The first implementation adds copyable creation task packages, material ingestion mapping, health checks, repair templates, realistic examples, and an expanded self-test. This does not mark the full v0.2 roadmap complete.

Modification scope:

- `docs/06-palsmith/`
- `pals/PalSmith-pal-governor/examples/`
- `pals/PalSmith-pal-governor/templates/task-packages/`
- `pals/PalSmith-pal-governor/evals/`

Must read:

- `docs/PalSmith.md`
- `pals/PalSmith-pal-governor/PAL.md`
- `pals/PalSmith-pal-governor/core/output-contract.md`
- existing PalSmith task-package templates

Do not do:

- do not create PalSmith CLI, importer, exporter, scanner, validator, UI, or script;
- do not update contacts or registry as part of the creation package;
- do not copy private memory into examples.

Acceptance criteria:

- user-level E2E doc exists;
- 2 concrete PalSmith examples exist;
- 2 task-package templates exist;
- PalSmith self-test covers goal input, clarification, design, asset generation plan, self-test, repair, usage, and release advice.

Suggested model: GPT-5.4-Codex.

Reasoning strength: high.

Task length: long.
