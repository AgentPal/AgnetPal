# AgentPal v0.5 Local Development Scope

Date: 2026-06-28

## Scope

AgentPal v0.5 local development starts from the completed v0.4 central organization workspace foundation. The v0.5 theme is Org Pack / AI Organization Package foundations plus Pal Asset foundations for reusable no-code organization assets.

This is a local development scope freeze. It is not GitHub sync, not a tag, and not a GitHub Release.

## v0.5 Focus

v0.5 focuses on:

- Org Pack / AI Organization Package standards
- Pal Asset standards for single-Pal role assets
- PalSmith v0.5 no-code creation, classification, and upgrade standards
- Pal Asset Resolver no-code judgement standards
- Pal Metadata v0.5 standards for `pal.json` and optional `asset-manifest.json`
- official Pal asset audit and upgrade planning
- official Pal metadata upgrade preflight and compatibility gates
- reusable organization packages
- customer-private versus reusable asset boundaries
- FDE / expert delivery pack foundations
- cross-organization collaboration prerequisites
- no-code governance, verification, and memory policies

v0.5 continues the v0.4 boundaries:

- external projects stay thin-bound
- AgentPal central assets stay in the AgentPal Workspace
- Pal roster truth stays in central contacts
- ownership and routing remain AI judgement, not keyword routing
- host runtimes execute real actions and must provide evidence

## Out Of Scope

v0.5 local development does not add:

- GitHub sync
- GitHub Release
- tag
- marketplace / hub program
- CLI
- Web App
- Desktop App
- installer
- scanner
- validator
- daemon
- database
- auto sync
- auto execution engine

- keyword router or deterministic semantic router

## Org Pack Boundary

An Org Pack is a no-code organization asset package. It may contain reusable organization descriptions, recommended Pals, workflow candidates, governance rules, capability references, memory policy, verification checklists, and public-safe examples.

An Org Pack is not a Pal Pack, not a project template, not an installer, not a connector, not a database, not a marketplace program, not a scanner, not a validator, and not a keyword router.

## Pal Asset Boundary

Pal Assets describe one Pal's long-term role assets: identity, work protocols, knowledge, Pal Skills, workflows, runbooks, learning, memory, reports, evals, examples, and root entry files.

Pal Asset standards are not runtime code, not a connector, not a scanner, not a validator, and not a keyword router.

Pal Skill and Agent Skill remain separate:

- Pal Skill: role-level comprehensive work capability owned by a Pal.
- Agent Skill: host Runtime tool-level execution capability referenced as a candidate.

Pal `skills/` defaults to Pal Skill. Agent Skill references belong in Runtime capability refs, task packages, or tool registries.

## Pal Metadata Boundary

Pal Metadata v0.5 adds standards, schemas, templates, and validation gates for
`pal.json` and optional `asset-manifest.json`.

`pal.json` remains metadata. It is not a knowledge base, route map, task log,
private memory store, or runtime executor.

`asset-manifest.json` is an optional index. It helps humans and host runtimes
find Pal-owned assets from evidence, but it is not a scanner, validator,
connector, marketplace program, or proof of job readiness by itself.

Official Pals still load by existing root-entry and directory conventions until
an explicit small-batch upgrade round is approved and validated.

## External Project Boundary

External projects still receive only thin binding entry points by default:

- `.agentpal/project.json`
- `.agentpal/AGENTPAL.md`
- protected root instruction blocks
- optional Claude Code local settings

Org Pack assets must not be copied into external project `.agentpal/org-pack/`, `.agentpal/pals/`, `.agentpal/memory/`, `.agentpal/reports/`, `.agentpal/workflows/`, `.agentpal/evals/`, `.agentpal/capability-inventory/`, or `.agentpal/business-systems/` by default.

## Central Roster Boundary

Org Packs may recommend Pals, but recommendations are AI judgement inputs only. They are not routes, not assignments, and not central roster edits.

The central roster remains:

- `workspace/organization/contacts/pals.json`
- `workspace/organization/contacts/PAL_CONTACTS.md`
- `official/pals/`

## Customer-Private Boundary

Reusable Org Packs must not contain customer private data, customer credentials, real project secrets, customer system identifiers, private project memory, or private evidence.

Customer-specific facts belong in private central project records, customer-private records, or explicitly approved private evidence locations, not in a reusable public Org Pack.

## R101 Integration Freeze

R101 integrates R100-A/B/C/D navigation and freezes the preflight gate for the
next official Pal metadata upgrade round.

Before any future official Pal metadata or manifest edit:

- run the R100-D compatibility gate before changes;
- change one Pal or a small batch only;
- rerun the gate after changes;
- rerun selected R95 groups when shared entries, official Pal loading, or public
  safety surfaces are affected;
- preserve missing, not-run, and requires-human-review evidence.

R101 itself is still navigation and gate preparation. It does not authorize a
bulk official Pal rewrite.
