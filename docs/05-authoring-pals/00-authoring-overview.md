# Authoring Overview

## Purpose

This directory helps contributors create public-safe Pal Packs.

## Current Status

Short placeholder for v0.1.0-rc.1.

## Basic Flow

1. Design the Pal identity.
2. Write `pal.json`.
3. Write `PAL.md`.
4. Write root `SKILL.md`.
5. Write `core/output-contract.md`.
6. Add knowledge, skills, runbooks, examples, and evals as needed.
7. Review the public/private boundary.
8. Add the Pal to contacts and registry only when it is a valid Pal Pack.
