# What PalBench Does Not Prove

PalBench R33 is intentionally modest. Its value comes from being clear about what it does not prove.

## It Does Not Prove Statistical Significance

R33 used six small smoke samples. That is useful for release readiness and qualitative observation, but it is not a large benchmark run.

Do not use PalBench R33 as statistical proof of performance improvement.

## It Does Not Prove Model Capability

The comparison is about agent usage workflows. It is not a test of the underlying model's intelligence, coding ability, reasoning ability, or general capability.

Do not use PalBench R33 as a model comparison or model capability claim.

## It Does Not Prove General Product Performance

The samples cover specific workflow behaviors: task packaging, verification, context scope, single entry, routing reuse, and active/future boundary control.

They do not prove that AgentPal improves every task, every user workflow, or every runtime condition.

## It Does Not Activate Future Runtime Behavior

PalBench can describe Fast Route and future Deep Conductor boundaries. It does not activate Deep Conductor as current runtime behavior in v0.1.0-rc.1.

Current release behavior remains Simple Pal Mode only.

## Safe Claim

Use this claim:

```text
In a six-sample smoke validation, AgentPal showed observed benefits for task clarity, verification awareness, context-scope control, user routing burden, reusable routing records, and clear current/future workflow boundaries.
```
