# What AgentPal Is Not

AgentPal v0.5 has a strict boundary. This page states the non-goals clearly so readers do not mistake no-code protocols or host-runtime-dependent material for current AgentPal execution.

## AgentPal Is Not

- an Agent runtime
- a multi-Agent runtime
- an execution layer
- a desktop app
- a web app
- a hosted service
- a marketplace
- an automatic installer
- a daemon, scanner, or validator
- a connector layer
- a customer-system executor
- a token meter or cost calculator

## Not Active In v0.5

- automatic Deep Conductor execution
- automatic subagent launch
- automatic parallel child workflows
- automatic external Agent orchestration
- automatic runtime-wide scanning
- automatic Runtime Skill execution by AgentPal
- automatic project takeover
- automatic writeback to external projects
- automatic customer-data ingestion into public examples

## Important Language Boundary

- A Pal is not an Agent process.
- A Skill is not a Pal.
- `/pal Name` does not start a separate Agent.
- Deep Conductor is current as no-code collaboration and mode-decision protocol, not as automatic runtime execution.
- Future or host-dependent behavior must stay labeled as future, unavailable, blocked, not-run, or host-runtime-dependent.
