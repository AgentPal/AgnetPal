# Pal Metadata v0.5 Upgrade Path

Date: 2026-06-28

## Purpose

This guide connects the R100 Pal Metadata v0.5 standards, templates, official
Pal preflight plans, and R101 integration gate.

It is a navigation and upgrade-order document. It is not an official Pal upgrade
record and does not authorize bulk edits under `official/pals/**`.

## What Pal Metadata v0.5 Adds

Pal Metadata v0.5 has two layers:

- `pal.json` v0.5 metadata for one Pal Pack.
- optional `asset-manifest.json` indexes for Pal-owned assets.

Primary files:

- `standards/pal-asset/pal-json-v0.5-standard.md`
- `standards/pal-asset/asset-manifest-standard.md`
- `standards/schemas/pal.schema.json`
- `standards/schemas/pal-asset-manifest.schema.json`
- `templates/pal-asset/pal-json-template.json`
- `templates/pal-asset/asset-manifest-template.json`

## Boundary

`pal.json` is metadata. It is not a knowledge base, task log, memory store,
route map, runtime executor, or private data store.

`asset-manifest.json` is an optional index. It is not a scanner, validator,
connector, marketplace program, route map, or proof of job readiness by itself.

Both files must preserve:

- AI owner judgement instead of keyword routing;
- central roster as source of truth;
- thin external project binding;
- Pal Skill and Agent Skill separation;
- missing and not-run evidence;
- public safety and credential exclusion.

## Upgrade Order

1. Read the v0.5 standards and templates.
2. Run the official Pal metadata preflight.
3. Backfill safe index/README files only when explicitly approved.
4. Upgrade `pal.json` metadata for one Pal or a small batch.
5. Generate real `asset-manifest.json` files only after metadata and index
   evidence pass review.
6. Rerun the R100-D compatibility gate and selected R95 groups.
7. Commit only after evidence is recorded.
