# Pal Teams And Deep Conductor

This page explains how Pal Teams relate to Deep Conductor in AgentPal v0.5.

## Current Status

- Pal Team concepts are current as no-code collaboration methodology.
- Mira-first, Fast Route, and direct `/pal Name` calls are current.
- Deep Conductor is current as no-code collaboration and mode-decision protocol.
- Subagent calls and multi-Agent execution are host-runtime capabilities, not AgentPal Pal-layer capabilities.

## What A Pal Team Is

A Pal Team is a group of professional Pal Packs that coordinate work through the current host runtime.

A Pal Team can help with:

- task understanding
- ownership judgement
- context organization
- Task Package creation
- professional review
- risk checks
- verification standards
- result summaries
- repeated-task learning

These Pals do not all need to execute actions. Each Pal contributes a professional lens, bounded context needs, evidence requirements, or review standards.

## What A Pal Team Is Not

A Pal Team is not:

- a group of independent Agent runtimes
- every Pal becoming an Agent process
- a multi-Agent runtime by itself
- a desktop app, web app, installer, daemon, connector, scanner, or marketplace
- permission to call tools, run commands, or publish without runtime evidence and user approval

## Fast Route

Use Fast Route when one owner Pal can reasonably handle the task and deeper orchestration would add overhead without improving the result.

Owner selection is still case-specific AI judgement from the current central contacts.

## Deep Conductor

Use Deep Conductor-style judgement when a task has several stages, high risk, capability uncertainty, privacy concerns, verification needs, or multiple candidate owner/reviewer Pals.

Deep Conductor can decide:

- task complexity
- risk level
- host capability needs
- model and reasoning candidates
- Pal ownership and review needs
- context access boundaries
- verification path
- memory or Skill candidate handling

It does not start execution by itself.

## Relationship Summary

| Layer | Meaning | v0.5 status |
| --- | --- | --- |
| Skill | A capability or reusable method | Current |
| Pal | A working partner that organizes capability, context, judgement, and output | Current |
| Pal Team | Collaboration across multiple Pals | Current as no-code method |
| Subagent / multi-Agent execution | Runtime-layer execution capability | Host-dependent |
| Deep Conductor | No-code collaboration and mode-decision protocol | Current, not automatic execution |
