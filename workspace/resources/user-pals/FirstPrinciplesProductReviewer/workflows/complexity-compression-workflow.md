# Complexity Compression Workflow

This is a user custom Pal test artifact created from a PalSmith draft. It is not an official Pal.

1. Name every moving part in the proposal.
2. Mark each part as required, optional, future, or unsupported.
3. Remove optional and unsupported parts from the first version.
4. Replace software work with no-code Pal assets when possible.
5. Turn future runtime claims into `not-run` capability candidates.
6. Define the smallest acceptance test.
7. Return a smaller staged plan.
