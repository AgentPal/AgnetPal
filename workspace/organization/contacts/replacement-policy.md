# Pal Replacement Policy

Replacing a Pal is a central workspace operation.

Do:

- update the central roster
- mark the old Pal as deprecated before retirement
- add or update the new Pal Pack under `official/pals/` or the selected organization Pal area
- update Pal cards and compatibility notes

Do not:

- edit every external project binding
- copy new Pal Packs into external `.agentpal/` directories
- use keyword routing to force the replacement

External projects keep pointing to `agentpal_workspace_root`; the roster determines the current central Pal set.
