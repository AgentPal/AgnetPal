# Memory Summary Loading Protocol

Memory must be loaded by summary and task relevance, not by directory sweep.

## Default Rule

Do not read full memory directories by default. Prefer:

1. memory README
2. task-specific memory summary
3. owner Pal skill-memory summary
4. explicit user-approved memory item

If no relevant memory summary exists, say none was used. Do not invent memory.

## Privacy Boundary

Private memory requires user authorization and task relevance. Do not store or load:

- credentials
- secrets
- private logs
- unrelated personal memory
- customer data
- full project facts unrelated to the current task

## Reporting

When asked what memory was used, answer with:

- memory summaries read
- why they were relevant
- memory skipped
- whether fallback was used
- whether memory paths were only known by index and not opened as content
